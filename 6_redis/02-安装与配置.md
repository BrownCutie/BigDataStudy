## 二、方式一：Homebrew 安装（推荐）

```bash

brew install redis

# 验证安装
redis-server --version
# 输出示例：Redis server v=7.2.4 sha=00000000:0 malloc=jemalloc-5.3.0 bits=64
```

## 三、方式二：源码编译安装

如果你需要自定义编译参数，或者想体验最新版本：

```bash
# 1. 下载源码（以 7.2.4 为例）
curl -O https://github.com/redis/redis/archive/refs/tags/7.2.4.tar.gz

# 2. 解压
tar xzf 7.2.4.tar.gz
cd redis-7.2.4

# 3. 编译（macOS 自带 clang 编译器，不需要额外安装）
make

# 4. 编译完成后，可执行文件在 src/ 目录下
ls src/redis-server src/redis-cli

# 5. 可选：安装到系统目录
sudo make install
# 默认安装到 /usr/local/bin/

# 6. 测试编译是否成功
make test
```

### 4.2 后台启动（推荐）

```bash
# 修改配置文件，将 daemonize 改为 yes
# 或者直接启动时指定
redis-server --daemonize yes

# 后台启动并指定配置文件
redis-server /opt/homebrew/etc/redis.conf --daemonize yes
```

### 4.3 使用 Homebrew 服务管理

```bash
# 启动 Redis 服务（开机自启）
brew services start redis

# 停止 Redis 服务
brew services stop redis

# 重启 Redis 服务
brew services restart redis

# 查看服务状态
brew services info redis
```

### 4.4 停止 Redis

```bash
# 方式 1：通过 redis-cli 优雅关闭
redis-cli shutdown

# 方式 2：强制关闭（不推荐，可能丢失数据）
redis-cli shutdown nosave
# 或者
kill -9 $(pgrep redis-server)
```


## 五、redis-cli 基本使用

### 5.1 连接 Redis

```bash
# 连接本地 Redis（默认 127.0.0.1:6379）
redis-cli

# 连接指定主机和端口
redis-cli -h 127.0.0.1 -p 6379

# 连接带密码的 Redis
redis-cli -h 127.0.0.1 -p 6379 -a your_password

# 连接指定数据库（Redis 有 16 个数据库，编号 0-15）
redis-cli -n 1
```

### 5.2 基本命令

```bash
# 连接 Redis 后执行命令
127.0.0.1:6379> PING
PONG

# 设置键值对
127.0.0.1:6379> SET name "browncutie"
OK

# 获取值
127.0.0.1:6379> GET name
"browncutie"

# 查看所有键
127.0.0.1:6379> KEYS *
1) "name"

# 删除键
127.0.0.1:6379> DEL name
(integer) 1

# 检查键是否存在
127.0.0.1:6379> EXISTS name
(integer) 0

# 设置过期时间（10 秒）
127.0.0.1:6379> SET temp_key "hello" EX 10
OK

# 查看剩余过期时间
127.0.0.1:6379> TTL temp_key
(integer) 8
```

### 5.3 不进入交互模式直接执行

```bash
# 直接执行命令
redis-cli SET greeting "hello redis"
redis-cli GET greeting

# 执行多条命令
redis-cli SET count 100 INCR count INCR count GET count
```

## 六、redis.conf 核心配置详解

Redis 的配置文件非常丰富，我们只看最核心的那些。

```
│                  redis.conf 核心配置项                             │
├────────────────────┬──────────────┬───────────────────────────────┤
│       配置项        │   默认值     │          说明                  │
├────────────────────┼──────────────┼───────────────────────────────┤

│ port              │ 6379         │ 监听端口                        │
```

### 6.1 网络配置

```bash
# redis.conf 中的网络相关配置

# 绑定地址（安全配置！）
# 默认只绑定 127.0.0.1，只允许本机访问
bind 127.0.0.1

# 如果需要远程访问，改为：
# bind 0.0.0.0

# 监听端口
port 6379

# 密码认证（生产环境必须设置！）
requirepass your_strong_password_here

# 关闭保护模式（远程访问时需要）
# protected-mode no
```

```
┌──────────────────────────────────────────────────────────────┐
│                    安全配置提醒                                │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  生产环境必须做的事：                                           │
│  1. 设置密码：requirepass <强密码>                              │
│  2. 绑定内网 IP：bind <内网IP>，不要绑定 0.0.0.0               │
│  3. 使用防火墙限制访问来源                                      │
│  4. 禁用危险命令：FLUSHALL、CONFIG 等                          │
│  5. 不要用默认端口 6379                                        │
│                                                              │
│  本地开发环境：                                                │
│  - 保持默认配置即可                                            │
│  - bind 127.0.0.1 已经足够安全                                │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 6.2 内存配置

```bash
# redis.conf 中的内存相关配置

# 最大内存限制
# 建议设置为物理内存的 50%-70%，留一些给操作系统
maxmemory 2gb

# 内存淘汰策略（当内存满了怎么办）
# noeviction    - 不淘汰，写入直接报错（默认）
# allkeys-lru   - 从所有键中淘汰最近最少使用的
# volatile-lru  - 从设置了过期时间的键中淘汰最近最少使用的
# allkeys-lfu   - 从所有键中淘汰使用频率最低的（Redis 4.0+）
# volatile-lfu  - 从设置了过期时间的键中淘汰使用频率最低的
# allkeys-random - 从所有键中随机淘汰
# volatile-random - 从设置了过期时间的键中随机淘汰
# volatile-ttl  - 淘汰即将过期的键
maxmemory-policy allkeys-lru
```

### 6.3 持久化配置

```bash
# RDB 持久化（快照）
# 以下三个条件满足任一即触发快照
save 900 1        # 900 秒内有至少 1 次修改
save 300 10       # 300 秒内有至少 10 次修改
save 60 10000     # 60 秒内有至少 10000 次修改

# RDB 文件名
dbfilename dump.rdb

# RDB 文件存放目录
dir /opt/homebrew/var/db/redis/

# AOF 持久化
appendonly yes                    # 开启 AOF
appendfilename "appendonly.aof"   # AOF 文件名
appendfsync everysec              # 每秒同步一次（推荐）
# appendfsync always              # 每次写入都同步（最安全，但最慢）
# appendfsync no                  # 由操作系统决定何时同步（最快）
```

### 6.4 日志配置

```bash
# 日志级别
# debug    - 大量信息，开发调试用
# verbose  - 有用信息，比 debug 少
# notice   - 适度详细，生产环境推荐
# warning  - 只记录警告和错误
loglevel notice

# 日志文件路径（空字符串表示输出到标准输出）
logfile /opt/homebrew/var/log/redis.log
```

## 九、面试题

### Q1：Redis 的默认端口是多少？为什么是 6379？

**参考答案：**

Redis 默认端口是 **6379**。

这个数字的来源很有趣：6379 对应手机九宫格键盘上字母 **MERZ** 的位置。

```
┌───────┬───────┬───────┐
│  M    │  E    │  R    │
│  6    │  3    │  7    │
├───────┼───────┼───────┤
│       │  Z    │       │
│       │  9    │       │
├───────┼───────┼───────┤
│  1    │  2    │  3    │
└───────┴───────┴───────┘
```

MERZ 是 Redis 作者 Salvatore Sanfilippo 的偶像 -- 意大利女歌手 Alessia Merz 的名字。这只是一个有趣的彩蛋，没有技术含义。

在生产环境中，建议不要使用默认端口，以减少被扫描攻击的风险。

---

### Q2：Redis 有多少个数据库？生产环境应该用多个数据库吗？

**参考答案：**

Redis 默认有 **16 个数据库**（编号 0-15），可以通过 `databases` 配置项修改。

但**生产环境不建议使用多个数据库**，原因如下：

1. **Redis 是单线程的**：多个数据库共享同一个线程，FLUSHDB 操作会阻塞其他数据库
2. **不适合逻辑隔离**：多个数据库只是简单的命名空间隔离，没有权限隔离
3. **集群模式不支持**：Redis Cluster 只支持 database 0
4. **调试困难**：在数据库间切换容易出错

**正确的做法：** 使用不同的 Redis 实例或不同的 key 前缀来隔离数据。例如用 `demo:article:1001`、`demo:user:1001` 这种前缀来区分。

---

### Q3：如何安全地配置生产环境的 Redis？

**参考答案：**

1. **绑定内网 IP**：`bind 10.0.0.1`，不要绑定 0.0.0.0
2. **设置密码**：`requirepass <复杂密码>`，至少 16 位
3. **禁用危险命令**：通过 `rename-command` 重命名 FLUSHALL、CONFIG 等命令
4. **关闭保护模式**：远程访问时需要 `protected-mode no`（配合 bind 和密码使用）
5. **修改默认端口**：避免使用 6379
6. **开启防火墙**：只允许特定 IP 访问 6379 端口
7. **定期备份数据**：配置 RDB + AOF 持久化
8. **限制内存**：设置 `maxmemory`，防止 OOM
