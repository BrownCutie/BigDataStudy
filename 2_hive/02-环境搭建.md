# Hive 4.0.1 环境搭建详解

> Hive 4.0.1 学习系列 - 第 2 篇
> 适用环境：macOS Apple Silicon / JDK 21 / MySQL 8.0 / Hive 4.0.1

## 二、MySQL 准备

### 2.1 为什么需要 MySQL

Hive 默认使用内嵌的 Derby 数据草稿储元数据。Derderby 的限制是同一时间只能有一个进程连接，这在生产环境中完全不可用。因此需要替换为 MySQL：

```
┌────────────────────────────────────────────────────────────┐
│                Metastore 元数据存储对比                     │
│                                                            │
│   Derby（默认）              MySQL（推荐生产）              │
│   ┌──────────┐              ┌──────────────┐               │
│   │ 单用户   │              │ 多用户并发   │               │
│   │ 嵌入式   │              │ 独立服务     │               │
│   │ 无需配置 │              │ 需要配置     │               │
│   │ 仅开发用 │              │ 生产推荐     │               │
│   │ 数据丢失 │              │ 数据持久化   │               │
│   └──────────┘              └──────────────┘               │
│                                                            │
│   Metastore 存储的元数据包括：                              │
│   - 数据库列表（DATABASES）                                │
│   - 表定义（TABLES）：表名、列、数据类型、分区              │
│   - 存储位置（SDS）：HDFS 路径、输入/输出格式              │
│   - 分区信息（PARTITIONS）                                │
│   - 表参数（TABLE_PARAMS）：压缩、SerDe 等                 │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### 2.2 创建 Metastore 数据库和用户

```sql
-- 登录 MySQL
mysql -u root -p

-- 创建 Hive Metastore 专用数据库
-- 注意：Hive 4.x 要求使用 utf8mb4 字符集
CREATE DATABASE hive_metastore
    DEFAULT CHARACTER SET utf8mb4
    DEFAULT COLLATE utf8mb4_general_ci;

-- 创建专用用户（替换为你的密码）
CREATE USER 'hive'@'%' IDENTIFIED BY 'Hive_123456';

-- 授予权限
GRANT ALL PRIVILEGES ON hive_metastore.* TO 'hive'@'%';
GRANT ALL PRIVILEGES ON hive_metastore.* TO 'hive'@'localhost';

-- 刷新权限
FLUSH PRIVILEGES;

-- 验证
SHOW DATABASES;
SELECT User, Host FROM mysql.user WHERE User = 'hive';
```

## 三、Hive 安装

### 3.1 下载和解压

```bash
# 下载 Hive 4.0.1（从 Apache 镜像站）
cd /opt
wget https://archive.apache.org/dist/hive/hive-4.0.1/apache-hive-4.0.1-bin.tar.gz

# 解压
tar -zxvf apache-hive-4.0.1-bin.tar.gz

# 重命名为简洁路径
mv apache-hive-4.0.1-bin hive/4.0.1

# 设置环境变量（追加到 ~/.zshrc）
export HIVE_HOME=/opt/hive/4.0.1
export PATH=$PATH:$HIVE_HOME/bin

# 生效
source ~/.zshrc

# 验证
hive --version
```


## 五、hive-env.sh 配置详解

`hive-env.sh` 是 Hive 的环境变量配置文件，控制 JVM 参数、路径等：

```bash
# /opt/hive/4.0.1/conf/hive-env.sh

# ==================== 路径配置 ====================

# Hadoop 安装目录（必须配置，Hive 需要 HDFS 和 YARN）
export HADOOP_HOME=/opt/hadoop/3.4.2

# Hive 配置文件目录
# 如果不设置，默认为 $HIVE_HOME/conf
export HIVE_CONF_DIR=/opt/hive/4.0.1/conf

# Hive 附加 JAR 包路径
# UDF JAR 包、自定义 SerDe 等可以放在这里
export HIVE_AUX_JARS_PATH=/opt/hive/4.0.1/lib

# ==================== 内存配置 ====================

# HiveServer2 JVM 堆内存（单位 MB）
# 生产环境建议 4096+，学习环境 1024 足够
export HIVE_SERVER2_HEAPSIZE=1024

# Hive CLI 堆内存（默认 256MB，大查询时需要增大）
# export HADOOP_HEAPSIZE=2048

# Metastore JVM 堆内存
# export METASTORE_HEAPSIZE=2048
```

### 5.1 --add-opens 原理详解

从 JDK 9 开始，Java 引入了模块系统（JPMS，Java Platform Module System）。JDK 16+ 默认启用强封装（Strong Encapsulation），限制反射访问内部 API。Hive 依赖的一些第三方库（如 Guava、Xerces）会通过反射访问 JDK 内部类，在 JDK 21 下会报 `InaccessibleObjectException` 错误：

```
错误示例（不加 --add-opens 时）：
java.lang.reflect.InaccessibleObjectException:
  Unable to make field private java.lang.String java.net.URI.string
  accessible: module java.base does not "opens java.net" to unnamed module
```

`--add-opens` 的语法为：

```
--add-opens <模块名>/<包名>=<目标模块>
```

- `模块名/包名`：要开放反射访问的模块和包
- `ALL-UNNAMED`：允许所有未命名模块（即我们的应用代码和第三方 JAR）访问

```bash
# hive-env.sh 中的 JDK 21 兼容性配置
export HADOOP_OPTS="$HADOOP_OPTS \
  --add-opens java.base/java.nio=ALL-UNNAMED \
  --add-opens java.base/java.lang=ALL-UNNAMED \
  --add-opens java.base/java.lang.reflect=ALL-UNNAMED \
  --add-opens java.base/java.util=ALL-UNNAMED \
  --add-opens java.base/java.net=ALL-UNNAMED \
  --add-opens java.base/java.io=ALL-UNNAMED \
  --add-opens java.base/java.util.concurrent=ALL-UNNAMED \
  --add-opens java.base/java.text=ALL-UNNAMED \
  --add-opens java.base/sun.security.util=ALL-UNNAMED \
  --add-opens java.base/sun.security.x509=ALL-UNNAMED"
```

各包的作用说明：

```
+----------------------------+-----------------------------------+
|          包名               |         被谁需要                  |
+----------------------------+-----------------------------------+
| java.nio                   | HDFS 客户端 NIO 操作             |
| java.lang                  | 反射访问 Class、String 等         |
| java.lang.reflect          | 反射调用 Method、Field 等         |
| java.util                  | 集合框架内部实现                  |
| java.net                   | URI/URL 解析                      |
| java.io                    | 文件 I/O 操作                     |
| java.util.concurrent       | 线程池、并发工具类                |
| java.text                  | 日期/数字格式化                   |
| sun.security.util          | 安全相关（Derby/Hadoop Auth）     |
| sun.security.x509          | 证书处理                          |
+----------------------------+-----------------------------------+
```

## 六、hive-site.xml 关键配置详解

`hive-site.xml` 是 Hive 最核心的配置文件。以下列出关键配置项并逐条解释：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<configuration>

    <!-- ==================== Metastore 配置 ==================== -->

    <!--
        Metastore 服务地址（Thrift URI）
        - 格式：thrift://<host>:<port>
        - 默认端口 9083
        - 如果为空，Hive 使用本地内嵌 Metastore（仅单用户）
        - 设置后，HiveServer2 和 CLI 通过 Thrift RPC 连接远程 Metastore
    -->
    <property>
        <name>hive.metastore.uris</name>
        <value>thrift://localhost:9083</value>
    </property>

    <!--
        Metastore 连接 MySQL 的 JDBC URL
        - 必须包含 createDatabaseIfNotExist=true
        - 建议使用 utf8mb4 字符集避免编码问题
        - useSSL=false 避免 SSL 警告（生产环境建议开启）
    -->
    <property>
        <name>javax.jdo.option.ConnectionURL</name>
        <value>jdbc:mysql://localhost:3306/hive_metastore?createDatabaseIfNotExist=true&amp;useSSL=false&amp;characterEncoding=utf8mb4&amp;serverTimezone=Asia/Shanghai</value>
    </property>

    <!--
        MySQL JDBC 驱动类名
        - MySQL 5.x：com.mysql.jdbc.Driver
        - MySQL 8.x：com.mysql.cj.jdbc.Driver
    -->
    <property>
        <name>javax.jdo.option.ConnectionDriverName</name>
        <value>com.mysql.cj.jdbc.Driver</value>
    </property>

    <!--
        Metastore 连接 MySQL 的用户名
    -->
    <property>
        <name>javax.jdo.option.ConnectionUserName</name>
        <value>hive</value>
    </property>

    <!--
        Metastore 连接 MySQL 的密码
        生产环境建议使用加密存储或配置管理工具
    -->
    <property>
        <name>javax.jdo.option.ConnectionPassword</name>
        <value>Hive_123456</value>
    </property>

    <!-- ==================== HiveServer2 配置 ==================== -->

    <!--
        HiveServer2 绑定地址
        - 0.0.0.0 允许所有 IP 连接（远程访问）
        - localhost 仅允许本机连接
    -->
    <property>
        <name>hive.server2.thrift.bind.host</name>
        <value>0.0.0.0</value>
    </property>

    <!--
        HiveServer2 Thrift 端口
        - 默认 10000
        - Beeline 和 JDBC 客户端通过此端口连接
    -->
    <property>
        <name>hive.server2.thrift.port</name>
        <value>10000</value>
    </property>

    <!--
        HiveServer2 Web UI 端口
        - 默认 10002
        - 浏览器访问查看服务状态
    -->
    <property>
        <name>hive.server2.webui.port</name>
        <value>10002</value>
    </property>

    <!-- ==================== 连接池配置 ==================== -->

    <!--
        Metastore 连接池大小
        - 控制与 MySQL 的最大连接数
        - 默认值通常太小，建议调大到 10-50
    -->
    <property>
        <name>datanucleus.connectionPool.maxPoolSize</name>
        <value>10</value>
    </property>

    <!-- ==================== HDFS 配置 ==================== -->

    <!--
        Hive 在 HDFS 上的工作目录
        - 用于存放查询中间结果、临时文件
        - 每个用户会创建子目录：/tmp/hive/<username>
    -->
    <property>
        <name>hive.exec.scratchdir</name>
        <value>/tmp/hive</value>
    </property>

    <!--
        Hive 默认数据仓库目录
        - 内部表数据默认存储在此目录下
        - 格式：/user/hive/warehouse/<database>.db/<table>
    -->
    <property>
        <name>hive.metastore.warehouse.dir</name>
        <value>/user/hive/warehouse</value>
    </property>

</configuration>
```

### 6.1 配置优先级

```
┌─────────────────────────────────────────────────────────────┐
│                   Hive 配置加载优先级（高 → 低）             │
│                                                             │
│  1. hive-site.xml          ← 最高优先级，用户自定义配置     │
│  2. hive-default.xml       ← Hive 内置默认值               │
│  3. hadoop-site.xml        ← Hadoop 配置（如 core-site.xml）│
│  4. JVM 系统属性           ← -D 参数                       │
│  5. hivevar / hiveconf     ← 命令行 --hivevar / --conf     │
│                                                             │
│  原则：hive-site.xml 中只需配置与默认值不同的项             │
│  hive-site.xml 不存在的配置项会自动使用 hive-default.xml    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 七、Metastore 初始化

### 7.1 schematool 命令详解

`schematool` 是 Hive 提供的 Metastore 数据库管理工具：

```bash
# 查看 schematool 帮助
schematool -help

# 子命令列表：
#   info        - 显示 Metastore 信息（数据库类型、版本等）
#   initSchema  - 初始化元数据表结构
#   upgradeSchema - 升级元数据表结构
#   create      - 创建 Metastore 数据库（仅限某些 DB）
#   validate    - 验证元数据表结构是否完整
#   dbOptions   - 显示数据库选项
```

### 7.2 初始化步骤

```bash
# 步骤1：确认 MySQL JDBC 驱动已就位
ls /opt/hive/4.0.1/lib/mysql-connector*.jar

# 步骤2：查看 Metastore 当前信息
schematool -info -dbType mysql

# 步骤3：初始化 Metastore（创建所有元数据表）
# 注意：如果表已存在会报错，可先执行 validate 检查
schematool -initSchema -dbType mysql

# 预期输出：
# Starting metastore schema initialization to 4.0.1
# Initialization script completed
# schemaTool completed

# 步骤4：验证初始化结果
schematool -validate -dbType mysql

# 步骤5：查看 MySQL 中创建的表
mysql -u hive -p hive_metastore
mysql> SHOW TABLES;
# 应该看到 70+ 张表，包括：
#   DBS           - 数据库信息
#   TBLS          - 表信息
#   COLUMNS_V2    - 列信息
#   PARTITIONS    - 分区信息
#   SDS           - 存储描述信息
#   TABLE_PARAMS  - 表参数
#   VERSION       - Metastore 版本
```

## 八、启动服务

### 8.1 启动 Metastore

Metastore 是 Hive 的元数据管理服务，必须先启动：

```bash
# 前台启动（可以看到日志输出，适合调试）
hive --service metastore

# 后台启动（推荐）
nohup hive --service metastore > /opt/hive/4.0.1/logs/metastore.log 2>&1 &

# 查看进程
ps aux | grep metastore

# 查看端口是否监听
lsof -i :9083
# 或
netstat -an | grep 9083
```

### 8.2 启动 HiveServer2

HiveServer2 提供 Thrift 服务，供 Beeline 和 JDBC 客户端连接：

```bash
# 前台启动
hive --service hiveserver2

# 后台启动（推荐）
nohup hive --service hiveserver2 > /opt/hive/4.0.1/logs/hiveserver2.log 2>&1 &

# 查看进程
ps aux | grep hiveserver2

# 查看端口（Thrift 10000 + Web UI 10002）
lsof -i :10000
lsof -i :10002

# 访问 Web UI（浏览器打开）
# http://localhost:10002
```

### 8.3 启动顺序总结

```
正确的启动顺序：

  [1] HDFS          →  start-dfs.sh
         │
  [2] MySQL         →  确保 MySQL 服务运行中
         │
  [3] Metastore     →  hive --service metastore
         │              等待端口 9083 就绪
         │
  [4] HiveServer2   →  hive --service hiveserver2
         │              等待端口 10000 就绪
         │
  [5] Beeline       →  beeline -u jdbc:hive2://localhost:10000

  停止顺序（与启动相反）：
  HiveServer2 → Metastore → HDFS
```

## 九、Beeline 连接测试

### 9.1 Beeline 简介

Beeline 是 Hive 官方推荐的命令行客户端，基于 SQLLine 项目，通过 JDBC 连接 HiveServer2：

```
┌──────────────────────────────────────────────────────────────┐
│                   Beeline 工作原理                            │
│                                                              │
│  ┌──────────┐    JDBC     ┌───────────┐   Thrift   ┌──────┐ │
│  │ Beeline  │───────────→ │HiveServer2│──────────→│Meta  │ │
│  │ (客户端) │  :10000    │ (服务端)  │   :9083   │store │ │
│  └──────────┘            └─────┬─────┘           └──┬───┘ │
│                                │                    │     │
│                                │ HDFS              │ MySQL│
│                                ▼                    ▼     │
│                          ┌──────────┐         ┌────────┐  │
│                          │  HDFS    │         │  MySQL │  │
│                          │(数据文件)│         │(元数据)│  │
│                          └──────────┘         └────────┘  │
```

## 十、JDK 21 兼容性踩坑记录

### 10.1 常见错误及解决方案

#### 错误 1：InaccessibleObjectException

```
错误信息：
java.lang.reflect.InaccessibleObjectException: Unable to make
  field private final byte[] java.lang.String.value accessible:
  module java.base does not "opens java.lang" to unnamed module

原因：JDK 16+ 强封装，Hive/Guava 反射访问内部 API 被拒绝

解决：在 hive-env.sh 中添加 --add-opens 参数（见第五节）
```

#### 错误 2：NoClassDefFoundError (JDK 内部类)

```
错误信息：
java.lang.NoClassDefFoundError: Could not initialize class
  org.apache.hadoop.hive.ql.session.SessionState

原因：JDK 版本不兼容或 CLASSPATH 冲突

解决：
1. 确认使用 JDK 17 或 21
2. 检查 JAVA_HOME 是否正确设置
3. 确保 HADOOP_OPTS 中的 --add-opens 参数完整
```

#### 错误 3：Metastore 连接 MySQL 失败

```
错误信息：
javax.jdo.JDOFatalInternalException: Error creating transactional
  connection factory

原因：MySQL JDBC 驱动未放入 lib/ 目录

解决：
1. 下载 mysql-connector-j-8.x.jar
2. 放到 /opt/hive/4.0.1/lib/
3. 确认文件名不冲突
```

#### 错误 4：Apple Silicon 兼容性

```
错误信息：
Unable to load native-hadoop library for your platform

原因：Hadoop native library 未编译 ARM64 版本

解决：此为警告，不影响功能，可忽略
如需消除，设置环境变量：
export HADOOP_OPTS="$HADOOP_OPTS -Djava.library.path=$HADOOP_HOME/lib/native"
```

### 10.2 一键启动脚本

为了方便日常使用，可以编写启动脚本：

```bash
#!/bin/bash
# /opt/hive/4.0.1/bin/start-all.sh
# Hive 一键启动脚本

echo "=== 启动 Metastore ==="
nohup hive --service metastore > $HIVE_HOME/logs/metastore.log 2>&1 &
METASTORE_PID=$!
echo "Metastore PID: $METASTORE_PID"
echo $METASTORE_PID > $HIVE_HOME/conf/metastore.pid

# 等待 Metastore 启动完成
echo "等待 Metastore 就绪..."
for i in {1..30}; do
    if lsof -i :9083 > /dev/null 2>&1; then
        echo "Metastore 已启动 (端口 9083)"
        break
    fi
    sleep 1
done

echo "=== 启动 HiveServer2 ==="
nohup hive --service hiveserver2 > $HIVE_HOME/logs/hiveserver2.log 2>&1 &
HS2_PID=$!
echo "HiveServer2 PID: $HS2_PID"
echo $HS2_PID > $HIVE_HOME/conf/hiveserver2.pid

# 等待 HiveServer2 启动完成
echo "等待 HiveServer2 就绪..."
for i in {1..60}; do
    if lsof -i :10000 > /dev/null 2>&1; then
        echo "HiveServer2 已启动 (端口 10000)"
        break
    fi
    sleep 1
done

echo "=== 启动完成 ==="
echo "Metastore:  localhost:9083"
echo "HiveServer2: localhost:10000"
echo "Web UI:     http://localhost:10002"
echo ""
echo "使用 Beeline 连接："
echo "  beeline -u jdbc:hive2://localhost:10000 -n browncutie"
```
