## 二、下载与安装

### 2.1 下载 Flink

```bash
# 使用阿里云镜像加速下载
cd ~/Programme/claude-study
curl -O https://mirrors.aliyun.com/apache/flink/flink-2.2.0/flink-2.2.0-bin-scala_2.12.tgz
```

### 2.2 解压安装

```bash
# 创建安装目录
mkdir -p /opt/flink

# 解压
tar -xzf flink-2.2.0-bin-scala_2.12.tgz -C /opt/flink/

# 重命名为版本号目录
mv /opt/flink/flink-2.2.0 /opt/flink/2.2.0

# 清理安装包
rm flink-2.2.0-bin-scala_2.12.tgz
```

### 2.3 配置环境变量

```bash
# 添加到 ~/.zshrc
echo 'export FLINK_HOME=/opt/flink/2.2.0' >> ~/.zshrc
echo 'export PATH=$PATH:$FLINK_HOME/bin' >> ~/.zshrc

# 使配置生效
source ~/.zshrc

# 验证
flink --version
```

## 四、核心配置详解

Flink 的配置文件在 `/opt/flink/2.2.0/conf/` 目录下，最重要的文件是 `flink-conf.yaml`。

### 4.1 flink-conf.yaml 核心配置

```yaml
# ============================================================
# Flink 2.2.0 核心配置
# 配置文件路径：/opt/flink/2.2.0/conf/flink-conf.yaml
# ============================================================

# -------------------- JobManager 配置 --------------------

# JobManager 进程总内存（包含 JVM 堆内存和堆外内存）
jobmanager.memory.process.size: 1024m

# JobManager RPC 地址（TaskManager 连接 JobManager 用）
jobmanager.rpc.address: localhost

# JobManager RPC 端口
jobmanager.rpc.port: 6123

# -------------------- TaskManager 配置 --------------------

# TaskManager 进程总内存
taskmanager.memory.process.size: 1024m

# 每个 TaskManager 的 Slot 数量
# Slot 是资源分配的基本单元，每个 Slot 可以运行一个并行任务
# Slot 数量通常设为 CPU 核心数
taskmanager.numberOfTaskSlots: 2

# -------------------- Web UI 配置 --------------------

# Web UI 端口（默认 8081）
rest.port: 8081

# Web UI 绑定地址
rest.address: localhost

# -------------------- 并行度配置 --------------------

# 默认并行度（如果没有特别指定，所有算子使用这个并行度）
parallelism.default: 1

# -------------------- Checkpoint 配置 --------------------

# 是否启用 Checkpoint（默认关闭）
# execution.checkpointing.interval: 30s

# Checkpoint 存储路径
# state.checkpoints.dir: file:///opt/flink/checkpoints

# 状态后端（默认 HashMapStateBackend）
# state.backend: hashmap
```

### 7.2 使用 PyFlink 提交作业

```bash
# 安装 PyFlink
pip3 install apache-flink==2.2.0

# 创建一个简单的测试作业
cat > /tmp/test-flink.py << 'EOF'
from pyflink.datastream import StreamExecutionEnvironment

# 创建执行环境
env = StreamExecutionEnvironment.get_execution_environment()

# 从集合创建数据流
data_stream = env.from_collection([1, 2, 3, 4, 5])

# 对每个元素乘以 2
result = data_stream.map(lambda x: x * 2)

# 打印结果
result.print()

# 执行作业
env.execute("第一个 Flink 作业")
EOF

# 提交作业
flink run -py /tmp/test-flink.py
```

### 7.3 常用作业管理命令

```bash
# 查看运行中的作业
flink list

# 查看所有作业（包括已完成的）
flink list -a

# 查看计划中的作业
flink list -r

# 取消正在运行的作业（替换 <jobId>）
flink cancel <jobId>

# 带 Savepoint 取消作业
flink cancel -s <savepointPath> <jobId>

# 从 Savepoint 恢复作业
flink run -s <savepointPath> -py /path/to/job.py

# 保存 Savepoint（不停止作业）
flink savepoint <jobId> <savepointPath>
```

demo_events = [

## 九、面试题

### Q1：Flink 的 JobManager 和 TaskManager 分别负责什么？

**参考答案：**

**JobManager** 是 Flink 集群的主节点，包含三个核心组件：
- **Dispatcher**：接收客户端提交的作业，启动 JobMaster，并提供 REST 接口（Web UI）
- **ResourceManager**：管理 TaskManager 的资源（Slot），负责 Slot 的分配和回收
- **JobMaster**：管理单个作业的执行，包括算子链优化、调度 Task、协调 Checkpoint

**TaskManager** 是工作节点，负责：
- 执行具体的数据处理任务（Task）
- 管理本地的 Slot 资源
- 管理本地的状态数据（State）
- 向 JobManager 汇报心跳和状态

### Q2：什么是 Slot？Slot 和并行度的关系是什么？

**参考答案：**

Slot 是 TaskManager 中的资源分配单元。一个 TaskManager 可以有多个 Slot，每个 Slot 拥有固定份额的 TaskManager 内存。

Slot 和并行度的关系：
- 一个作业的**并行度**（Parallelism）决定了该作业需要多少个 Slot
- 如果并行度为 4，则需要 4 个 Slot 来运行
- 如果只有一个 TaskManager 且只有 2 个 Slot，则无法运行并行度为 4 的作业
- **Slot 共享**：Flink 允许同一个作业的不同算子共享 Slot，这样可以减少资源需求，比如一个完整的管道（Source → Map → Sink）可以运行在同一个 Slot 中

### Q3：Flink 的 Standalone 模式适合生产环境吗？

**参考答案：**

Standalone 模式可以用于生产环境，但通常只适合中小规模场景（TaskManager 在 10 台以内）。在大规模生产环境中，通常推荐使用 YARN 或 Kubernetes 模式，因为：

1. **资源管理**：YARN/K8s 可以更好地管理集群资源，支持动态伸缩
2. **高可用**：Standalone 模式实现高可用需要配置 ZooKeeper，而 YARN/K8s 天然支持
3. **多租户**：YARN/K8s 支持多个团队共享集群资源

不过，对于我们的数据分析项目学习环境，Standalone 模式完全够用。
