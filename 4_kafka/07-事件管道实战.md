## 二、消息格式设计（JSON Schema）

### 2.1 统一的消息信封格式

所有消息使用统一的"信封"格式，方便后续扩展：

```json
{
  "eventId": "uuid-v4-string",
  "eventType": "post_published",
  "version": "1.0",
  "timestamp": "2026-04-15T10:00:00.000Z",
  "source": "web-backend",
  "data": {
    "postId": 1,
    "title": "Kafka 入门指南",
    "author": "browncutie"
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `eventId` | string | 全局唯一的事件 ID（UUID v4），用于幂等去重 |
| `eventType` | string | 事件类型，决定消费者如何处理 |
| `version` | string | 消息格式版本，用于向后兼容 |
| `timestamp` | string | 事件发生时间（ISO 8601 格式） |
| `source` | string | 事件来源（web-backend、mobile-app 等） |
| `data` | object | 事件的具体数据（不同事件类型结构不同） |

```python
"""
数据平台后端 Flask + Kafka 集成
在 Flask 的路由中发送事件到 Kafka

import json
import uuid
from datetime import datetime, timezone
from flask import Flask, request, jsonify
from kafka import KafkaProducer

app = Flask(__name__)

# 初始化 Kafka 生产者
producer = KafkaProducer(
    bootstrap_servers=['localhost:9092'],
    acks='all',
    retries=3,
    enable_idempotence=True,
    compression_type='lz4',
    batch_size=32768,
    linger_ms=10,
    key_serializer=lambda k: k.encode('utf-8') if k else None,
    value_serializer=lambda v: json.dumps(v, ensure_ascii=False).encode('utf-8')
)


def send_event(topic, event_type, data, key=None):
    """
    发送事件到 Kafka
    统一封装，确保消息格式一致
    """
    event = {
        'eventId': str(uuid.uuid4()),
        'eventType': event_type,
        'version': '1.0',
        'timestamp': datetime.now(timezone.utc).isoformat(),
        'source': 'web-backend',
        'data': data
    }

    producer.send(topic, key=key, value=event)
    producer.flush()  # 在 API 中使用 flush 确保消息发出
    return event['eventId']


@app.route('/api/posts', methods=['POST'])
def create_post():
    """发布文章"""
    body = request.json
    post_id = save_post_to_db(body)

    # 发送文章发布事件
    event_id = send_event(
        topic='blog-events',
        event_type='post_published',
        data={
            'postId': post_id,
            'title': body['title'],
            'content': body['content'],
            'author': body['author'],
            'tags': body.get('tags', []),
            'category': body.get('category', '')
        },
        key=str(post_id)  # 用 postId 作为 Key

    return jsonify({'postId': post_id, 'eventId': event_id}), 201
```

### 3.2 批量采集行为日志

对于高频的浏览行为，可以使用批量发送提升性能：

```python
"""
行为日志批量采集器
攒够一批再发送，提升吞吐量
"""

import time
import threading
import json
from collections import deque
from kafka import KafkaProducer

class BatchEventCollector:
    """批量事件采集器"""

    def __init__(self, bootstrap_servers='localhost:9092',
                 topic='demo-logs', batch_size=100, flush_interval=5):
        self.topic = topic
        self.batch_size = batch_size
        self.flush_interval = flush_interval

        self.buffer = deque()
        self.lock = threading.Lock()

        self.producer = KafkaProducer(
            bootstrap_servers=[bootstrap_servers],
            acks='all',
            compression_type='lz4',
            batch_size=65536,  # 64KB
            linger_ms=50,
            value_serializer=lambda v: json.dumps(v, ensure_ascii=False).encode('utf-8')
        )

        # 启动后台刷写线程
        self._running = True
        self._flush_thread = threading.Thread(target=self._flush_loop, daemon=True)
        self._flush_thread.start()

    def collect(self, event):
        """采集一条事件"""
        with self.lock:
            self.buffer.append(event)

            if len(self.buffer) >= self.batch_size:
                self._flush()

    def _flush(self):
        """刷写缓冲区"""
        with self.lock:
            if not self.buffer:
                return

            batch = list(self.buffer)
            self.buffer.clear()

        # 批量发送
        for event in batch:
            self.producer.send(self.topic, value=event)

        self.producer.flush()
        print(f"批量发送 {len(batch)} 条事件")

    def _flush_loop(self):
        """定时刷写"""
        while self._running:
            time.sleep(self.flush_interval)
            self._flush()

    def close(self):
        """关闭采集器"""
        self._running = False
        self._flush()  # 刷写剩余数据
        self.producer.close()


# 使用示例
collector = BatchEventCollector(batch_size=50, flush_interval=3)

# 模拟高频浏览事件
for i in range(200):
    collector.collect({
        'eventType': 'post_viewed',
        'data': {'postId': i % 10, 'userId': 100 + i % 50}
    })

collector.close()
```

### 4.1 搜索索引更新消费者

```python
"""
搜索索引消费者
从 Kafka 消费用户事件，更新 Elasticsearch 索引
"""

import json
import logging
import requests
from kafka import KafkaConsumer
from datetime import datetime, timezone

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

class SearchIndexConsumer:
    """搜索索引更新消费者"""

    ES_URL = 'http://localhost:9200'
    INDEX_NAME = 'posts'

    def __init__(self):
        self.consumer = KafkaConsumer(
            'blog-events',
            bootstrap_servers=['localhost:9092'],
            group_id='search-indexer',
            auto_offset_reset='earliest',
            enable_auto_commit=False,
            max_poll_records=50,
            partition_assignment_strategy=[
                'org.apache.kafka.clients.consumer.CooperativeStickyAssignor'
            ],
            value_deserializer=lambda m: json.loads(m.decode('utf-8'))
        )

        # 已处理的事件 ID 集合（幂等去重）
        self.processed_events = set()

    def start(self):
        """启动消费循环"""
        logger.info(f"搜索索引消费者启动 (group: search-indexer)")

        try:
            for message in self.consumer:
                event = message.value
                self._process_event(event)

        except KeyboardInterrupt:
            logger.info("收到中断信号")
        finally:
            self.consumer.close()

    def _process_event(self, event):
        """处理单条事件"""
        event_id = event.get('eventId')
        event_type = event.get('eventType')
        data = event.get('data', {})

        # 幂等检查：已处理过的事件直接跳过
        if event_id in self.processed_events:
            logger.debug(f"跳过已处理事件: {event_id}")
            self.consumer.commit()
            return

        try:
            if event_type == 'post_published':
                self._index_post(data)
            elif event_type == 'comment_added':
                self._update_comment_count(data['postId'])
            elif event_type == 'post_viewed':
                self._update_view_count(data['postId'])
            else:
                logger.warning(f"未知事件类型: {event_type}")

            # 处理成功
            self.processed_events.add(event_id)
            self.consumer.commit()

            # 控制内存：保留最近 10000 个事件 ID
            if len(self.processed_events) > 10000:
                # 简单处理：清空（实际项目用 LRU 或 Redis）
                self.processed_events.clear()

        except Exception as e:
            logger.error(f"处理事件失败: {event_id}, error={e}")
            # 不提交 offset，下次还会消费这条消息

    def _index_post(self, data):
        """索引新文章"""
        doc = {
            'postId': data['postId'],
            'title': data['title'],
            'content': data['content'],
            'author': data['author'],
            'tags': data.get('tags', []),
            'category': data.get('category', ''),
            'commentCount': 0,
            'viewCount': 0,
            'indexedAt': datetime.now(timezone.utc).isoformat()
        }

        response = requests.put(
            f'{self.ES_URL}/{self.INDEX_NAME}/_doc/{data["postId"]}',
            json=doc
        )
        response.raise_for_status()
        logger.info(f"索引文章: postId={data['postId']}, title={data['title']}")

    def _update_comment_count(self, post_id):
        """更新评论数"""
        response = requests.post(
            f'{self.ES_URL}/{self.INDEX_NAME}/_update/{post_id}',
            json={
                'script': {
                    'source': 'ctx._source.commentCount += 1',
                    'lang': 'painless'
                }
            }
        )
        response.raise_for_status()
        logger.info(f"更新评论数: postId={post_id}")

    def _update_view_count(self, post_id):
        """更新浏览量"""
        response = requests.post(
            f'{self.ES_URL}/{self.INDEX_NAME}/_update/{post_id}',
            json={
                'script': {
                    'source': 'ctx._source.viewCount += 1',
                    'lang': 'painless'
                }
            }
        )
        response.raise_for_status()


if __name__ == '__main__':
    consumer = SearchIndexConsumer()
    consumer.start()
```

```python
"""
实时统计消费者
从 Kafka 消费行为日志，更新 Redis 中的统计数据

import json
import redis
import logging
from kafka import KafkaConsumer

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

class AnalyticsConsumer:
    """实时统计消费者"""

    def __init__(self):
        self.consumer = KafkaConsumer(
            'demo-logs',
            bootstrap_servers=['localhost:9092'],
            group_id='analytics',
            auto_offset_reset='earliest',
            enable_auto_commit=False,
            value_deserializer=lambda m: json.loads(m.decode('utf-8'))
        )

        self.redis = redis.Redis(host='localhost', port=6379, db=0)

    def start(self):
        """启动消费循环"""
        logger.info("实时统计消费者启动")

        try:
            for message in self.consumer:
                event = message.value
                self._process_event(event)
                self.consumer.commit()

        except KeyboardInterrupt:
            logger.info("收到中断信号")
        finally:
            self.consumer.close()

    def _process_event(self, event):
        """处理事件，更新统计数据"""
        event_type = event.get('eventType')
        data = event.get('data', {})

        if event_type == 'post_viewed':
            post_id = data.get('postId')
            # 文章浏览量 +1
            self.redis.incr(f'post:{post_id}:views')
            # 今日浏览量 +1（按天统计）
            today = data.get('timestamp', '')[:10]
            self.redis.incr(f'post:{post_id}:views:{today}')
            # 全站今日浏览量
            self.redis.incr(f'site:views:{today}')

        elif event_type == 'user_search':
            keyword = data.get('keyword', '')
            if keyword:
                # 搜索热词计数
                self.redis.zincrby('search:hot_keywords', 1, keyword)

        elif event_type == 'comment_added':
            post_id = data.get('postId')
            # 评论数 +1
            self.redis.incr(f'post:{post_id}:comments')

        logger.info(f"统计更新: type={event_type}")


if __name__ == '__main__':
    consumer = AnalyticsConsumer()
    consumer.start()
```

## 五、错误处理与重试机制

### 5.1 死信队列（Dead Letter Queue）

对于多次处理失败的消息，不应该让它无限重试（会阻塞其他消息），应该转入死信队列：

```
┌─────────────────────────────────────────────────────────────────────┐
│                    死信队列机制                                       │
│                                                                     │
│  blog-events ──→ 消费者处理                                          │
│                     │                                                │
│                ┌────┴────┐                                           │
│                │         │                                           │
│             成功       失败                                          │
│                │         │                                           │
│                │    重试 3 次？                                        │
│                │    ┌────┴────┐                                     │
│                │    │         │                                     │
│                │   是        否                                      │
│                │    │         │                                     │
│                │  重试      放弃                                     │
│                │    │         │                                     │
│                │    │         ▼                                     │
│                │    │   ┌──────────────────┐                        │
│                │    │   │ blog-events-dlq  │                        │
│                │    │   │ (死信队列)        │                        │
│                │    │   │ 人工处理 / 告警   │                        │
│                │    │   └──────────────────┘                        │
│                │                                                    │
│                ▼                                                    │
│           提交 offset                                               │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

```python
"""
带死信队列的消费者
"""

import json
import logging
import time
from kafka import KafkaProducer, KafkaConsumer

logger = logging.getLogger(__name__)

MAX_RETRIES = 3

class DLQConsumer:
    """带死信队列的消费者"""

    def __init__(self):
        self.consumer = KafkaConsumer(
            'blog-events',
            bootstrap_servers=['localhost:9092'],
            group_id='demo-processor-with-dlq',
            enable_auto_commit=False,
            value_deserializer=lambda m: json.loads(m.decode('utf-8'))
        )

        self.producer = KafkaProducer(
            bootstrap_servers=['localhost:9092'],
            value_serializer=lambda v: json.dumps(v, ensure_ascii=False).encode('utf-8')
        )

        # 记录每条消息的重试次数（event_id → retry_count）
        self.retry_counts = {}

    def start(self):
        for message in self.consumer:
            event = message.value
            event_id = event.get('eventId', str(message.offset))

            try:
                self._process_event(event)
                # 处理成功，清除重试计数，提交 offset
                self.retry_counts.pop(event_id, None)
                self.consumer.commit()

            except Exception as e:
                retry_count = self.retry_counts.get(event_id, 0) + 1
                self.retry_counts[event_id] = retry_count

                if retry_count >= MAX_RETRIES:
                    # 超过重试次数，发送到死信队列
                    event['error'] = str(e)
                    event['retryCount'] = retry_count
                    event['failedAt'] = time.strftime('%Y-%m-%dT%H:%M:%S')

                    self.producer.send('blog-events-dlq', value=event)
                    self.producer.flush()

                    logger.error(
                        f"消息转入死信队列: eventId={event_id}, "
                        f"retries={retry_count}, error={e}"
                    )

                    # 清除重试计数，提交 offset（不再重试）
                    self.retry_counts.pop(event_id, None)
                    self.consumer.commit()
                else:
                    logger.warning(
                        f"处理失败，将重试: eventId={event_id}, "
                        f"retry={retry_count}/{MAX_RETRIES}"
                    )
                    # 不提交 offset，下次 poll 还会拉到这条消息

    def _process_event(self, event):
        """实际处理逻辑"""
        # 模拟处理
        event_type = event.get('eventType')
        logger.info(f"处理事件: {event_type}")

    def close(self):
        self.consumer.close()
        self.producer.close()
```

### 5.2 重试策略建议

| 场景 | 重试策略 | 说明 |
|------|---------|------|
| 网络超时 | 立即重试 3 次 | 瞬时故障，重试通常能解决 |
| ES 写入失败 | 退避重试（1s, 2s, 4s） | ES 可能临时不可用 |
| 数据格式错误 | 不重试，直接 DLQ | 数据有问题，重试也没用 |
| 数据库死锁 | 退避重试（随机延迟） | 避免多个消费者同时重试 |

```python
        batch_size=65536,
        linger_ms=20,
        value_serializer=lambda v: json.dumps(v).encode('utf-8')
    )

    total_messages = 100000
    batch_size = 1000

    print(f"开始发送 {total_messages} 条消息...")
    start_time = time.time()

    for i in range(total_messages):
        producer.send('demo-logs', value={
            'eventType': 'post_viewed',
            'data': {'postId': i % 100, 'userId': i % 1000}
        })

        # 每 batch_size 条 flush 一次
        if (i + 1) % batch_size == 0:
            producer.flush()
            elapsed = time.time() - start_time
            throughput = (i + 1) / elapsed
            print(f"  已发送 {i+1:,} 条, 吞吐量: {throughput:,.0f} 条/秒")

    producer.flush()
    elapsed = time.time() - start_time
    throughput = total_messages / elapsed

    print(f"\n测试完成！")
    print(f"  总消息数: {total_messages:,}")
    print(f"  总耗时: {elapsed:.2f} 秒")
    print(f"  平均吞吐量: {throughput:,.0f} 条/秒")
    print(f"  平均延迟: {elapsed/total_messages*1000:.2f} ms/条")

    producer.close()


if __name__ == '__main__':
    benchmark()
```

## 七、面试题

### Q1：如何设计一个高可用的 Kafka 事件管道？

1. **生产者端**：异步发送 + 回调 + 死信队列 + 消息重试
2. **Broker 端**：多副本 + ISR + `min.insync.replicas>=2`
3. **消费者端**：手动提交 offset + 幂等处理 + 优雅关闭
4. **监控**：Consumer Lag 告警 + Broker 集群监控
5. **容灾**：多机房部署 + 消息备份

### Q2：消息积压了怎么办？

1. **紧急处理**：临时增加消费者数量（需要先增加分区数）
2. **排查原因**：消费者处理慢（优化代码）、外部依赖慢（ES/DB）、消费者挂了
3. **长期优化**：优化消费逻辑、批量处理、异步写入、增加消费者
4. **极端方案**：写一个临时消费者，跳过耗时逻辑，快速消费积压消息，后续再补偿处理

### Q3：如何保证 Kafka 消息的"恰好一次"语义？

严格意义上的 Exactly-Once 需要满足：

1. **生产者**：开启幂等性（`enable.idempotence=true`）或使用事务
2. **消费者**：手动提交 offset + 幂等处理（用 eventId 去重）
3. **端到端**：Kafka 事务 + 消费者 `isolation.level=read_committed`

实际项目中，更常见的做法是"至少一次 + 幂等处理"，因为实现简单且足够可靠。
