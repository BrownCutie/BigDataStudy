# Spark 3.5.8 环境搭建详解

## 1. PySpark 快速安装

### 1.1 前置条件

| 组件 | 最低版本 | 说明 |
|------|---------|------|
| Python | 3.8+ | 推荐 3.10 或 3.11 |
| Java JDK | 8 或 11 | 推荐 JDK 11（不要用 JDK 17+，有兼容问题） |

```bash
# 检查 Java 版本
java -version
# 应输出：openjdk version "11.x.x" 或 java version "1.8.0_xxx"

# 检查 Python 版本
python3 --version
# 应输出：Python 3.10.x 或更高
```

### 1.2 使用 pip 安装 PySpark

```bash
# 创建虚拟环境（推荐）
python3 -m venv spark-env
source spark-env/bin/activate

# 安装 PySpark（会自动安装匹配的 Spark 版本）
pip install pyspark==3.5.8

# 验证安装
python3 -c "import pyspark; print(pyspark.__version__)"
# 输出：3.5.8
```

### 1.3 pip 安装的目录结构

```
pip 安装后的 Spark 路由：
$ python3 -c "import pyspark; print(pyspark.SPARK_HOME)"
# 输出类似：/path/to/venv/lib/python3.10/site-packages/pyspark

对应目录结构：
venv/lib/python3.10/site-packages/pyspark/
├── __init__.py
├── sql/                    # Spark SQL 模块
├── ml/                     # MLlib 模块
├── streaming/              # Streaming 模块
├── RDD.py

├── SparkSession.py
└── jars/                   # Spark 运行所需 JAR 包
    ├── spark-core_*.jar
    ├── spark-sql_*.jar
    └── ...
```

## 2. Spark 独立安装

如果你需要完整的 Spark 安装（包含 spark-submit、spark-sql 等命令行工具），需要手动下载。

### 2.1 下载解压

```bash
# 下载 Spark 3.5.8（Pre-built for Apache Hadoop 3.3）
# 注意：选择 "Pre-built for Apache Hadoop" 版本，不要选择 "Source" 版本
wget https://archive.apache.org/dist/spark/spark-3.5.8/spark-3.5.8-bin-hadoop3.tgz

# 解压
tar -xzf spark-3.5.8-bin-hadoop3.tgz

# 移动到安装目录
mv spark-3.5.8-bin-hadoop3 ~/spark-3.5.8

# 创建软链接方便版本切换
ln -s ~/spark-3.5.8 ~/spark
```

## 3. spark-defaults.conf 配置详解

### 3.1 配置文件说明

Spark 的配置有三个层级，优先级从高到低：

```
配置优先级（从高到低）：

┌──────────────────────────────────────────────────┐
│ 1. 代码中硬编码 (spark.conf.set())                │  最高优先级
├──────────────────────────────────────────────────┤
│ 2. spark-submit 命令行参数 (--conf key=value)    │
├──────────────────────────────────────────────────┤
│ 3. spark-defaults.conf 文件                      │
├──────────────────────────────────────────────────┤
│ 4. spark-env.sh 环境变量                         │  最低优先级
└──────────────────────────────────────────────────┘
```

### 3.2 常用配置项

编辑 `$SPARK_HOME/conf/spark-defaults.conf`（复制模板文件后编辑）：

```bash
# 复制模板
cp $SPARK_HOME/conf/spark-defaults.conf.template $SPARK_HOME/conf/spark-defaults.conf
```

```properties
# ============================================================
# Spark 基础配置
# ============================================================

# 应用名称（可在代码中覆盖）
spark.app.name                    MySparkApp

# 运行模式
spark.master                      local[*]

# Driver 内存
spark.driver.memory               2g

# Executor 内存
spark.executor.memory             2g

# Executor CPU 核数
spark.executor.cores              2

# ============================================================
# 序列化配置
# ============================================================

# 使用 Kryo 序列化（比 Java 默认序列化快 10x）
spark.serializer                  org.apache.spark.serializer.KryoSerializer

# 注册需要 Kryo 序列化的类
spark.kryo.registrationRequired   false

# ============================================================
# Shuffle 配置
# ============================================================

# Shuffle 服务端口
spark.shuffle.service.port        7337

# Shuffle 文件排序（减少磁盘 I/O）
spark.shuffle.sort.bypassMergeThreshold 200

# Shuffle 压缩
spark.shuffle.compress            true

# Shuffle 文件清理延迟（秒）
spark.shuffle.file.cleanup.enabled true

# ============================================================
# 内存与执行配置
# ============================================================

# 动态资源分配
spark.dynamicAllocation.enabled           false
spark.dynamicAllocation.minExecutors      1
spark.dynamicAllocation.maxExecutors      10
spark.dynamicAllocation.initialExecutors  2

# 自适应查询执行（AQE）
spark.sql.adaptive.enabled                true
spark.sql.adaptive.coalescePartitions.enabled   true
spark.sql.adaptive.skewJoin.enabled             true

# ============================================================
# SQL 配置
# ============================================================

# Spark SQL 分区大小（字节）
spark.sql.shuffle.partitions           200

# 广播 Join 阈值（字节），10MB
spark.sql.autoBroadcastJoinThreshold   10485760

# 启用缓存
spark.sql.inMemoryColumnarStorage.compressed  true
spark.sql.inMemoryColumnarStorage.batchSize   10000

# ============================================================
# Hive 集成配置
# ============================================================

# 启用 Hive 支持
spark.sql.catalogImplementation       hive

# Hive Metastore 连接 URI
# spark.hadoop.hive.metastore.uris     thrift://localhost:9083

# ============================================================
# 日志配置
# ============================================================

# 事件日志目录（History Server 使用）
spark.eventLog.enabled              true
spark.eventLog.dir                  /tmp/spark-events

# 历史日志保留天数
spark.history.fs.logDirectory       /tmp/spark-events

# ============================================================
# 网络配置
# ============================================================

# Driver 端口
spark.driver.port                   4041

# BlockManager 端口
spark.blockManager.port             4042

# 网络超时（秒）
spark.network.timeout               120s

# RPC 最大重试次数
spark.rpc.numRetries                3

# RPC 重试等待时间（秒）
spark.rpc.retry.wait                3s
```

## 4. 与 Hive 集成配置

### 4.1 集成架构

```
Spark 与 Hive 集成架构：

┌───────────────────┐     ┌──────────────────────┐
│                   │     │                      │
│   Spark SQL       │     │   Hive Metastore     │
│                   │     │   (元数据存储)        │
│  ┌─────────────┐  │     │                      │
│  │ 读取 Hive   │  │     │  ┌────────────────┐ │
│  │ 表的元数据   │──┼────>│  │  MySQL / Derby │ │
│  │             │  │     │  │  (存储库)      │ │
│  │ 直接读取    │  │     │  └────────────────┘ │
│  │ HDFS 数据   │  │     │                      │
│  └─────────────┘  │     └──────────────────────┘
│                   │               ^
│                   │               |
└───────────────────┘     ┌────────┴─────────┐
                          │  hive-site.xml    │
                          │  (配置连接信息)    │
                          └──────────────────┘
                                    │
                          ┌─────────┴─────────┐
                          │  Metastore JAR    │
                          │  (MySQL 驱动等)   │
                          └───────────────────┘
```

### 4.2 配置步骤

**步骤 1：复制 hive-site.xml**

```bash
# 将 Hive 的配置文件复制到 Spark 的 conf 目录
cp $HIVE_HOME/conf/hive-site.xml $SPARK_HOME/conf/
```

**步骤 2：确认 Metastore JAR 包**

Spark 3.x 已内置了 Hive Metastore 客户端 JAR 包（`hive-metastore-*.jar`），但可能缺少数据库驱动。如果使用 MySQL 作为 Metastore 存储：

```bash
# 下载 MySQL JDBC 驱动
wget https://repo1.maven.org/maven2/mysql/mysql-connector-java/8.0.33/mysql-connector-java-8.0.33.jar

# 放到 Spark 的 jars 目录
cp mysql-connector-java-8.0.33.jar $SPARK_HOME/jars/
```

**步骤 3：修改 spark-defaults.conf**

```properties
# 启用 Hive 支持（关键配置）
spark.sql.catalogImplementation=hive

# 如果 Hive Metastore 服务已启动，使用以下方式连接
spark.hadoop.hive.metastore.uris=thrift://localhost:9083
```

**步骤 4：验证集成**

```bash
# 启动 Spark SQL 命令行
spark-sql

# 在 Spark SQL 中执行
spark-sql> SHOW DATABASES;
spark-sql> USE my_database;
spark-sql> SHOW TABLES;
spark-sql> SELECT * FROM my_table LIMIT 10;
```

### 4.3 Hive Metastore 服务模式 vs 嵌入模式

```
两种连接 Metastore 的方式：

方式 1：直接连接数据库（嵌入式）
┌──────────┐     ┌──────────────┐
│ Spark SQL│────>│   MySQL      │
│          │     │  (Metastore  │
│          │     │   数据库)     │
└──────────┘     └──────────────┘

方式 2：通过 Metastore 服务（推荐生产环境使用）
┌──────────┐     ┌──────────────┐     ┌──────────┐
│ Spark SQL│────>│ Hive         │────>│   MySQL   │
│          │     │ Metastore    │     │          │
│          │     │ Service      │     │          │
└──────────┘     │ :9083        │     └──────────┘
                 └──────────────┘

推荐方式 2 的原因：
- 多个 Spark/Hive/Presto 客户端可以共享同一个 Metastore
- Metastore 服务提供连接池，避免直连数据库的连接数问题
- 权限控制更方便
```

### 5.2 常用命令

```sql
-- 查看数据库
SHOW DATABASES;

-- 切换数据库
USE my_database;

-- 查看表
SHOW TABLES;

-- 查看表结构
DESCRIBE my_table;
DESCRIBE EXTENDED my_table;

-- 查看表创建语句
SHOW CREATE TABLE my_table;

-- 查看分区
SHOW PARTITIONS my_table;

-- 创建数据库
CREATE DATABASE IF NOT EXISTS blog_db
  COMMENT '业务数据库'
  WITH DBPROPERTIES ('creator' = 'admin');

-- 创建表
CREATE TABLE IF NOT EXISTS blog_db.articles (
    id          BIGINT          COMMENT '文章ID',
    title       STRING          COMMENT '标题',
    author      STRING          COMMENT '作者',
    views       BIGINT          COMMENT '浏览量',
    likes       INT             COMMENT '点赞数',
    publish_date DATE           COMMENT '发布日期'
)
USING PARQUET
PARTITIONED BY (year INT, month INT)
STORED AS PARQUET
LOCATION 'hdfs:///user/hive/warehouse/blog_db.db/articles';

-- 插入数据
INSERT INTO blog_db.articles
PARTITION (year=2024, month=1)
VALUES (1, 'Spark入门', '张三', 1500, 50, DATE '2024-01-15');

-- 查询
SELECT author, COUNT(*) as cnt
FROM blog_db.articles
GROUP BY author
ORDER BY cnt DESC;
```

### 7.2 各页面功能

**Jobs 页面**：
- 显示所有 Action 操作触发的作业
- 每个作业包含一个或多个 Stage
- 可以看到每个 Job 的执行时间和状态

**Stages 页面**：
- 显示 DAG 执行图（核心页面）
- 每个 Stage 的详细统计：任务数、读取/写入数据量、Shuffle 数据量
- 可以看到数据倾斜情况（某些 Task 特别慢）

**Tasks 页面**：
- 每个 Stage 中所有 Task 的详细信息
- 每个任务的执行时间、数据读取量、GC 时间
- 可以发现数据倾斜

**SQL 页面**：
- DataFrame/SQL 的执行计划
- Catalyst 优化前后的对比
- 物理执行计划详情

**Executors 页面**：
- 每个 Executor 的资源使用情况（内存、CPU）
- 缓存数据大小
- 完成和失败的任务数

### 7.3 实际使用示例

```python
from pyspark.sql import SparkSession

spark = SparkSession.builder \
    .appName("WebUIDemo") \
    .master("local[*]") \
    .getOrCreate()

# 创建业务数据
import random
random.seed(42)
demo_data = [(f"博客文章{i}", f"作者{random.randint(1,5)}", random.randint(100, 5000))
             for i in range(10000)]

df = spark.createDataFrame(demo_data, ["title", "author", "views"])

# 执行一些操作后，打开浏览器访问 http://localhost:4040
result = df.groupBy("author") \
    .agg({"views": "sum", "title": "count"}) \
    .orderBy("sum(views)", ascending=False)

result.show()

# 注意：Web UI 只在 SparkSession 存活期间可用
# 如果使用 spark-submit 提交，作业结束后 UI 就消失了
# 可以使用 History Server 查看历史记录

spark.stop()
```

## 8. 常见启动错误排查

### 8.1 Java 版本问题

```
错误信息：
  java.lang.UnsupportedClassVersionError: org/apache/spark/launcher/Main
  has been compiled by a more recent version of the Java Runtime

原因：JDK 版本过高或过低
解决：
  # 安装 JDK 11
  brew install openjdk@11

  # 设置 JAVA_HOME
  export JAVA_HOME=$(/usr/libexec/java_home -v 11)
```

### 8.2 端口被占用

```
错误信息：
  java.net.BindException: Address already in use

原因：4040 端口被其他 Spark 进程占用
解决：
  # 方法 1：结束占用进程
  lsof -i :4040
  kill -9 <PID>

  # 方法 2：指定其他端口
  spark-submit --conf spark.driver.port=4041 \
               --conf spark.ui.port=4041 my_app.py
```

### 8.3 Python 版本不匹配

```
错误信息：
  PYSPARK_PYTHON is not set

原因：PySpark 找不到 Python 解释器
解决：
  # 在 spark-env.sh 或 spark-defaults.conf 中设置
  export PYSPARK_PYTHON=/path/to/venv/bin/python3
  export PYSPARK_DRIVER_PYTHON=/path/to/venv/bin/python3

  # 或者在代码中设置
  import os
  os.environ['PYSPARK_PYTHON'] = '/path/to/venv/bin/python3'
```

### 8.4 内存不足

```
错误信息：
  java.lang.OutOfMemoryError: Java heap space
  Container killed by YARN for exceeding memory limits

原因：Driver 或 Executor 内存不足
解决：
  # 增加 Driver 内存
  spark-submit --driver-memory 4g my_app.py

  # 增加 Executor 内存
  spark-submit --executor-memory 4g my_app.py

  # 如果是 Local 模式
  spark-submit --master "local[*]" \
               --driver-memory 4g \
               --conf spark.driver.extraJavaOptions="-Xmx4g" \
               my_app.py
```

### 8.5 PySpark 安装但找不到 Spark

```
错误信息：
  ModuleNotFoundError: No module named 'pyspark'
  或
  SPARK_HOME is not set

解决：
  # 如果使用 pip 安装
  pip install pyspark==3.5.8
  python3 -c "import pyspark; print(pyspark.SPARK_HOME)"

  # 如果使用独立安装
  export SPARK_HOME=/path/to/spark
  export PATH=$SPARK_HOME/bin:$PATH

  # 如果两种方式都安装了，确保 PYTHONPATH 正确
  export PYTHONPATH=$SPARK_HOME/python:$SPARK_HOME/python/lib/py4j-*.zip:$PYTHONPATH
```

### 8.6 Hive 集成错误

```
错误信息：
  java.lang.ClassNotFoundException: org.apache.hadoop.hive.metastore.HiveMetaStoreClient

原因：缺少 Hive Metastore JAR 包
解决：
  # 确保使用预编译版 Spark（已包含 Hive 支持）
  # 下载时选择 spark-x.x.x-bin-hadoop3（不是用户自编译版）

  # 如果需要 MySQL 驱动
  cp mysql-connector-java-8.0.33.jar $SPARK_HOME/jars/

  # 检查 hive-site.xml 是否在 conf 目录下
  ls $SPARK_HOME/conf/hive-site.xml
```
