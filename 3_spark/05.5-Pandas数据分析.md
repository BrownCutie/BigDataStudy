## 一、Pandas 基础

### 1.1 Series 和 DataFrame 是什么

Pandas 有两个核心数据结构：

| 数据结构 | 说明 | 类比 |
|---------|------|------|
| **Series** | 一维数组，带标签索引 | Excel 的一列 |
| **DataFrame** | 二维表格，每列可以是不同类型 | Excel 的一张表 |

```python
import pandas as pd

# Series — 一维，类似带标签的列表
s = pd.Series([10, 20, 30], index=['a', 'b', 'c'])
print(s)
# a    10
# b    20
# c    30

# DataFrame — 二维表格
df = pd.DataFrame({
    'name': ['张三', '李四'],
    'age': [25, 30]
})
print(df)
#   name  age
# 0   张三   25
# 1   李四   30
```

> **Spark 对照**：Pandas 的 DataFrame 对应 Spark 的 `pyspark.sql.DataFrame`，Series 对应 Spark 的一列（`Column` 对象）。

### 1.2 创建 DataFrame 的多种方式

```python
# 方式1：字典创建（最常用）
df = pd.DataFrame({
    'title': ['Spark入门', 'Hadoop原理'],
    'views': [1500, 2300]
})

# 方式2：列表 of 字典
data = [
    {'title': 'Spark入门', 'views': 1500},
    {'title': 'Hadoop原理', 'views': 2300}
]
df = pd.DataFrame(data)

# 方式3：NumPy 数组创建
import numpy as np
df = pd.DataFrame(
    np.array([[1, 2], [3, 4]]),
    columns=['A', 'B']
)

# 方式4：读取 CSV（最常见的数据来源）
df = pd.read_csv('data.csv')

# 方式5：读取 JSON
df = pd.read_json('data.json')
```

### 1.3 查看 DataFrame 信息 — 这组函数每天都会用

| 函数 | 作用 | 输出内容 |
|------|------|---------|
| `df.head(n)` | 查看前 n 行（默认5行） | 数据预览 |
| `df.tail(n)` | 查看后 n 行 | 数据预览 |
| `df.info()` | 查看整体信息 | 列名、类型、非空数量、内存占用 |
| `df.describe()` | 统计摘要 | 计数、均值、标准差、最值、四分位数 |
| `df.shape` | 行列数 | `(行数, 列数)` 元组 |
| `df.columns` | 所有列名 | Index 对象 |
| `df.dtypes` | 每列的数据类型 | 列名 -> 类型的映射 |
| `df.index` | 行索引 | Index 对象 |
| `df.memory_usage()` | 内存占用 | 每列的内存字节数 |

```python
df = pd.DataFrame({
    'title': ['Spark入门', 'Hadoop原理', 'Flink流处理', 'Kafka消息队列'],
    'views': [1500, 2300, 3200, 1800],
    'likes': [50, 80, 150, 60]
})

print(df.head(2))          # 前2行
print(df.shape)            # (4, 3)
print(df.describe())       # views 和 likes 的统计摘要
print(df.dtypes)           # title: object, views: int64, likes: int64
print(df.info())           # 完整信息（含内存占用）
```

> **Spark 对照**：`df.show()` 对应 `df.head()`，`df.printSchema()` 对应 `df.info()`，`df.columns` 和 `df.dtypes` 在两边完全一样。

## 二、数据读取与写入

### 2.1 读取函数全家桶

| 函数 | 读取格式 | 常用参数 |
|------|---------|---------|
| `pd.read_csv()` | CSV 文件 | `sep`, `header`, `encoding`, `usecols` |
| `pd.read_excel()` | Excel 文件 | `sheet_name`, `header` |
| `pd.read_json()` | JSON 文件 | `orient`, `lines` |
| `pd.read_sql()` | SQL 数据库 | `sql`, `con`（需要 sqlalchemy） |
| `pd.read_parquet()` | Parquet 文件 | `columns`, `filters` |
| `pd.read_clipboard()` | 剪贴板 | 无 |

### 2.2 read_csv() 常用参数详解

```python
df = pd.read_csv(
    'data.csv',
    encoding='utf-8',        # 文件编码（中文文件常用 gbk）
    sep=',',                 # 分隔符（TSV 文件用 \t）
    header=0,                # 第几行作为列名（None 表示没有列名）
    index_col=0,             # 第几列作为行索引
    usecols=['name', 'age'], # 只读指定列（节省内存）
    dtype={'age': 'int32'},  # 指定列的数据类型
    na_values=['NA', 'null', '-'],  # 哪些值视为 NaN
    parse_dates=['date'],    # 自动解析日期列
    nrows=1000,              # 只读前1000行（大文件预览用）
    skiprows=2,              # 跳过前2行
)
```

### 2.3 写入函数

```python
# 写入 CSV
df.to_csv('output.csv', index=False, encoding='utf-8')

# 写入 Excel（需要 openpyxl）
df.to_excel('output.xlsx', sheet_name='Sheet1', index=False)

# 写入 JSON
df.to_json('output.json', orient='records', force_ascii=False)

# 写入 Parquet（Spark 友好格式）
df.to_parquet('output.parquet', index=False)

# 写入 SQL
df.to_sql('table_name', con=engine, if_exists='replace', index=False)
```

> **Spark 对照**：Spark 的 `spark.read.csv()` 和 `spark.read.parquet()` 参数与 Pandas 非常相似。Parquet 是 Spark 的默认格式，Pandas 也能直接读写，数据互通非常方便。

## 三、数据选择与过滤

这是 Pandas 最常用的操作，也是 Spark DataFrame 的核心操作。

### 3.1 选择列

```python
# 选择单列 — 返回 Series
s = df['title']

# 选择多列 — 返回 DataFrame
subset = df[['title', 'views']]

# 用点号选择（列名不能有空格和特殊字符）
views = df.views
```

### 3.2 选择行 — loc vs iloc

| 方法 | 索引方式 | 说明 |
|------|---------|------|
| `df.loc[]` | **标签索引** | 用行名/列名选取 |
| `df.iloc[]` | **位置索引** | 用行号/列号（从0开始）选取 |

```python
# loc — 按标签
df.loc[0]                # 第0行（标签为0的行）
df.loc[0:2]              # 第0到第2行（包含两端）
df.loc[0:2, 'title']     # 第0到2行的 title 列
df.loc[df['views'] > 2000, ['title', 'views']]  # 条件 + 指定列

# iloc — 按位置
df.iloc[0]               # 第0行
df.iloc[0:3]             # 第0、1、2行（不包含3）
df.iloc[0:3, 0:2]        # 前3行的前2列
df.iloc[-1]              # 最后一行
```

> **重要区别**：`loc[0:2]` 包含第2行，`iloc[0:2]` 不包含第2行。`loc` 是闭区间，`iloc` 是左闭右开区间。

### 3.3 条件过滤

```python
# 单条件
df[df['views'] > 2000]

# 多条件 — 用 & | ~ 连接，每个条件用括号包起来
df[(df['views'] > 1000) & (df['likes'] > 50)]
df[(df['category'] == 'Spark') | (df['category'] == 'Flink')]

# query() — 用字符串写条件，更简洁
df.query('views > 2000 and likes > 50')
df.query('category in ["Spark", "Flink"]')

# isin() — 在某个列表中
df[df['category'].isin(['Spark', 'Flink', 'Kafka'])]

# between() — 在某个范围内
df[df['views'].between(1000, 3000)]

# str.contains() — 字符串包含（支持正则）
df[df['title'].str.contains('Spark|Hadoop')]
```

### 3.4 随机采样

```python
df.sample(n=5)              # 随机抽5行
df.sample(frac=0.3)         # 随机抽30%的行
df.sample(n=3, replace=True)  # 允许重复抽样
df.sample(n=3, random_state=42)  # 固定随机种子（可复现）
```

> **Spark 对照**：Pandas 的 `df[df['col'] > 0]` 对应 Spark 的 `df.filter(col('col') > 0)` 或 `df.where(col('col') > 0)`。`query()` 在 Spark 里没有直接对应，需要用 `filter` + 表达式。

## 四、数据清洗

数据清洗是数据分析中最耗时的环节，Pandas 提供了丰富的工具。

### 4.1 缺失值处理

| 函数 | 作用 | 示例 |
|------|------|------|
| `df.isna()` / `df.isnull()` | 检测缺失值 | 返回布尔 DataFrame |
| `df.notna()` | 检测非缺失值 | isna 的反义 |
| `df.fillna()` | 填充缺失值 | `fillna(0)` / `fillna({'col': '未知'})` |
| `df.dropna()` | 删除缺失值 | `dropna()` / `dropna(subset=['col'])` |
| `df.interpolate()` | 插值填充 | 线性插值、时间插值 |
| `df.dropna(thresh=n)` | 保留至少有 n 个非空值的行 | |

```python
# 检测缺失值
print(df.isna().sum())           # 每列有多少缺失值
print(df.isna().any())           # 每列是否有缺失值

# 填充缺失值
df['views'] = df['views'].fillna(0)                    # 用固定值填充
df['category'] = df['category'].fillna('未分类')         # 用字符串填充
df['views'] = df['views'].fillna(df['views'].mean())    # 用均值填充
df.fillna(method='ffill')                              # 用前一个值填充（forward fill）
df.fillna(method='bfill')                              # 用后一个值填充（backward fill）

# 删除缺失值
df.dropna()                          # 删除任何含缺失值的行
df.dropna(subset=['title'])          # 只检查 title 列
df.dropna(how='all')                 # 只删除全为空的行
df.dropna(thresh=3)                  # 保留至少3个非空值的行

# 插值
df['views'].interpolate(method='linear')  # 线性插值
```

### 4.2 重复值处理

```python
# 检测重复行
df.duplicated()                        # 返回布尔 Series
df.duplicated(subset=['title'])        # 基于 title 列判断
df.duplicated(keep='first')            # 保留第一个（默认）
df.duplicated(keep='last')             # 保留最后一个
df.duplicated(keep=False)              # 全部标记为重复

# 删除重复行
df.drop_duplicates()
df.drop_duplicates(subset=['title'], keep='first')
```

### 4.3 数据类型转换

```python
# astype — 转换数据类型
df['views'] = df['views'].astype('float')     # 转 float
df['views'] = df['views'].astype('int32')     # 转 int32（省内存）
df['category'] = df['category'].astype('category')  # 转分类类型

# pd.to_numeric — 智能转数字（处理异常值）
pd.to_numeric(df['views'], errors='coerce')   # 无法转换的变成 NaN
pd.to_numeric(df['views'], errors='ignore')   # 无法转换的保持原样
pd.to_numeric(df['views'], downcast='integer')  # 自动选最小整数类型

# pd.to_datetime — 转日期时间
df['date'] = pd.to_datetime(df['date'])                    # 自动解析
df['date'] = pd.to_datetime(df['date'], format='%Y-%m-%d')  # 指定格式
```

### 4.4 字符串处理 — str 访问器

所有字符串方法都要通过 `.str.` 访问器调用：

| 方法 | 作用 | 示例 |
|------|------|------|
| `str.lower()` | 转小写 | `.str.lower()` |
| `str.upper()` | 转大写 | `.str.upper()` |
| `str.strip()` | 去首尾空格 | `.str.strip()` |
| `str.lstrip()` / `str.rstrip()` | 去左/右空格 | |
| `str.split()` | 分割 | `.str.split(',')` |
| `str.replace()` | 替换 | `.str.replace('a', 'b')` |
| `str.contains()` | 包含 | `.str.contains('Spark')` |
| `str.startswith()` | 以...开头 | `.str.startswith('S')` |
| `str.endswith()` | 以...结尾 | `.str.endswith('教程')` |
| `str.len()` | 长度 | `.str.len()` |
| `str.cat()` | 拼接 | `.str.cat(sep=', ')` |
| `str.extract()` | 正则提取 | `.str.extract(r'(\d+)')` |
| `str.get()` | 取第 n 个字符 | `.str.get(0)` |
| `str[:3]` | 切片 | 取前3个字符 |

```python
# 字符串处理示例
df['title_lower'] = df['title'].str.lower()
df['title_stripped'] = df['title'].str.strip()
df['tags_list'] = df['tags'].str.split(',')           # "a,b,c" -> ["a", "b", "c"]
df['title_new'] = df['title'].str.replace('入门', '基础')
df['title_len'] = df['title'].str.len()
df['has_spark'] = df['title'].str.contains('Spark', na=False)
```

### 4.5 重命名

```python
# rename — 重命名列名（推荐）
df.rename(columns={'old_name': 'new_name'})
df.rename(columns=str.upper)              # 所有列名转大写
df.rename(columns=lambda x: x.strip())    # 去列名空格

# rename — 重命名索引
df.rename(index={0: '第一行', 1: '第二行'})

# 直接赋值（全量替换列名）
df.columns = ['新列1', '新列2', '新列3']

# 添加前缀/后缀
df.add_prefix('col_')      # col_title, col_views, ...
df.add_suffix('_val')      # title_val, views_val, ...
```

> **Spark 对照**：`df.withColumnRenamed('old', 'new')` 对应 `df.rename()`，`df.dropna()` 在两边完全一样。

## 五、数据排序与排名

### 5.1 排序

```python
# sort_values — 按值排序
df.sort_values('views')                          # 按 views 升序
df.sort_values('views', ascending=False)         # 按 views 降序
df.sort_values(['category', 'views'],            # 先按分类，再按浏览量
               ascending=[True, False])

# sort_index — 按索引排序
df.sort_index()                   # 升序
df.sort_index(ascending=False)    # 降序
```

### 5.2 排名

```python
# rank — 排名
df['views_rank'] = df['views'].rank()            # 默认平均排名
df['views_rank'] = df['views'].rank(method='min')    # 最小排名（并列取最小值）
df['views_rank'] = df['views'].rank(method='max')    # 最大排名
df['views_rank'] = df['views'].rank(ascending=False) # 降序排名
```

### 5.3 取 Top N

```python
# nlargest / nsmallest — 比排序再取 head 更高效
df.nlargest(5, 'views')        # 浏览量前5
df.nsmallest(3, 'views')       # 浏览量后3
df.nlargest(5, ['views', 'likes'])  # 先按 views，再按 likes
```

> **Spark 对照**：`sort_values()` 对应 `orderBy()`，`nlargest()` 对应 Spark 没有直接对应，需要 `orderBy(col('views').desc()).limit(5)`。

## 六、数据聚合与分组

这是数据分析的核心，也是面试的高频考点。

### 6.1 基础聚合函数

| 函数 | 作用 | 说明 |
|------|------|------|
| `df.sum()` | 求和 | |
| `df.mean()` | 均值 | |
| `df.median()` | 中位数 | |
| `df.min()` / `df.max()` | 最小/最大 | |
| `df.count()` | 计数 | 不计 NaN |
| `df.std()` / `df.var()` | 标准差/方差 | |
| `df.idxmax()` / `df.idxmin()` | 最大/最小值所在索引 | |
| `df.corr()` | 相关系数 | 列间相关系数矩阵 |
| `df.cov()` | 协方差 | |

```python
# 对单列
print(df['views'].sum())
print(df['views'].mean())
print(df['views'].max())

# 对整个 DataFrame（只计算数值列）
print(df[['views', 'likes']].mean())
print(df[['views', 'likes']].corr())  # 相关系数矩阵
```

### 6.2 groupby 分组聚合

```python
# 单列分组
df.groupby('category')['views'].sum()
df.groupby('category')['views'].agg(['sum', 'mean', 'count', 'max'])

# 多列分组
df.groupby(['category', 'author'])['views'].sum()

# 对不同列用不同聚合方式
df.groupby('category').agg({
    'views': 'sum',
    'likes': 'mean',
    'title': 'count'
})

# agg 支持命名聚合
df.groupby('category').agg(
    total_views=pd.NamedAgg(column='views', aggfunc='sum'),
    avg_likes=pd.NamedAgg(column='likes', aggfunc='mean'),
    article_count=pd.NamedAgg(column='title', aggfunc='count')
)

# 分组后用 transform — 保持原 DataFrame 的形状
df['category_avg_views'] = df.groupby('category')['views'].transform('mean')
```

> **Spark 对照**：`df.groupby('col').agg(...)` 对应 Spark 的 `df.groupBy('col').agg(...)`，几乎一模一样。

### 6.3 apply 自定义函数

```python
# 对列 apply
df['views'].apply(lambda x: x * 2)

# 对行 apply
df.apply(lambda row: row['likes'] / row['views'], axis=1)

# groupby + apply
df.groupby('category').apply(lambda g: g.nlargest(2, 'views'))
```

### 6.4 pivot_table 透视表

```python
# 透视表 — 类似 Excel 的数据透视表
df.pivot_table(
    values='views',       # 要聚合的值
    index='category',     # 行分组
    columns='author',     # 列分组
    aggfunc='sum',        # 聚合函数
    fill_value=0,         # 缺失值填充
    margins=True          # 显示汇总行/列
)

# crosstab 交叉表 — 计算频率
pd.crosstab(df['category'], df['author'])
pd.crosstab(df['category'], df['author'], margins=True)  # 加总计
```

### 6.5 value_counts 与 unique

```python
# value_counts — 计数（最常用的统计函数之一）
df['category'].value_counts()                # 计数并降序排列
df['category'].value_counts(normalize=True)  # 计算占比（频率）
df['category'].value_counts(ascending=True)  # 升序

# unique / nunique
df['category'].unique()      # 唯一值列表
df['category'].nunique()     # 唯一值个数
```

> **Spark 对照**：`value_counts()` 对应 Spark 的 `df.groupBy('col').count().orderBy(desc('count'))`。

## 七、数据合并与连接

### 7.1 pd.concat — 拼接

```python
# 纵向拼接（行方向）
pd.concat([df1, df2], axis=0)

# 横向拼接（列方向）
pd.concat([df1, df2], axis=1)

# 忽略索引
pd.concat([df1, df2], ignore_index=True)

# 外连接（保留所有数据）
pd.concat([df1, df2], axis=0, join='outer')

# 内连接（只保留共有索引）
pd.concat([df1, df2], axis=0, join='inner')
```

### 7.2 pd.merge — 连接（SQL JOIN）

| 参数 | 说明 | 对应 SQL |
|------|------|---------|
| `how='inner'` | 内连接（默认） | INNER JOIN |
| `how='outer'` | 外连接 | FULL OUTER JOIN |
| `how='left'` | 左连接 | LEFT JOIN |
| `how='right'` | 右连接 | RIGHT JOIN |
| `on='key'` | 连接键 | ON key |
| `left_on` / `right_on` | 不同列名连接 | ON a.id = b.user_id |

```python
# 内连接
pd.merge(df1, df2, on='id', how='inner')

# 左连接（保留左表所有行）
pd.merge(df1, df2, on='id', how='left')

# 外连接（保留所有行）
pd.merge(df1, df2, on='id', how='outer')

# 不同列名连接
pd.merge(df1, df2, left_on='user_id', right_on='id', how='left')

# 多键连接
pd.merge(df1, df2, on=['id', 'date'], how='inner')

# 合并后加后缀区分同名列
pd.merge(df1, df2, on='id', suffixes=('_left', '_right'))
```

### 7.3 df.join

```python
# join — 基于索引连接（比 merge 简单，但只能用索引）
df1.join(df2, how='left')

# join 也可以指定列
df1.join(df2.set_index('key'), on='key')
```

### 7.4 combine_first

```python
# combine_first — 用另一个 DataFrame 填充缺失值
df1.combine_first(df2)
# df1 有值用 df1，df1 缺失用 df2 补充
```

> **Spark 对照**：`pd.merge()` 对应 Spark 的 `df.join()`，`pd.concat()` 对应 Spark 的 `union()`。

## 八、时间序列

时间序列处理是数据分析的重要场景，Pandas 提供了强大的时间序列支持。

### 8.1 时间创建与转换

```python
# pd.to_datetime — 转换为日期时间
pd.to_datetime('2024-01-15')
pd.to_datetime(['2024-01-15', '2024-02-28'])
pd.to_datetime('20240115', format='%Y%m%d')

# pd.date_range — 生成日期范围
pd.date_range('2024-01-01', '2024-12-31', freq='D')     # 每天一个
pd.date_range('2024-01-01', periods=12, freq='M')        # 每月最后一个工作日
pd.date_range('2024-01-01', periods=10, freq='W')        # 每周
pd.date_range('2024-01-01', periods=5, freq='H')         # 每小时
```

## 九、高级操作

### 9.1 apply / applymap / map 的区别

| 方法 | 作用对象 | 说明 |
|------|---------|------|
| `df.apply(func)` | 列或行 | 对整列/整行应用函数 |
| `df.applymap(func)` | 每个元素 | 对 DataFrame 的每个元素应用函数 |
| `Series.map(func)` | 每个元素 | 对 Series 的每个元素映射 |
| `Series.apply(func)` | 每个元素 | 与 map 类似，但支持更复杂的函数 |

```python
# apply — 对列/行操作
df['views'].apply(lambda x: x ** 2)              # 对列
df.apply(lambda row: row['likes'] / row['views'], axis=1)  # 对行

# applymap — 对每个元素（注意：Pandas 2.1.0+ 改名为 map）
df[['views', 'likes']].map(lambda x: x * 2)      # 所有值翻倍

# map — 对 Series 的每个元素
df['category'].map({'Spark': 'SP', 'Flink': 'FL'})  # 值映射
df['category'].map(len)                            # 函数映射
```

### 9.2 pipe 管道操作

```python
# pipe — 将多个操作串联成管道，代码更清晰
def remove_missing(df):
    return df.dropna()

def add_engagement_rate(df):
    df['engagement'] = df['likes'] / df['views']
    return df

def filter_popular(df):
    return df[df['views'] > 1000]

# 管道式调用（从左到右，一目了然）
result = (df
    .pipe(remove_missing)
    .pipe(add_engagement_rate)
    .pipe(filter_popular)
)
```

### 9.3 窗口函数

```python
# shift — 行偏移（前一行/后一行的值）
df['prev_views'] = df['views'].shift(1)    # 前一行的浏览量
df['next_views'] = df['views'].shift(-1)   # 后一行的浏览量
df['views_change'] = df['views'] - df['views'].shift(1)  # 浏览量变化

# diff — 差分
df['views_diff'] = df['views'].diff()      # 与前一行差值（同 shift(1) 后做差）

# pct_change — 百分比变化
df['views_pct'] = df['views'].pct_change()  # 环比增长率

# cumsum / cummax / cummin / cumprod — 累计
df['views_cumsum'] = df['views'].cumsum()    # 累计和
df['views_cummax'] = df['views'].cummax()    # 累计最大值
df['views_cummin'] = df['views'].cummin()    # 累计最小值
df['value_cumprod'] = df['value'].cumprod()  # 累计乘积
```

### 9.4 where / mask / clip

```python
# where — 条件为 True 保留，False 替换
df['views'].where(df['views'] > 1000, 0)  # 小于1000的替换为0

# mask — 与 where 相反，条件为 True 替换
df['views'].mask(df['views'] > 1000, 1000)  # 大于1000的截断为1000

# clip — 截断到指定范围
df['views'].clip(lower=100, upper=5000)  # 小于100的变100，大于5000的变5000
```

### 9.5 分箱 — cut / qcut

```python
# cut — 等宽分箱（按值范围分）
df['views_level'] = pd.cut(
    df['views'],
    bins=[0, 1000, 3000, 5000, 10000],
    labels=['低', '中', '高', '热门']
)

# qcut — 等频分箱（每箱数量大致相同）
df['views_quartile'] = pd.qcut(df['views'], q=4, labels=['Q1', 'Q2', 'Q3', 'Q4'])
```

### 9.6 独热编码 — get_dummies

```python
# get_dummies — 分类变量转独热编码
pd.get_dummies(df['category'])
pd.get_dummies(df, columns=['category'])  # 直接在原 DataFrame 上转换
pd.get_dummies(df, columns=['category'], drop_first=True)  # 去掉第一列（避免共线性）
```

### 9.7 宽长表转换 — melt / pivot

```python
# melt — 宽表转长表（列转行）
pd.melt(
    df,
    id_vars=['title', 'author'],   # 保留的列
    value_vars=['views', 'likes'], # 要转换的列
    var_name='metric',             # 变量名列名
    value_name='value'             # 值列名
)

# pivot — 长表转宽表（行转列）
df.pivot(
    index='title',
    columns='metric',
    values='value'
)
```

> **Spark 对照**：`get_dummies()` 对应 Spark 的 `OneHotEncoder`，`pivot()` 在两边都有，`melt()` 在 Spark 中没有直接对应。

## 十、性能优化

### 10.1 向量化操作 vs 循环

```python
# 慢 — 循环
result = []
for i in range(len(df)):
    result.append(df.loc[i, 'views'] * 2)
df['views_double'] = result

# 快 — 向量化
df['views_double'] = df['views'] * 2

# 更快 — numpy
df['views_double'] = np.where(df['views'] > 1000, df['views'] * 2, df['views'])
```

**原则：永远优先使用向量化操作，避免在 DataFrame 上用循环。**

### 10.2 eval 表达式求值

```python
# eval — 对复杂表达式求值，节省中间内存
df.eval('engagement = likes / views', inplace=True)

# 多个表达式
df.eval('''
    ratio = likes / views
    score = ratio * 100
''', inplace=True)
```

### 10.3 数据类型优化

```python
# 查看内存占用
print(df.memory_usage(deep=True))

# category 类型 — 重复值多的字符串列，能省大量内存
df['category'] = df['category'].astype('category')  # 能省 10-50 倍内存

# downcast — 数值类型降级
df['views'] = pd.to_numeric(df['views'], downcast='integer')  # int64 -> int32/int16
df['ratio'] = pd.to_numeric(df['ratio'], downcast='float')    # float64 -> float32

# 一键优化函数
def optimize_dtypes(df):
    for col in df.select_dtypes(include=['int64']).columns:
        df[col] = pd.to_numeric(df[col], downcast='integer')
    for col in df.select_dtypes(include=['float64']).columns:
        df[col] = pd.to_numeric(df[col], downcast='float')
    for col in df.select_dtypes(include=['object']).columns:
        if df[col].nunique() / len(df[col]) < 0.5:  # 唯一值比例小于50%
            df[col] = df[col].astype('category')
    return df
```

### 10.4 分块读取大文件

```python
# 方式1：指定 nrows 只读一部分（预览用）
df = pd.read_csv('big_file.csv', nrows=10000)

# 方式2：chunksize 分块读取
chunks = pd.read_csv('big_file.csv', chunksize=100000)
for chunk in chunks:
    # 对每个 chunk 进行处理
    process(chunk)

# 方式3：分块处理后合并
chunks = pd.read_csv('big_file.csv', chunksize=100000)
results = []
for chunk in chunks:
    filtered = chunk[chunk['views'] > 1000]
    results.append(filtered)
df = pd.concat(results, ignore_index=True)

# 方式4：只读需要的列
df = pd.read_csv('big_file.csv', usecols=['title', 'views', 'likes'])
```

## 十一、Pandas 与 Spark 对照表

学完 Pandas 之后，看这张表就能快速上手 Spark DataFrame API。

| 操作 | Pandas | Spark DataFrame |
|------|--------|----------------|
| **创建** | `pd.DataFrame(data)` | `spark.createDataFrame(data)` |
| **查看数据** | `df.head()`, `df.show()` | `df.show()` |
| **查看结构** | `df.info()`, `df.dtypes` | `df.printSchema()`, `df.dtypes` |
| **选择列** | `df[['a', 'b']]` | `df.select('a', 'b')` |
| **选择列(表达式)** | `df['a'] * 2` | `df.select(col('a') * 2)` |
| **过滤** | `df[df['a'] > 0]` | `df.filter(col('a') > 0)` |
| **排序** | `df.sort_values('a')` | `df.orderBy('a')` |
| **重命名** | `df.rename(columns={'a':'b'})` | `df.withColumnRenamed('a', 'b')` |
| **新增列** | `df['new'] = df['a'] * 2` | `df.withColumn('new', col('a') * 2)` |
| **删除列** | `df.drop('a', axis=1)` | `df.drop('a')` |
| **缺失值填充** | `df.fillna(0)` | `df.fillna(0)` |
| **删除缺失值** | `df.dropna()` | `df.dropna()` |
| **去重** | `df.drop_duplicates()` | `df.dropDuplicates()` |
| **分组聚合** | `df.groupby('a').agg(...)` | `df.groupBy('a').agg(...)` |
| **连接** | `pd.merge(df1, df2, on='id')` | `df1.join(df2, 'id')` |
| **拼接** | `pd.concat([df1, df2])` | `df1.union(df2)` |
| **类型转换** | `df['a'].astype('int')` | `df.withColumn('a', col('a').cast('int'))` |
| **字符串操作** | `df['a'].str.lower()` | `lower(col('a'))` |
| **统计** | `df.describe()` | `df.describe().show()` |
| **计数** | `df['a'].value_counts()` | `df.groupBy('a').count()` |
| **自定义函数** | `df.apply(func)` | `udf` 或 `pandas_udf` |
| **写入** | `df.to_parquet('f')` | `df.write.parquet('f')` |

### 核心差异

| 维度 | Pandas | Spark |
|------|--------|-------|
| **执行模式** | 立即执行（eager） | 惰性求值（lazy） |
| **数据规模** | 单机，通常 < 内存 | 分布式，可处理 TB 级 |
| **索引** | 有显式索引 | 无索引概念 |
| **不可变性** | 可原地修改 | 不可变，返回新 DataFrame |
| **API 风格** | `df['col']` 取列 | `col('col')` 取列 |

**总结**：Pandas 的 API 更 Pythonic（用 `[]` 取列），Spark 的 API 更 SQL 化（用函数式调用）。但核心操作逻辑是一样的：选择、过滤、排序、分组、连接。掌握了 Pandas，Spark DataFrame 就只是换个写法而已。
