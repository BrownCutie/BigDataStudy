# 作业提交与部署

## 1. spark-submit 详解

### 1.1 基本语法

```bash
spark-submit [选项] [应用程序 JAR 或 Python 文件] [应用程序参数]
```

```
spark-submit 执行流程：

┌──────────────────────────────────────────────────────────┐
│  用户执行 spark-submit                                   │
│       │                                                  │
│       v                                                  │
│  ┌──────────────────┐                                   │
│  │  解析命令行参数   │                                   │
│  │  --master        │                                   │
│  │  --deploy-mode   │                                   │
│  │  --jars/--conf   │                                   │
│  └──────┬───────────┘                                   │
│         │                                                │
│         v                                                │
│  ┌──────────────────┐     ┌──────────────────┐         │
│  │  启动 Driver JVM  │────>│ Cluster Manager  │         │
│  │  运行 main()     │     │ 申请资源         │         │
│  └──────┬───────────┘     └──────┬───────────┘         │
│         │                        │                      │
│         v                        v                      │
│  ┌──────────────────────────────────────────┐           │
│  │  Cluster Manager 启动 Executor            │           │
│  │  Executor 向 Driver 注册                  │           │
│  └──────────────┬───────────────────────────┘           │
│                 │                                        │
│                 v                                        │
│  ┌──────────────────────────────────────────┐           │
│  │  Driver 调度 Task 到 Executor 执行        │           │
│  │  所有 Task 完成后，应用程序退出            │           │
│  └──────────────────────────────────────────┘           │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 1.2 完整参数说明

```bash
# ============ 基本参数 ============

# 应用程序（JAR 或 Python 文件）
spark-submit my_app.py
spark-submit my_app.jar
spark-submit --class com.example.MyApp my_app.jar

# 运行模式
spark-submit --master local[*] my_app.py          # Local 模式
spark-submit --master spark://host:7077 my_app.py  # Standalone 模式
spark-submit --master yarn my_app.py               # YARN 模式
spark-submit --master k8s://... my_app.py          # K8s 模式

# ============ 部署模式 ============

# client 模式（Driver 运行在提交机器上）
spark-submit --master yarn --deploy-mode client my_app.py

# cluster 模式（Driver 运行在集群内部）
spark-submit --master yarn --deploy-mode cluster my_app.py

# ============ 资源配置 ============

# Driver 配置
--driver-memory 4g           # Driver 内存（默认 1g）
--driver-cores 2             # Driver CPU 核数（默认 1）
--driver-library-path /path  # Driver 库路径

# Executor 配置
--executor-memory 8g         # 每个 Executor 内存（默认 1g）
--executor-cores 4           # 每个 Executor CPU 核数（默认 1）
--num-executors 10           # Executor 数量（YARN 模式）

# ============ 依赖管理 ============

# JAR 依赖
--jars /path/to/dep1.jar,/path/to/dep2.jar

# Python 依赖
--py-files /path/to/dep1.py,/path/to/dep2.zip

# Maven 包
--packages com.databricks:spark-xml_2.12:0.17.0

# 排除依赖
--exclude-packages org.slf4j:slf4j-log4j12

# Maven 仓库
--repositories https://maven.aliyun.com/repository/public

# ============ 配置参数 ============

# 通过 --conf 设置
--conf spark.sql.shuffle.partitions=200
--conf spark.sql.adaptive.enabled=true
--conf spark.serializer=org.apache.spark.serializer.KryoSerializer
--conf spark.executor.extraJavaOptions="-Xmx8g"
--conf spark.driver.extraJavaOptions="-Xmx4g"

# 通过 --properties-file 加载配置文件
--properties-file /path/to/spark-defaults.conf

# ============ 应用配置 ============

# 应用名称
--name "MyDemoAnalysis"

# 应用 JAR 的主类（Scala/Java）
--class com.example.demo.DemoAnalysis

# ============ 其他参数 ============

# 传递给应用程序的参数（放在最后）
spark-submit my_app.py --input /data/demos --output /result

# 帮助信息
spark-submit --help

# 强制终止
--kill <submission-id>

# 查看状态
--status <submission-id>
```

## 2. 核心参数详解

### 2.1 --master

```
--master 指定集群管理器：

┌──────────────────────────────────────────────────────────┐
│  值                          │ 说明                      │
├──────────────────────────────┼───────────────────────────┤
│  local                       │ 1 个线程                  │
│  local[K]                    │ K 个线程                  │
│  local[*]                    │ 使用所有可用 CPU 核心      │
│  spark://host:port           │ Standalone 模式           │
│  yarn                        │ YARN 模式                 │
│  k8s://https://host:port     │ Kubernetes 模式           │
│  mesos://host:port           │ Mesos 模式（已过时）      │
└──────────────────────────────┴───────────────────────────┘
```

### 2.2 --deploy-mode

```
deploy-mode 对比：

client 模式：
  ┌────────────────┐         ┌──────────────────────┐
  │  提交机器       │         │  YARN Cluster         │
  │  ┌──────────┐  │         │                      │
  │  │ Driver   │  │ ──┐    │  ┌──────────────────┐ │
  │  │ (本地)   │  │   │    │  │  Executor 1      │ │
  │  │ WebUI:4040│  │   │    │  └──────────────────┘ │
  │  └──────────┘  │   │    │  ┌──────────────────┐ │
  └────────────────┘   ├────> │  │  Executor 2      │ │
                       │    │  └──────────────────┘ │
  特点：                  │    └──────────────────────┘
  - Driver 运行在提交机器上
  - 可以直接看到日志输出
  - 提交机器关闭则任务失败
  - 适合交互式/调试

cluster 模式：
  ┌────────────────┐         ┌──────────────────────┐
  │  提交机器       │         │  YARN Cluster         │
  │  spark-submit  │ ──>    │  ┌──────────────────┐ │
  │  (仅提交)      │         │  │  ApplicationMaster│ │
  └────────────────┘         │  │  (Driver)        │ │
                             │  └──────────────────┘ │
                             │  ┌──────────────────┐ │
                             │  │  Executor 1      │ │
                             │  └──────────────────┘ │
                             │  ┌──────────────────┐ │
                             │  │  Executor 2      │ │
                             │  └──────────────────┘ │
                             └──────────────────────┘

  特点：
  - Driver 运行在 YARN 集群内部
  - 提交机器可以关闭，不影响任务
  - 日志需要通过 yarn logs 查看
  - 适合生产环境
```

### 2.3 --executor-memory / --executor-cores / --num-executors

```
Executor 资源配置：

                    ┌──────────────────────┐
                    │  YARN NodeManager    │
                    │  总内存: 64GB        │
                    │  总核心: 32          │
                    │                      │
                    │  ┌──────┐  ┌──────┐ │
                    │  │Exec 1│  │Exec 2│ │
                    │  │8GB   │  │8GB   │ │
                    │  │4核   │  │4核   │ │
                    │  │      │  │      │ │
                    │  │┌────┐│  │┌────┐│ │
                    │  ││Task││  ││Task││ │
                    │  ││Task││  ││Task││ │
                    │  ││Task││  ││Task││ │
                    │  ││Task││  ││Task││ │
                    │  │└────┘│  │└────┘│ │
                    │  └──────┘  └──────┘ │
                    └──────────────────────┘

参数说明：
  --executor-memory 8g    每个 Executor 的内存
  --executor-cores 4      每个 Executor 的 CPU 核数
  --num-executors 10      Executor 总数量

总资源计算：
  总内存 = num-executors x executor-memory
  总核心 = num-executors x executor-cores
```

## 4. Spark 作业资源计算公式

### 4.1 YARN 模式资源计算

```
YARN 模式资源计算：

总可用资源：
  集群总内存 = NodeManager 数量 x 每个 NodeManager 内存
  集群总核心 = NodeManager 数量 x 每个 NodeManager 核心

资源分配公式：
  ┌────────────────────────────────────────────────────────┐
  │                                                        │
  │  num-executors = (集群总核心 - Driver 核心数) /         │
  │                  executor-cores - 1                     │
  │                                                        │
  │  executor-memory = (集群总内存 x 0.8 - Driver 内存) /  │
  │                    num-executors                        │
  │                                                        │
  │  注意：                                                │
  │  - 预留 20% 内存给操作系统和 YARN 本身                │
  │  - 每个 NodeManager 预留 1-2 GB                       │
  │  - Driver 内存通常 2-4 GB                              │
  │  - executor-memory 不要超过 NodeManager 内存的 80%     │
  │                                                        │
  └────────────────────────────────────────────────────────┘
```

### 4.2 资源计算示例

```
场景：YARN 集群有 5 个 NodeManager，每个 64GB 内存、32 核心

Step 1：计算可用资源
  总内存 = 5 x 64 = 320 GB
  总核心 = 5 x 32 = 160 核心
  可用内存 = 320 x 0.8 = 256 GB（预留 20%）
  可用核心 = 160 - 2 = 158 核心（预留 2 给 Driver）

Step 2：确定 Executor 配置
  executor-cores = 5（每个 Executor 5 个核心，推荐 4-6）
  num-executors = 158 / 5 = 31（取整，减去 1）
  num-executors = 30

Step 3：确定 Executor 内存
  Driver 内存 = 4 GB
  executor-memory = (256 - 4) / 30 = 8.4 GB
  executor-memory = 8 GB（向下取整到 2 的幂次）

Step 4：验证
  实际使用内存 = 4 + 30 x 8 = 244 GB < 256 GB ✓
  实际使用核心 = 2 + 30 x 5 = 152 < 158 ✓
  每个 NodeManager：
    内存 = 244 / 5 = 48.8 GB < 64 x 0.8 = 51.2 GB ✓
    核心 = 152 / 5 = 30.4 < 32 ✓

最终配置：
  --num-executors 30 \
  --executor-memory 8g \
  --executor-cores 5 \
  --driver-memory 4g
```

## 5. 动态资源分配（Dynamic Allocation）

### 5.1 原理

```
静态资源分配 vs 动态资源分配：

静态分配：
  ┌──────────────────────────────────────────────────────────┐
  │  提交时指定 10 个 Executor                                │
  │                                                          │
  │  时间轴：                                                │
  │  ┌──────────────────────────────────────────────────────┐│
  │  │Exec: 10 10 10 10 10 10 10 10 10 10                  ││
  │  │Task: 20 15  5  0  0  0 10  8  2  0                  ││
  │  └──────────────────────────────────────────────────────┘│
  │  问题：后 5 个 Executor 空闲但仍然占用资源               │
  └──────────────────────────────────────────────────────────┘

动态分配：
  ┌──────────────────────────────────────────────────────────┐
  │  根据负载动态调整 Executor 数量                           │
  │                                                          │
  │  时间轴：                                                │
  │  ┌──────────────────────────────────────────────────────┐│
  │  │Exec: 10 10  8  3  2  2  6  5  2  0                  ││
  │  │Task: 20 15  5  0  0  0 10  8  2  0                  ││
  │  └──────────────────────────────────────────────────────┘│
  │  优势：空闲时自动释放 Executor，需要时自动申请            │
  └──────────────────────────────────────────────────────────┘

动态分配参数：
  spark.dynamicAllocation.enabled = true
  spark.dynamicAllocation.minExecutors = 2       # 最少保留 2 个
  spark.dynamicAllocation.maxExecutors = 20      # 最多 20 个
  spark.dynamicAllocation.initialExecutors = 5   # 初始 5 个
  spark.dynamicAllocation.schedulerBacklogTimeout = 1s  # 有积压任务时申请
  spark.dynamicAllocation.sustainedSchedulerBacklogTimeout = 5s
  spark.dynamicAllocation.executorIdleTimeout = 60s  # 空闲 60s 后释放
  spark.dynamicAllocation.cachedExecutorIdleTimeout = 120s  # 有缓存时 120s
```

### 5.2 配置和使用

```bash
# 启用动态资源分配
spark-submit \
    --master yarn \
    --deploy-mode client \
    --conf spark.dynamicAllocation.enabled=true \
    --conf spark.shuffle.service.enabled=true \
    --conf spark.dynamicAllocation.minExecutors=2 \
    --conf spark.dynamicAllocation.maxExecutors=20 \
    --conf spark.dynamicAllocation.initialExecutors=5 \
    --conf spark.dynamicAllocation.executorIdleTimeout=60s \
    --executor-memory 8g \
    --executor-cores 4 \
    my_app.py
```

```python
# 代码中配置
spark.conf.set("spark.dynamicAllocation.enabled", "true")
spark.conf.set("spark.shuffle.service.enabled", "true")
spark.conf.set("spark.dynamicAllocation.minExecutors", "2")
spark.conf.set("spark.dynamicAllocation.maxExecutors", "20")
```

### 5.3 注意事项

```
动态资源分配的注意事项：

1. 必须启用 Shuffle Service
   spark.shuffle.service.enabled=true
   Shuffle Service 需要在每个 NodeManager 上启动

2. 与缓存冲突
   动态分配可能回收有缓存数据的 Executor
   解决方案：
   - 增大 cachedExecutorIdleTimeout
   - 或关闭动态分配（spark.dynamicAllocation.enabled=false）

3. YARN 模式下
   需要配置 yarn-site.xml 中的 shuffle service

4. 不适合所有场景
   - 实时流处理：需要稳定的资源
   - 频繁使用缓存的任务
   - 需要严格资源隔离的场景
```

## 7. 常见部署错误排查

### 7.1 错误排查流程

```
Spark 作业部署错误排查流程：

┌──────────────────────────────────────────────────────────┐
│                                                          │
│  1. 作业无法提交？                                       │
│     ├── spark-submit 命令语法错误 -> 检查参数格式         │
│     ├── 连不上 Cluster Manager -> 检查 --master 地址     │
│     ├── 找不到主类 -> 检查 --class 参数和包名             │
│     └── 文件不存在 -> 检查路径是否正确                    │
│                                                          │
│  2. 资源申请失败？                                       │
│     ├── 内存不足 -> 减少 executor-memory/num-executors   │
│     ├── 核心不足 -> 减少 executor-cores/num-executors    │
│     ├── YARN 队列资源不足 -> 检查 yarn queue 配置        │
│     └── 动态分配问题 -> 检查 shuffle service 是否启动   │
│                                                          │
│  3. 作业运行中失败？                                     │
│     ├── Executor OOM -> 增大 executor-memory             │
│     ├── Driver OOM -> 增大 driver-memory                 │
│     ├── Container killed -> 增大 memoryOverhead           │
│     ├── Task 失败 -> 检查日志中的异常信息                │
│     └── Shuffle 失败 -> 检查磁盘空间和网络               │
│                                                          │
│  4. 作业运行缓慢？                                       │
│     ├── 数据倾斜 -> 查看任务耗时分布                     │
│     ├── GC 过多 -> 增大内存或优化代码                    │
│     ├── Shuffle 过大 -> 优化 Join/GroupBy                │
│     ├── 小文件过多 -> 合并文件/调整分区数                │
│     └── 序列化慢 -> 使用 Kryo 序列化                    │
│                                                          │
└──────────────────────────────────────────────────────────┘
```
