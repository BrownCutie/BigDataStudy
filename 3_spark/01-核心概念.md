# Spark 3.5.8 核心概念

## 1. 什么是 Apache Spark

Apache Spark 是一个开源的分布式计算引擎，专为大规模数据处理而设计。它由 UC Berkeley AMPLab 开发，后捐赠给 Apache 软件基金会，成为顶级项目。

### 1.1 Spark 的核心优势

```
+------------------+     +------------------+     +------------------+
|    MapReduce     |     |      Spark       |     |    对比结论      |
+------------------+     +------------------+     +------------------+
| 磁盘读写为主     |     | 内存计算为主     |     | Spark 快 10-100x |
| 每次任务写磁盘   |     | DAG 优化执行     |     | 减少 I/O 开销    |
| 只支持批处理     |     | 批/流/ML/图一体  |     | 统一计算平台     |
| 编程模型复杂     |     | 高级 API 简洁    |     | 开发效率高       |
| 任务间无优化     |     | Catalyst 优化器  |     | 自动查询优化     |
+------------------+     +------------------+     +------------------+
```

### 1.2 为什么 Spark 比 MapReduce 快

MapReduce 的执行模型是 **stage 之间必须落盘**，每个 Map 阶段结束后，结果写入 HDFS，再由 Reduce 阶段读取。这种频繁的磁盘 I/O 严重拖慢了性能。

Spark 引入了以下关键优化：

```
MapReduce 执行流程：
┌──────┐    ┌──────┐    ┌──────┐    ┌──────┐
| Map1 | -> | 磁盘 | -> | Reduce1 | -> | 磁盘 |
└──────┘    └──────┘    └──────┘    └──────┘
                                          |
                                          v
┌──────┐    ┌──────┐    ┌──────┐    ┌──────┐
| Map2 | -> | 磁盘 | -> | Reduce2 | -> | 磁盘 |
└──────┘    └──────┘    └──────┘    └──────┘

Spark 执行流程：
┌──────┐    ┌──────┐    ┌──────┐    ┌──────┐
| Map1 | -> | 内存 | -> | Map2  | -> | Reduce|
└──────┘    └──────┘    └──────┘    └──────┘
                          ^             |
                          |             v
                       DAG 编译器自动优化执行计划
```

核心原因有三个：

1. **内存计算**：Spark 将中间结果缓存在内存中，避免频繁磁盘 I/O
2. **DAG 执行引擎**：DAG（有向无环图）调度器将多个操作优化为一个执行计划，减少 stage 数量
3. **流水线执行**：窄依赖（narrow dependency）的 Transformation 可以在一个 stage 内流水线执行

```
                    └─────────┘
```

### 2.2 核心组件说明

| 组件 | 职责 |
|------|------|
| **Driver（驱动程序）** | 运行用户程序的 main() 函数，创建 SparkContext，将用户代码转化为 DAG，负责任务调度 |
| **Cluster Manager（集群管理器）** | 管理集群资源分配，支持 YARN、Standalone、K8s 等模式 |
| **Worker Node（工作节点）** | 集群中实际执行计算任务的节点 |
| **Executor（执行器）** | 每个 Worker 上运行的进程，负责执行 Task，存储数据缓存 |
| **Task（任务）** | 被分配到 Executor 上执行的最小计算单元 |

### 2.3 Spark 作业执行流程

```
用户代码（spark-submit / pyspark）
         │
         v
  ┌──────────────┐
  │   Driver     │
  │              │
  │ 1. 解析代码  │
  │ 2. 构建 DAG  │
  │ 3. 划分 Stage│
  │ 4. 生成 Task │
  └──────┬───────┘
         │ 注册 Application
         v
  ┌──────────────┐
  │Cluster Manager│
  │              │
  │ 分配资源      │
  │ 启动 Executor │
  └──────┬───────┘
         │ 资源分配完成
         v
  ┌──────────────┐
  │   Executor   │
  │              │
  │ 接收 Task    │
  │ 执行计算     │
  │ 返回结果     │
  │ 存储缓存     │
  └──────────────┘
```

详细步骤：

1. 用户提交 Spark 应用程序
2. Driver 程序启动，创建 `SparkContext`
3. `SparkContext` 向 Cluster Manager 注册并申请资源
4. Cluster Manager 在 Worker 节点上启动 Executor 进程
5. Executor 向 Driver 反向注册，报告可用资源
6. Driver 将用户代码解析为 DAG，划分 Stage 和 Task
7. TaskScheduler 将 Task 分发给 Executor 执行
8. Executor 执行 Task 并将结果返回给 Driver
9. 所有 Task 完成后，Driver 程序退出

## 3. 三代 API 演进

Spark 提供了三代数据处理 API，从底层到高层逐步演进：

```
┌─────────────────────────────────────────────────┐
│              Spark API 演进路线                   │
├─────────────────────────────────────────────────┤
│                                                 │
│  Spark 1.0          Spark 1.3        Spark 1.6  │
│  ┌─────────┐       ┌──────────┐    ┌────────┐  │
│  │   RDD   │ ───> │DataFrame │──> │Dataset │  │
│  │(底层API)│       │(高层API) │    │(类型安全)│ │
│  └─────────┘       └──────────┘    └────────┘  │
│       │                 │               │       │
│       v                 v               v       │
│   Java/Python/      Python/R/       Scala/Java │
│   Scala 全支持       Scala 支持      强类型支持 │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 3.1 RDD（Resilient Distributed Dataset）

RDD 是 Spark 最底层的数据抽象，是一个不可变、分区的、容错的分布式数据集。

```python
from pyspark import SparkContext

sc = SparkContext("local[*]", "RDDExample")

# 从集合创建 RDD
rdd = sc.parallelize([1, 2, 3, 4, 5])

# Transformation（转换）
squared = rdd.map(lambda x: x * x)

# Action（行动）
result = squared.collect()
print(result)  # [1, 4, 9, 16, 25]

sc.stop()
```

**RDD 特点**：
- 类型安全（Scala/Java），但 Python 中退化为动态类型
- 提供底层的分区控制
- 性能优化完全依赖开发者手动处理
- API 偏底层，操作不够直观

### 3.2 DataFrame

DataFrame 是以命名列组织的分布式数据集，概念上等同于关系数据库中的表。

```python
from pyspark.sql import SparkSession

spark = SparkSession.builder \
    .appName("DataFrameExample") \
    .getOrCreate()

# 从 JSON 创建 DataFrame
df = spark.read.json("demos.json")

# 使用 DataFrame API 查询
result = df.filter(df["views"] > 1000) \
           .select("title", "views") \
           .orderBy("views", ascending=False)

result.show()
```

**DataFrame 特点**：
- 提供了 Schema 信息（列名和类型）
- 内置 Catalyst 优化器，自动优化查询计划
- 支持 SQL 查询
- Python/R/Scala/Java 全语言支持
- 对开发者友好，类似 Pandas 的操作方式

### 3.3 Dataset

Dataset 是 DataFrame 的类型安全扩展（Scala/Java），结合了 RDD 的类型安全和 DataFrame 的高性能。

```scala
// Scala 示例
case class Demo(title: String, author: String, views: Long)

val spark = SparkSession.builder().appName("DatasetExample").getOrCreate()
import spark.implicits._

// 创建 Dataset
val demosDS = Seq(
  Demo("Spark入门", "张三", 1500L),
  Demo("Hadoop教程", "李四", 2300L)
).toDS()

// 类型安全的操作
val popularDemos = demosDS.filter(_.views > 1000).map(_.title)
```

**Dataset 特点**：
- 编译期类型检查（仅 Scala/Java）
- 同时拥有 RDD 的类型安全和 DataFrame 的性能优化
- 在 Python 中，DataFrame 和 Dataset 是同一回事（Python 没有静态类型系统）

### 3.4 三代 API 对比

| 特性 | RDD | DataFrame | Dataset |
|------|-----|-----------|---------|
| API 层级 | 底层 | 高层 | 高层 |
| 类型安全 | 是（编译期） | 否（运行时） | 是（编译期） |
| 自动优化 | 无 | Catalyst + Tungsten | Catalyst + Tungsten |
| 序列化 | Java 原生 | Tungsten 二进制 | Tungsten 二进制 |
| 语言支持 | 全语言 | 全语言 | Scala/Java |
| 自定义函数 | map/filter | UDF | map/filter + UDF |
| 适用场景 | 底层控制、非结构化数据 | 结构化/半结构化数据 | 需要 Scala 类型安全 |

### 3.5 统一 API（Spark 2.0+）

```
从 Spark 2.0 开始，DataFrame 和 Dataset 统一了：

  DataFrame = Dataset[Row]  （在 Scala 中）
  DataFrame = Dataset        （在 Python 中，因为 Python 没有泛型）

  ┌─────────────────────────────────┐
  │       DataFrame / Dataset       │
  │                                 │
  │  ┌───────────┐  ┌────────────┐ │
  │  │ DataFrame │  │  Dataset[T]│ │
  │  │ (Row 类型)│  │ (自定义类型)│ │
  │  └───────────┘  └────────────┘ │
  │         共享 Catalyst 优化引擎   │
  └─────────────────────────────────┘
```

## 6. 惰性求值（Lazy Evaluation）

### 6.1 原理

Spark 的 Transformation 操作是**惰性求值**的——它们不会立即执行，而是记录数据的转换逻辑，构建出一个执行计划图（DAG）。只有遇到 Action 操作时，才会真正触发计算。

```
惰性求值执行流程：

代码：
  rdd = sc.parallelize([1,2,3,4,5])
  rdd2 = rdd.map(lambda x: x * 2)      # Transformation，不执行
  rdd3 = rdd2.filter(lambda x: x > 5)   # Transformation，不执行
  rdd4 = rdd3.map(lambda x: x + 1)      # Transformation，不执行
  result = rdd4.collect()                # Action，触发执行！

执行计划 DAG：
  ┌──────────┐    ┌──────────┐    ┌──────────┐
  │  map(*2) │ -> │ filter(>5)│ -> │ map(+1) │ -> collect()
  │ (懒执行) │    │ (懒执行)  │    │ (懒执行) │    (触发)
  └──────────┘    └──────────┘    └──────────┘

  没有 Action 时，以上三个 Transformation 都不会执行
  遇到 collect() 时，Spark 将整个链路编译为一个任务执行
```

### 6.2 为什么使用惰性求值

1. **整体优化**：Spark 可以看到整个转换链，通过 Catalyst 优化器进行全局优化
2. **减少中间结果**：不需要为每个 Transformation 生成中间数据
3. **容错恢复**：记录血缘关系（Lineage），丢失分区可以重新计算
4. **管道合并**：多个窄依赖操作可以合并到一个 Task 中执行

```
不使用惰性求值（传统方式）：
  step1: 读取数据 -> 写入磁盘
  step2: 读取磁盘 -> map -> 写入磁盘
  step3: 读取磁盘 -> filter -> 写入磁盘
  step4: 读取磁盘 -> collect
  总共 3 次磁盘读写

使用惰性求值（Spark 方式）：
  step1: 记录 map 操作
  step2: 记录 filter 操作
  step3: collect() 触发执行，流水线处理
  总共 0 次中间磁盘读写
```

### 6.3 血缘关系（Lineage）

每个 RDD/DataFrame 都记录了它是从哪个父数据集通过什么操作转换而来的。

```
血缘关系示例：

  textFile("demos.txt")     -- RDD_A
         |
         map(parse_line)     -- RDD_B
         |
         filter(valid)       -- RDD_C
         |
         map(extract_views)  -- RDD_D

  如果 RDD_D 的某个分区丢失：
  -> Spark 通过血缘关系找到 RDD_A
  -> 重新计算 RDD_A -> RDD_B -> RDD_C -> RDD_D 的该分区
  -> 而不需要从头计算所有数据
```

## 7. Transformations vs Actions

### 7.1 核心区别

```
┌────────────────────────────────────────────────────────┐
│                  Transformations                        │
│                                                        │
│  特点：                                                  │
│  - 惰性执行，返回新的 RDD/DataFrame                      │
│  - 不会立即触发计算                                       │
│  - 定义数据如何从一个形式转换为另一个形式                    │
│                                                        │
│  分类：                                                  │
│  ┌──────────────────┐  ┌──────────────────┐           │
│  │    窄依赖         │  │    宽依赖         │           │
│  │  Narrow Dep.     │  │  Wide/Shuffle Dep│           │
│  │                  │  │                  │           │
│  │  map, filter,    │  │  groupByKey,     │           │
│  │  flatMap, union  │  │  reduceByKey,    │           │
│  │                  │  │  join, distinct  │           │
│  │  数据不需要跨分区  │  │  数据需要跨分区   │           │
│  │  移动，可流水线   │  │  移动（Shuffle）  │           │
│  └──────────────────┘  └──────────────────┘           │
└────────────────────────────────────────────────────────┘

┌────────────────────────────────────────────────────────┐
│                     Actions                             │
│                                                        │
│  特点：                                                  │
│  - 立即执行，触发所有 Transformation 计算                  │
│  - 返回结果给 Driver 或写入外部存储                       │
│  - 产生最终结果                                          │
│                                                        │
│  分类：                                                  │
│  ┌──────────────────┐  ┌──────────────────┐           │
│  │    驱动端结果     │  │    数据写入       │           │
│  │                  │  │                  │           │
│  │  collect()       │  │  write.parquet() │           │
│  │  count()         │  │  saveAsTextFile()│           │
│  │  first()         │  │  insertInto()    │           │
│  │  take(n)         │  │  saveAsTable()   │           │
│  │  reduce()        │  │                  │           │
│  │  foreach()       │  │                  │           │
│  └──────────────────┘  └──────────────────┘           │
└────────────────────────────────────────────────────────┘
```

### 7.2 窄依赖 vs 宽依赖

```
窄依赖（Narrow Dependency）：
每个父分区最多被一个子分区使用

  父 RDD          子 RDD
  ┌──────┐       ┌──────┐
  │ P0   │──────>│ P0'  │
  ├──────┤       ├──────┤
  │ P1   │──────>│ P1'  │
  ├──────┤       ├──────┤
  │ P2   │──────>│ P2'  │
  └──────┘       └──────┘

  例：map, filter, flatMap
  特点：可以在一个 Task 中流水线执行，不需要 Shuffle


宽依赖（Wide Dependency / Shuffle Dependency）：
每个父分区可能被多个子分区使用

  父 RDD          Shuffle           子 RDD
  ┌──────┐       ┌──────┐        ┌──────┐
  │ P0   │──┐    │      │   ┌───>│ P0'  │
  ├──────┤  ├──->│ 网络 │   │    ├──────┤
  │ P1   │──┤    │ Shuffle│──>──>│ P1'  │
  ├──────┤  │    │      │   │    ├──────┤
  │ P2   │──┘    │      │   └───>│ P2'  │
  └──────┘       └──────┘        └──────┘

  例：groupByKey, reduceByKey, join
  特点：需要 Shuffle，是 Stage 划分的边界
```

### 7.3 常用 Transformation 和 Action 列表

**Transformations**：

| 操作 | 说明 | 示例 |
|------|------|------|
| `map(func)` | 对每个元素应用函数 | `rdd.map(lambda x: x * 2)` |
| `filter(func)` | 过滤元素 | `rdd.filter(lambda x: x > 0)` |
| `flatMap(func)` | 映射并展平 | `rdd.flatMap(lambda x: x.split(" "))` |
| `union(other)` | 合并两个 RDD | `rdd1.union(rdd2)` |
| `distinct()` | 去重 | `rdd.distinct()` |
| `reduceByKey(func)` | 按 key 聚合 | `kv_rdd.reduceByKey(lambda a, b: a + b)` |
| `sortByKey()` | 按 key 排序 | `kv_rdd.sortByKey()` |
| `join(other)` | 关联两个 RDD | `rdd1.join(rdd2)` |

**Actions**：

| 操作 | 说明 | 返回值 |
|------|------|--------|
| `collect()` | 收集所有数据到 Driver | 列表 |
| `count()` | 返回元素总数 | 整数 |
| `first()` | 返回第一个元素 | 单个元素 |
| `take(n)` | 返回前 n 个元素 | 列表 |
| `reduce(func)` | 聚合所有元素 | 单个元素 |
| `foreach(func)` | 对每个元素执行函数 | None |
| `saveAsTextFile(path)` | 保存为文本文件 | None |

### 7.4 完整示例：博客文章处理

```python
from pyspark.sql import SparkSession

spark = SparkSession.builder \
    .appName("DemoAnalysis") \
    .getOrCreate()

# 创建业务数据
demo_data = [
    ("Spark入门教程", "张三", 1500, "2024-01-15"),
    ("Hadoop核心原理", "张三", 2300, "2024-02-20"),
    ("Hive数据仓库", "李四", 800, "2024-03-10"),
    ("Flink流处理", "王五", 3200, "2024-04-05"),
    ("Spark性能调优", "张三", 4500, "2024-05-18"),
]

columns = ["title", "author", "views", "publish_date"]
df = spark.createDataFrame(demo_data, columns)

# 以下全部是 Transformation，不会执行
step1 = df.filter(df["views"] > 1000)          # 过滤浏览量 > 1000
step2 = step1.select("author", "views")         # 选择列
step3 = step2.groupBy("author") \               # 按作者分组
    .agg({"views": "sum"}) \                     # 计算总浏览量
    .withColumnRenamed("sum(views)", "total_views")  # 重命名

# Action：触发执行
step3.orderBy("total_views", ascending=False).show()

# 输出：
# +------+-----------+
# |author|total_views|
# +------+-----------+
# |  张三 |      8300|
# |  王五 |      3200|
# |  李四 |       800|
# +------+-----------+

spark.stop()
```
