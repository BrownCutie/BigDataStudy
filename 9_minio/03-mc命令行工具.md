# 03 - mc 命令行工具详解：对象存储的瑞士军刀

## 一、课前回顾

在上一节 [02-安装与配置.md](02-安装与配置.md) 中，我们完成了 MinIO 的安装和配置，也安装了 mc 客户端并创建了 `demo-assets` 和 `demo-data` 两个存储桶。本节我们将系统学习 mc 命令行工具，掌握日常操作对象存储所需的所有命令。

## 二、mc 基础认知

### 2.1 mc 是什么？

**mc**（MinIO Client）是 MinIO 官方提供的命令行工具，用来管理 MinIO 和其他 S3 兼容的存储服务。

```
┌─────────────────────────────────────────────────────────────────┐
│                    mc 的定位                                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  mc 之于对象存储，就像：                                         │
│  - ls 之于文件系统                                              │
│  - mysql 之于 MySQL                                            │
│  - hdfs dfs 之于 HDFS                                          │
│                                                                 │
│  它是你日常操作 MinIO 最常用的工具。                              │
│                                                                 │
│  mc 支持的存储服务：                                             │
│  ┌────────────────────────────────────────────────────────┐    │
│  │  MinIO    │  Amazon S3  │  阿里 OSS  │  腾讯 COS      │    │
│  │  Google   │  Azure      │  Ceph      │  任何 S3 兼容   │    │
│  └────────────────────────────────────────────────────────┘    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 mc 命令的基本格式

```bash
mc <命令> <别名/路径> [参数]
```

```
┌─────────────────────────────────────────────────────────────────┐
│                    mc 命令格式解析                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  mc ls myminio/demo-assets/images/                             │
│     │   │        │              │                               │
│     │   │        │              └── Key 前缀（可选）             │
│     │   │        └── Bucket 名称                                │
│     │   └── alias（别名，在 mc alias set 时配置）               │
│     └── 操作命令                                                │
│                                                                 │
│  路径格式：alias/bucket/key                                     │
│  例如：  myminio/demo-assets/posts/2024/cover.jpg               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 三、Bucket 管理命令

### 3.1 创建 Bucket（mb - make bucket）

```bash
# 创建新桶
mc mb myminio/test-bucket

# 创建桶时指定区域
mc mb myminio/test-bucket --region us-east-1
```

### 3.2 删除 Bucket（rb - remove bucket）

```bash
# 删除空桶
mc rb myminio/test-bucket

# 强制删除非空桶（慎用！会删除桶内所有文件）
mc rb --force myminio/test-bucket
```

### 3.3 列出 Bucket（ls - list）

```bash
# 列出所有 Bucket
mc ls myminio

# 列出指定 Bucket 中的对象
mc ls myminio/demo-assets

# 递归列出所有对象（包括子目录）
mc ls --recursive myminio/demo-assets

# 只列出最新版本的对象
mc ls myminio/demo-assets --latest
```

### 3.4 查看 Bucket 信息（stat - statistics）

```bash
# 查看 Bucket 信息
mc stat myminio/demo-assets

# 查看某个对象的信息
mc stat myminio/demo-assets/test-upload.txt
```

输出示例：
```
Name      : test-upload.txt
Date      : 2024-03-15 10:30:00 CST
Size      : 42 B
ETag      : d41d8cd98f00b204e9800998ecf8427e
Type      : file
Metadata  :
  Content-Type: text/plain
```

### 3.5 查看 Bucket 策略（anonymous）

```bash
# 查看当前桶的访问策略
mc anonymous get myminio/demo-assets

# 设置桶为公开下载（任何人都可以下载，但不能上传）
mc anonymous set download myminio/demo-assets

# 设置桶为公开上传和下载
mc anonymous set upload myminio/demo-assets

# 设置桶为私有（默认）
mc anonymous set none myminio/demo-assets

# 列出所有桶的策略
mc anonymous list myminio
```

## 四、文件操作命令

### 4.1 上传文件（cp - copy）

```bash
# 上传单个文件
mc cp /tmp/photo.jpg myminio/demo-assets/images/photo.jpg

# 上传并重命名
mc cp /tmp/photo.jpg myminio/demo-assets/images/2024-cover.jpg

# 上传整个目录（注意末尾的 /）
mc cp /tmp/demo-images/ myminio/demo-assets/images/

# 上传时添加自定义元数据
mc cp /tmp/photo.jpg myminio/demo-assets/images/photo.jpg \
  --attr "x-amz-meta-author=browncutie;x-amz-meta-category=tech"
```

```
┌─────────────────────────────────────────────────────────────────┐
│                    mc cp 的两种模式                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  模式 1：上传本地文件到 MinIO                                     │
│  mc cp <本地路径> <MinIO路径>                                    │
│  mc cp /tmp/file.txt myminio/bucket/key.txt                    │
│                                                                 │
│  模式 2：从 MinIO 下载文件到本地                                 │
│  mc cp <MinIO路径> <本地路径>                                    │
│  mc cp myminio/bucket/key.txt /tmp/file.txt                    │
│                                                                 │
│  模式 3：在 MinIO 内部复制                                       │
│  mc cp <MinIO路径1> <MinIO路径2>                                │
│  mc cp myminio/bucket1/file.txt myminio/bucket2/file.txt       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4.2 下载文件（cp - copy）

```bash
# 下载单个文件
mc cp myminio/demo-assets/images/photo.jpg /tmp/downloaded-photo.jpg

# 下载整个目录
mc cp --recursive myminio/demo-assets/images/ /tmp/downloaded-images/
```

### 4.3 查看文件内容（cat）

```bash
# 查看文本文件内容（直接输出到终端）
mc cat myminio/demo-assets/test-upload.txt

# 将文件内容通过管道传递给其他命令
mc cat myminio/demo-assets/data.csv | head -10
mc cat myminio/demo-assets/config.json | python3 -m json.tool
```

### 4.4 删除文件（rm - remove）

```bash
# 删除单个文件
mc rm myminio/demo-assets/test-upload.txt

# 递归删除（删除桶内所有文件）
mc rm --recursive myminio/demo-assets/images/

# 递归删除并强制确认
mc rm --recursive --force myminio/demo-assets/images/

# 删除所有旧版本（仅保留最新版本）
mc rm --recursive --force --versions myminio/demo-assets/
```

### 4.5 批量操作

```bash
# 批量上传（通配符）
mc cp /tmp/images/*.jpg myminio/demo-assets/images/

# 批量删除（通配符）
mc rm myminio/demo-assets/images/tmp_*.jpg

# 查找大于 10MB 的文件
mc ls --recursive myminio/demo-assets | awk '$4 > 10*1024*1024'

# 统计 Bucket 内文件数量和总大小
mc ls --recursive --summarize myminio/demo-assets
```

## 五、高级操作

### 5.1 mc mirror — 目录同步

`mc mirror` 是一个非常有用的命令，用于在本地目录和 MinIO 之间进行双向同步：

```
┌─────────────────────────────────────────────────────────────────┐
│                    mc mirror 工作原理                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  本地目录                        MinIO Bucket                   │
│  ┌──────────┐                   ┌──────────────┐              │
│  │ file1.txt│ ──── mirror ────→ │ file1.txt    │              │
│  │ file2.jpg│                   │ file2.jpg    │              │
│  │ file3.pdf│ ──── 新增 ────→  │ file3.pdf    │              │
│  │ (已删除) │ ──── 删除 ────→  │ (同步删除)   │              │
│  └──────────┘                   └──────────────┘              │
│                                                                 │
│  mirror 会：                                                    │
│  1. 比较源和目标，找出差异                                       │
│  2. 上传源中有但目标中没有的文件                                  │
│  3. 删除目标中有但源中没有的文件（加 --overwrite 才生效）         │
│  4. 跳过内容相同的文件                                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

```bash
# 本地 → MinIO（将本地目录同步到 MinIO）
mc mirror /tmp/demo-images/ myminio/demo-assets/images/

# MinIO → 本地（将 MinIO 同步到本地）
mc mirror myminio/demo-assets/images/ /tmp/backup/

# 只同步，不删除目标中多余的文件
mc mirror --watch /tmp/demo-images/ myminio/demo-assets/images/

# 覆盖模式（目标文件会被完全替换为源）
mc mirror --overwrite /tmp/demo-images/ myminio/demo-assets/images/

# 排除特定文件
mc mirror --exclude "*.tmp" --exclude ".DS_Store" \
  /tmp/demo-images/ myminio/demo-assets/images/
```

### 5.2 mc admin — 管理员命令

`mc admin` 用于管理 MinIO 服务本身，包括用户管理、策略管理等：

```bash
# 查看服务器信息
mc admin info myminio

# 查看服务器配置
mc admin config get myminio

# 查看服务日志
mc admin logs myminio

# 健康检查
mc admin info myminio --health
```

### 5.3 用户管理

```bash
# 创建新用户
mc admin user add myminio demo-user demo-password123

# 列出所有用户
mc admin user list myminio

# 禁用用户
mc admin user disable myminio demo-user

# 启用用户
mc admin user enable myminio demo-user

# 删除用户
mc admin user remove myminio demo-user

# 查看用户信息
mc admin user info myminio demo-user
```

### 5.4 策略管理

策略（Policy）定义了用户的权限：

```bash
# 列出所有内置策略
mc admin policy list myminio

# 查看某个策略的详细内容
mc admin policy info myminio readwrite

# 将策略附加给用户
mc admin policy attach myminio readwrite --user demo-user

# 创建自定义策略（从 JSON 文件）
mc admin policy create myminio demo-policy /tmp/demo-policy.json

# 删除自定义策略
mc admin policy remove myminio demo-policy
```

**常用内置策略：**

```
┌─────────────────────────────────────────────────────────────────┐
│                    MinIO 内置策略                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  策略名           权限范围                                        │
│  ─────────────────────────────────────────────                  │
│  consoleAdmin    控制台管理员权限                                 │
│  diagnostics     诊断信息查看权限                                 │
│  readonly        所有 Bucket 的只读权限                           │
│  readwrite       所有 Bucket 的读写权限                           │
│  writeonly       所有 Bucket 的只写权限                           │
│                                                                 │
│  自定义策略可以精确控制到特定 Bucket 和特定操作                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**自定义策略示例（只允许访问 demo-assets 桶）：**

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:GetBucketLocation",
        "s3:GetObject",
        "s3:ListBucket",
        "s3:PutObject"
      ],
      "Resource": [
        "arn:aws:s3:::demo-assets",
        "arn:aws:s3:::demo-assets/*"
      ]
    }
  ]
}
```

```bash
# 创建并应用自定义策略
cat > /tmp/demo-policy.json << 'EOF'
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:GetBucketLocation",
        "s3:GetObject",
        "s3:ListBucket",
        "s3:PutObject"
      ],
      "Resource": [
        "arn:aws:s3:::demo-assets",
        "arn:aws:s3:::demo-assets/*"
      ]
    }
  ]
}
EOF

mc admin policy create myminio demo-policy /tmp/demo-policy.json
mc admin policy attach myminio demo-policy --user demo-user
```

### 6.2 操作步骤

```bash
# ===== 第一步：准备测试数据 =====
mkdir -p /tmp/demo-demo/images/posts
mkdir -p /tmp/demo-demo/images/avatars
mkdir -p /tmp/demo-demo/attachments

# 创建一些测试文件
echo "封面图片 1" > /tmp/demo-demo/images/posts/cover-spark.jpg

# 上传所有附件
mc cp --recursive /tmp/demo-demo/attachments/ myminio/demo-assets/attachments/

# ===== 第三步：验证上传结果 =====
# 查看所有文件
mc ls --recursive myminio/demo-assets

# 查看特定目录
mc ls myminio/demo-assets/images/posts/
mc ls myminio/demo-assets/images/avatars/

# ===== 第四步：设置访问策略 =====

# ===== 第五步：查看文件详情 =====
mc stat myminio/demo-assets/images/posts/cover-spark.jpg

# ===== 第六步：下载备份 =====
mc mirror myminio/demo-assets/ /tmp/demo-backup/
```

## 八、面试题

### 题目：如何安全地管理 MinIO 的访问权限？请描述最小权限原则在对象存储中的应用。

**参考答案：**

最小权限原则（Principle of Least Privilege）要求每个用户只拥有完成其工作所需的最小权限。在 MinIO 中，可以通过以下方式实现：

1. **创建专用用户**：不为应用使用 root 账号，而是创建专用用户
   ```bash
   mc admin user add myminio demo-app demo-secret-key
   ```

2. **自定义策略**：精确控制到 Bucket 级别和操作级别
   ```json
   {
     "Statement": [
       {
         "Effect": "Allow",
         "Action": ["s3:GetObject", "s3:PutObject"],
         "Resource": ["arn:aws:s3:::demo-assets/*"]
       }
     ]
   }
   ```

3. **分离读写权限**：前端只读用户、后端读写用户、管理员分开
4. **使用预签名 URL**：临时授权访问，避免暴露长期凭证
5. **定期轮换密钥**：定期更换 Access Key / Secret Key

### 题目：mc mirror 和 mc cp --recursive 有什么区别？

**参考答案：**

| 维度 | mc cp --recursive | mc mirror |
|------|------------------|-----------|
| **行为** | 单向复制 | 双向同步 |
| **删除** | 不会删除目标中多余的文件 | 会删除目标中有但源中没有的文件 |
| **增量** | 每次都复制所有文件 | 只传输有差异的文件 |
| **监控** | 一次性执行 | 支持 --watch 持续监控 |
| **用途** | 初始数据迁移 | 持续数据同步 |

`mc mirror` 更适合持续同步场景，`mc cp --recursive` 更适合一次性复制。
