# 02 - MinIO 安装与配置：从零搭建对象存储服务

## 一、课前回顾

在上一节 [01-对象存储概述.md](01-对象存储概述.md) 中，我们学习了对象存储的基本概念，了解了 Bucket、Object、Key、Metadata 等核心概念，也知道了 MinIO 在大数据生态中的位置。本节我们正式动手，在你的 Mac 上搭建一个 MinIO 服务。

## 二、MinIO 下载与安装

### 2.1 为什么选择 Go 二进制部署？

MinIO 的部署方式有很多种：Docker、Kubernetes、Helm Chart、源码编译等。我们选择 **Go 二进制部署**，原因很简单：

```
┌─────────────────────────────────────────────────────────────────┐
│              选择 Go 二进制部署的原因                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 最简单：下载一个文件，chmod +x 就能运行                      │
│  2. 无依赖：不需要 Docker、不需要 JVM、不需要编译环境            │
│  3. 性能最好：直接运行在宿主机上，没有虚拟化开销                  │
│  4. 方便学习：可以清楚看到端口、数据目录、配置文件                │
│  5. 生产推荐：MinIO 官方推荐二进制部署方式                       │
│                                                                 │
│  其他方式对比：                                                  │
│  - Docker：方便但多了虚拟化层，调试不方便                        │
│  - Kubernetes：适合大规模集群，学习成本高                        │
│  - 源码编译：需要 Go 环境，没必要                               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 下载 MinIO

MinIO 已经更名为 AIStor，但核心功能不变。我们下载 macOS Apple Silicon 版本：

```bash
# 下载 MinIO 二进制文件
cd ~/Programme/claude-study
curl -O https://dl.min.io/aistor/minio/release/darwin-arm64/archive/minio.RELEASE.2026-03-26T21-24-40Z

# 创建安装目录
mkdir -p /opt/minio

# 移动到安装目录并赋权
mv minio.RELEASE.2026-03-26T21-24-40Z /opt/minio/minio
chmod +x /opt/minio/minio
```

```
┌─────────────────────────────────────────────────────────────────┐
│                  MinIO 安装后的目录结构                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  /opt/minio/                                                    │
│  ├── minio              ← 可执行文件（Go 编译的单二进制）         │
│  └── data/              ← 数据目录（稍后创建）                    │
│      └── ...            ← 所有 Bucket 和 Object 的数据           │
│                                                                 │
│  就这么简单！MinIO 是单个二进制文件，                            │
│  不需要像 Hadoop 那样有几十个配置文件。                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3 配置环境变量

将 MinIO 添加到环境变量中，方便在任何目录下使用：

```bash
# 编辑 ~/.zshrc，添加以下内容
export MINIO_HOME=/opt/minio
export MINIO_ROOT_USER=minioadmin
export MINIO_ROOT_PASSWORD=minioadmin123
export PATH=$PATH:$MINIO_HOME

# 使环境变量生效
source ~/.zshrc
```

**环境变量说明：**

| 环境变量 | 值 | 说明 |
|---------|-----|------|
| `MINIO_HOME` | `/opt/minio` | MinIO 安装目录，方便管理 |
| `MINIO_ROOT_USER` | `minioadmin` | 管理员用户名（相当于 AWS 的 Access Key） |
| `MINIO_ROOT_PASSWORD` | `minioadmin123` | 管理员密码（相当于 AWS 的 Secret Key） |
| `PATH` | `$PATH:$MINIO_HOME` | 将 minio 命令加入系统路径 |

> **注意**：`MINIO_ROOT_PASSWORD` 至少需要 8 个字符。

## 三、启动 MinIO

### 3.1 创建数据目录

```bash
mkdir -p /opt/minio/data
```

### 3.2 端口规划

我们的环境中 Hadoop NameNode 占用了 9000 端口，所以 MinIO 的 API 端口改用 9002：

```
┌─────────────────────────────────────────────────────────────────┐
│                    MinIO 端口规划                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  端口 9000 → 已被 Hadoop NameNode 占用                           │
│                                                                 │
│  MinIO 使用两个端口：                                            │
│                                                                 │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │  端口 9002  →  API 端口（S3 协议接口）                    │   │
│  │                                                          │   │
│  │  程序通过这个端口上传下载文件：                             │   │
│  │  - Python SDK (boto3 / minio-py)                         │   │
│  │  - mc 命令行工具                                          │   │
│  │  - Spark / Flink 读写数据                                  │   │
│  │  - Milvus 存储向量索引                                    │   │
│  └──────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │  端口 9001  →  Web 控制台端口                             │   │
│  │                                                          │   │
│  │  浏览器通过这个端口访问管理界面：                          │   │
│  │  - 创建/删除 Bucket                                      │   │
│  │  - 上传/下载文件                                          │   │
│  │  - 查看存储用量                                           │   │
│  │  - 管理用户和策略                                         │   │
│  └──────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.3 启动服务

```bash
# 后台启动 MinIO
nohup minio server /opt/minio/data \
  --address ":9002" \
  --console-address ":9001" \
  > /opt/minio/minio.log 2>&1 &
```

### 3.4 验证启动成功

```bash
# 方法一：检查端口是否在监听
lsof -i :9002   # 应该看到 minio 进程
lsof -i :9001   # 应该看到 minio 进程

# 方法二：查看日志
tail -20 /opt/minio/minio.log
# 你应该能看到类似这样的输出：
# API: http://192.168.1.xxx:9002
# Console: http://192.168.1.xxx:9001
```

### 3.5 访问 Web 控制台

打开浏览器访问 `http://localhost:9001`，使用以下凭据登录：

```
用户名：minioadmin
密码：  minioadmin123
```

### 3.6 停止 MinIO

```bash
# 查找进程
lsof -i :9002 | grep LISTEN

# 停止进程
kill <PID>
```

## 四、单机模式 vs 分布式模式

MinIO 支持两种运行模式，理解它们的区别非常重要：

```
┌─────────────────────────────────────────────────────────────────────┐
│                   单机模式 vs 分布式模式                               │
│                                                                     │
│  单机模式（当前使用）：                                              │
│  ┌───────────────────────────────────────────────────────────┐      │
│  │                                                           │      │
│  │  一台机器 + 一个数据目录                                    │      │
│  │                                                           │      │
│  │  minio server /opt/minio/data                             │      │
│  │                                                           │      │
│  │  优点：简单、适合开发和测试                                  │      │
│  │  缺点：单点故障、容量受限                                   │      │
│  │                                                           │      │
│  └───────────────────────────────────────────────────────────┘      │
│                                                                     │
│  分布式模式（生产环境）：                                            │
│  ┌───────────────────────────────────────────────────────────┐      │
│  │                                                           │      │
│  │  多台机器 + 多个数据目录                                    │      │
│  │                                                           │      │
│  │  minio server http://node{1...4}/data{1...2}              │      │
│  │                                                           │      │
│  │  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐     │      │
│  │  │  Node1  │  │  Node2  │  │  Node3  │  │  Node4  │     │      │
│  │  │ data1   │  │ data1   │  │ data1   │  │ data1   │     │      │
│  │  │ data2   │  │ data2   │  │ data2   │  │ data2   │     │      │
│  │  └─────────┘  └─────────┘  └─────────┘  └─────────┘     │      │
│  │                                                           │      │
│  │  优点：高可用、容量大、纠删码保护                           │      │
│  │  缺点：部署复杂、需要多台机器                               │      │
│  │                                                           │      │
│  └───────────────────────────────────────────────────────────┘      │
│                                                                     │
│  纠删码（Erasure Coding）：                                        │
│  ┌───────────────────────────────────────────────────────────┐      │
│  │                                                           │      │
│  │  将数据分成 N 份，加上 M 份校验，共 N+M 份                 │      │
│  │  只要任意 N 份存活，就能恢复完整数据                       │      │
│  │                                                           │      │
│  │  例如 4+2 模式（默认）：                                   │      │
│  │  原始数据被分成 4 份 + 2 份校验 = 6 份                     │      │
│  │  最多可以坏 2 块磁盘，数据仍然安全                          │      │
│  │                                                           │      │
│  └───────────────────────────────────────────────────────────┘      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**分布式模式的启动命令示例（4 节点，每节点 2 块盘）：**

```bash
# 分布式模式启动（了解即可，当前我们使用单机模式）
minio server http://node1/data1 http://node1/data2 \
             http://node2/data1 http://node2/data2 \
             http://node3/data1 http://node3/data2 \
             http://node4/data1 http://node4/data2
```

> **提示**：分布式模式要求所有节点的 MinIO 版本一致，且数据目录数量相同。

## 五、mc 客户端安装与配置

**mc**（MinIO Client）是 MinIO 提供的命令行工具，功能类似 AWS CLI，是我们管理 MinIO 的主要工具。

### 5.1 安装 mc

```bash
# 下载 mc（macOS Apple Silicon）
curl -O https://dl.min.io/client/mc/release/darwin-arm64/mc
chmod +x mc
sudo mv mc /usr/local/bin/
```

### 5.2 配置连接（alias）

mc 使用 alias（别名）来管理不同的存储服务。我们给本地的 MinIO 起个别名 `myminio`：

```bash
# 配置连接
mc alias set myminio http://localhost:9002 minioadmin minioadmin123

# 验证连接
mc admin info myminio
```

```
┌─────────────────────────────────────────────────────────────────┐
│                    mc alias 的理解                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  mc alias set <别名> <Endpoint> <AccessKey> <SecretKey>          │
│                                                                 │
│  常用别名示例：                                                  │
│                                                                 │
│  mc alias set myminio http://localhost:9002 minioadmin xxx      │
│                 ▲                                                │
│                 │                                                │
│              自定义别名                                          │
│                                                                 │
│  之后所有操作都用这个别名：                                       │
│  mc ls myminio      ← 列出所有 Bucket                           │
│  mc cp file myminio/bucket/key  ← 上传文件                      │
│                                                                 │
│  你可以配置多个 alias，管理不同的存储服务：                        │
│  mc alias set production https://s3.amazonaws.com ak sk         │
│  mc alias set backup     https://oss-cn-hangzhou.aliyuncs.com   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 5.3 验证安装

```bash
# 检查 mc 版本
mc --version

# 列出所有已配置的 alias
mc alias list

# 查看 MinIO 服务信息
mc admin info myminio
```

## 六、数据分析项目：创建 demo-assets Bucket

现在让我们用刚学到的知识，为数据分析项目创建第一个存储桶。

### 6.1 通过 mc 创建 Bucket

```bash
# 创建数据平台资源桶
mc mb myminio/demo-assets

# 创建业务数据桶
mc mb myminio/demo-data

# 列出所有桶，确认创建成功
mc ls myminio
```

### 6.2 通过 Web 控制台创建

```
┌─────────────────────────────────────────────────────────────────┐
│              通过 Web 控制台创建 Bucket                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 浏览器打开 http://localhost:9001                             │
│  2. 使用 minioadmin / minioadmin123 登录                        │
│  3. 点击左侧 "Buckets" 菜单                                     │
│  4. 点击 "Create Bucket" 按钮                                    │
│  5. 输入桶名：demo-assets                                       │
│  6. 点击 "Create Bucket" 确认                                   │
│                                                                 │
│  创建完成后，你可以在 Buckets 列表中看到新创建的桶                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 6.3 上传第一个测试文件

```bash
# 创建一个测试文件
echo "Hello MinIO! This is my first object." > /tmp/test-upload.txt

# 上传到 demo-assets 桶
mc cp /tmp/test-upload.txt myminio/demo-assets/test-upload.txt

# 确认文件已上传
mc ls myminio/demo-assets

# 查看文件详情
mc stat myminio/demo-assets/test-upload.txt
```

## 七、配置文件详解

MinIO 的配置存储在数据目录下的 `.minio.sys/config/` 中：

```
┌─────────────────────────────────────────────────────────────────┐
│                    MinIO 配置文件结构                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  /opt/minio/data/                                               │
│  └── .minio.sys/                                               │
│      └── config/                                               │
│          ├── config.json          ← 主配置文件                   │
│          └── ...                                                │
│                                                                 │
│  主要配置项（通过环境变量设置，不用手动编辑配置文件）：             │
│                                                                 │
│  ┌────────────────────────┬──────────────────┬─────────────┐   │
│  │ 环境变量                │ 默认值            │ 说明        │   │
│  ├────────────────────────┼──────────────────┼─────────────┤   │
│  │ MINIO_ROOT_USER        │ minioadmin       │ 管理员用户名│   │
│  │ MINIO_ROOT_PASSWORD    │ minioadmin       │ 管理员密码  │   │
│  │ MINIO_BROWSER          │ on               │ 启用控制台  │   │
│  │ MINIO_SERVER_URL       │ 自动检测          │ 服务 URL   │   │
│  │ MINIO_DOMAIN           │ (空)             │ 域名        │   │
│  │ MINIO_REGION           │ us-east-1        │ 区域        │   │
│  └────────────────────────┴──────────────────┴─────────────┘   │
│                                                                 │
│  启动参数：                                                      │
│  ┌────────────────────────┬──────────────────┬─────────────┐   │
│  │ 参数                   │ 值               │ 说明        │   │
│  ├────────────────────────┼──────────────────┼─────────────┤   │
│  │ --address              │ :9000            │ API 端口   │   │
│  │ --console-address      │ :9001            │ 控制台端口  │   │
│  │ --quiet                │ (关闭)           │ 安静模式    │   │
│  └────────────────────────┴──────────────────┴─────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
