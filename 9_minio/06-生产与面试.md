### 2.3 systemd 服务配置（生产推荐）

```ini
# /etc/systemd/system/minio.service
[Unit]
Description=MinIO Object Storage
After=network-online.target
Wants=network-online.target

[Service]
User=minio-user
Group=minio-user
Environment="MINIO_ROOT_USER=minioadmin"
Environment="MINIO_ROOT_PASSWORD=minioadmin123"
ExecStart=/opt/minio/minio server /opt/minio/data \
  --address ":9002" \
  --console-address ":9001"
Restart=always
RestartSec=5
LimitNOFILE=65536

[Install]
WantedBy=multi-user.target
```

# 启用并启动服务

## 五、MinIO 在大数据管道中的应用

### 5.1 Spark 读写 MinIO

```python
from pyspark.sql import SparkSession

# 创建 Spark Session，配置 S3a 连接
spark = SparkSession.builder \
    .appName("MinIO-ReadWrite") \
    .config("spark.hadoop.fs.s3a.endpoint", "http://localhost:9002") \
    .config("spark.hadoop.fs.s3a.access.key", "minioadmin") \
    .config("spark.hadoop.fs.s3a.secret.key", "minioadmin123") \
    .config("spark.hadoop.fs.s3a.path.style.access", "true") \
    .config("spark.hadoop.fs.s3a.impl", "org.apache.hadoop.fs.s3a.S3AFileSystem") \
    .getOrCreate()

# ===== 写入 MinIO =====

# 创建测试数据
df = spark.createDataFrame([
    (1, "Spark 入门指南", "2024-03-15", 1500),
    (2, "Flink 实战教程", "2024-03-20", 2300),
    (3, "MinIO 最佳实践", "2024-04-01", 1800),
], ["id", "title", "publish_date", "views"])

# 写入 Parquet 到 MinIO
df.write.mode("overwrite").parquet("s3a://demo-data/analytics/page_views.parquet")

# 写入分区表（按日期分区）
df.write.mode("overwrite").partitionBy("publish_date") \
    .parquet("s3a://demo-data/analytics/page_views_partitioned/")

# ===== 读取 MinIO =====

# 读取 Parquet
df2 = spark.read.parquet("s3a://demo-data/analytics/page_views.parquet")
df2.show()

# 读取分区表（自动分区裁剪）
df3 = spark.read.parquet("s3a://demo-data/analytics/page_views_partitioned/")
df3.filter(df3.publish_date == "2024-03-15").show()

# 写入 CSV
df.select("title", "views").write.mode("overwrite").csv(
    "s3a://demo-data/exports/popular_posts.csv",
    header=True
)

spark.stop()
```

## 七、10 道高频面试题

### 题目 1：什么是对象存储？和文件存储、块存储有什么区别？

**参考答案：**

对象存储是一种基于扁平命名空间的存储方式，将数据以"对象"的形式存储。每个对象包含数据、元数据和唯一标识符（Key）。

三种存储的区别：
- **块存储**：裸数据块，由操作系统管理，低延迟高 IOPS，适合数据库、虚拟机
- **文件存储**：目录树结构，通过路径访问，适合文件共享、通用场景
- **对象存储**：扁平命名空间，HTTP API 访问，无限扩展，适合非结构化数据

### 题目 2：MinIO 的 S3 兼容性意味着什么？

**参考答案：**

S3 兼容意味着 MinIO 实现了 Amazon S3 的 API 接口，所有使用 S3 SDK 的代码无需修改就能连接 MinIO。这包括：
- boto3（Python）、AWS SDK（Java/Go/JS）等客户端库
- Spark 的 S3a 连接器、Flink 的 S3 文件系统
- mc 命令行工具（兼容 AWS CLI 的大部分命令）
- 预签名 URL、分片上传、生命周期管理等高级特性

好处是：避免了厂商锁定，代码可以在 MinIO 和 AWS S3 之间自由迁移。

### 题目 3：纠删码和多副本有什么区别？MinIO 用的是哪种？

**参考答案：**

- **多副本**：完整复制数据 N 份，存储开销 = N 倍。读取快（直接读副本），但存储效率低。HDFS 使用这种方式（默认 3 副本）。
- **纠删码**：将数据分成 N 份，加上 M 份校验，存储开销 = (N+M)/N 倍。读取需要解码，但存储效率高。MinIO 使用这种方式。

举例：4+2 纠删码，6 块盘存 4 份数据，存储开销 1.5 倍，可容忍 2 块盘故障。对比 3 副本，存储开销 3 倍，可容忍 2 块盘故障。纠删码存储效率提升了一倍。

### 题目 4：如何实现大文件的高效上传？

**参考答案：**

1. **分片上传**：将大文件分成多个分片（默认 5MB），每个分片独立上传
2. **并行上传**：多个分片同时上传（minio-py 的 `num_parallel_uploads` 参数）
3. **断点续传**：记录已上传的分片，失败后只重传失败的分片
4. **分片大小选择**：根据网络带宽和文件大小选择，通常 5MB-100MB

```python
client.fput_object(
    bucket_name="bucket",
    object_name="large-file.tar.gz",
    file_path="/tmp/large-file.tar.gz",
    part_size=16*1024*1024,      # 16MB 分片
    num_parallel_uploads=4        # 4 个线程并行上传
)
```

### 题目 5：预签名 URL 的工作原理是什么？

**参考答案：**

预签名 URL 是将访问凭证嵌入 URL 中的机制：

1. 服务端使用 Secret Key 对请求参数（Bucket、Key、过期时间等）进行 HMAC-SHA256 签名
2. 签名结果作为查询参数附加到 URL
3. 客户端拿着 URL 访问 MinIO，MinIO 用同样的算法验证签名
4. 签名有效且未过期则允许访问

优势：不需要暴露 Access Key/Secret Key，可以精确控制过期时间和操作权限。

### 题目 6：MinIO 如何保证数据一致性？

**参考答案：**

MinIO 提供强一致性保证：
- **写入后立即可读**：PUT 操作成功返回后，GET 一定能读到最新数据
- **覆盖后立即可读新版本**：同一 Key 的新版本写入成功后，立即可以读取
- **删除后立即不可见**：DELETE 操作成功后，GET 立即返回 404

这不同于早期 AWS S3 的最终一致性模型。强一致性对大数据处理非常重要，确保 Spark/Flink 读取的数据是一致的。

### 题目 7：如何设计一个图片存储系统？

**参考答案：**

1. **存储方案**：使用 MinIO 对象存储，配合 CDN 加速
2. **Key 设计**：按日期/类型组织，如 `images/{year}/{month}/{uuid}.jpg`
3. **上传流程**：后端生成预签名 URL → 前端直传 MinIO → 回调后端记录
4. **访问控制**：公开图片设为 download 策略，私密图片用预签名 URL
5. **图片处理**：通过 Webhook 事件通知触发缩略图生成、压缩等
6. **生命周期**：临时文件 1 天过期，草稿 7 天过期，历史版本 30 天过期
7. **版本控制**：启用版本控制防止误覆盖和误删除
8. **备份**：定期 mc mirror 到备份存储

### 题目 8：Spark 如何读写 MinIO？需要哪些配置？

**参考答案：**

Spark 通过 Hadoop 的 S3a 连接器访问 MinIO：

```python
spark = SparkSession.builder \
    .config("spark.hadoop.fs.s3a.endpoint", "http://localhost:9002") \
    .config("spark.hadoop.fs.s3a.access.key", "minioadmin") \
    .config("spark.hadoop.fs.s3a.secret.key", "minioadmin123") \
    .config("spark.hadoop.fs.s3a.path.style.access", "true") \
    .config("spark.hadoop.fs.s3a.impl", "org.apache.hadoop.fs.s3a.S3AFileSystem") \
    .getOrCreate()
```

关键配置项：
- `fs.s3a.endpoint`：MinIO 的 API 地址
- `fs.s3a.access.key` / `fs.s3a.secret.key`：认证信息
- `fs.s3a.path.style.access`：必须设为 true（MinIO 使用路径风格访问）
- `fs.s3a.impl`：S3A 文件系统实现类

注意：需要确保 hadoop-aws 依赖包在 classpath 中。

### 题目 9：MinIO 的版本控制和生命周期管理如何配合使用？

**参考答案：**

版本控制会保留所有历史版本，导致存储空间无限增长。生命周期管理可以自动清理：

1. **启用版本控制**：`mc version enable myminio/bucket`
2. **配置非当前版本过期**：历史版本保留 N 天后自动删除
   ```bash
   mc ilm rule add myminio/bucket --noncurrent-expire-days 30
   ```
3. **配置过期删除标记清理**：删除标记 N 天后自动清理
   ```bash
   mc ilm rule add myminio/bucket --expire-delete-marker
   ```
4. **配置当前版本过期**：对象 N 天后自动删除（变成删除标记）

这样既能享受版本控制的安全保障，又能通过生命周期自动清理释放空间。

### 题目 10：在生产环境中，如何监控 MinIO 的健康状态？

**参考答案：**

1. **Prometheus 指标采集**：MinIO 内置 `/minio/v2/metrics/cluster` 端点
2. **Grafana Dashboard**：MinIO 官方提供现成的 Grafana Dashboard
3. **关键监控指标**：
   - 磁盘使用率（`minio_cluster_disk_free_bytes`）
   - 在线磁盘数（`minio_cluster_disk_online_total`）
   - API 请求量（`minio_s3_requests_total`）
   - API 错误率（`minio_s3_requests_errors_total`）
   - 网络流量（`minio_s3_traffic_*`）
4. **告警规则**：
   - 磁盘使用率 > 80%
   - API 错误率 > 1%
   - 磁盘离线
   - 服务不可达
5. **健康检查**：`mc admin info myminio --health`
