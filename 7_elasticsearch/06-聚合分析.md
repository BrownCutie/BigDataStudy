# 06 - 聚合分析：数据的统计分析能力

## 一、聚合概述

Elasticsearch 的聚合框架（Aggregation Framework）功能非常强大，类似于 SQL 中的 `GROUP BY`、`COUNT()`、`SUM()`、`AVG()` 等操作，但能力远超 SQL。

```
┌──────────────────────────────────────────────────────────────┐
│              聚合 vs SQL 对比                                 │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  SQL:                                                        │
│  SELECT category, COUNT(*) as count, AVG(views) as avg_views │
│  FROM posts                                                  │
│  WHERE is_published = true                                   │
│  GROUP BY category                                           │
│  ORDER BY count DESC                                         │
│                                                              │
│  ES 聚合:                                                    │
│  {                                                           │
│    "query": { "term": { "is_published": true } },            │
│    "size": 0,                                                │
│    "aggs": {                                                 │
│      "by_category": {                                        │
│        "terms": { "field": "category" },                     │
│        "aggs": {                                             │
│          "avg_views": { "avg": { "field": "views" } }       │
│        }                                                    │
│      }                                                      │
│    }                                                        │
│  }                                                          │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 二、聚合的两大类

```
┌──────────────────────────────────────────────────────────────┐
│              聚合的两大类                                     │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  1. 桶聚合（Bucket Aggregation）                              │
│     - 类似 SQL 的 GROUP BY                                   │
│     - 将文档按某个条件分成多个"桶"（组）                      │
│     - 每个桶包含一组满足条件的文档                             │
│     - 例：按分类分组、按日期范围分组                           │
│                                                              │
│  2. 指标聚合（Metric Aggregation）                            │
│     - 类似 SQL 的聚合函数                                    │
│     - 对一组文档计算数值指标                                   │
│     - 例：平均值、总和、最大值、最小值                         │
│                                                              │
│  3. 管道聚合（Pipeline Aggregation）                          │
│     - 对其他聚合的结果再做聚合                                │
│     - 例：求移动平均、衍生桶                                  │
│     - 本教程不深入讲解，了解即可                               │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 三、指标聚合（Metric Aggregation）

指标聚合用于计算数值指标。

### 3.1 avg — 平均值

```bash
# 计算所有文章的平均浏览量
curl -X GET "http://localhost:9200/posts/_search?pretty" -H 'Content-Type: application/json' -d '{
  "size": 0,
  "aggs": {
    "avg_views": {
      "avg": {
        "field": "views"
      }
    }
  }
}'
```

返回示例：

```json
{
  "aggregations": {
    "avg_views": {
      "value": 296.5
    }
  }
}
```

> **`"size": 0`** 表示不返回搜索结果，只返回聚合结果。纯统计分析时建议设置。

### 3.2 sum — 总和

```bash
curl -X GET "http://localhost:9200/posts/_search?pretty" -H 'Content-Type: application/json' -d '{
  "size": 0,
  "aggs": {
    "total_views": {
      "sum": {
        "field": "views"
      }
    }
  }
}'
```

### 3.3 min / max — 最小值 / 最大值

```bash
curl -X GET "http://localhost:9200/posts/_search?pretty" -H 'Content-Type: application/json' -d '{
  "size": 0,
  "aggs": {
    "max_views": { "max": { "field": "views" } },
    "min_views": { "min": { "field": "views" } }
  }
}'
```

### 3.4 stats — 统计信息（综合）

```bash
# 一次性返回 count、min、max、avg、sum
curl -X GET "http://localhost:9200/posts/_search?pretty" -H 'Content-Type: application/json' -d '{
  "size": 0,
  "aggs": {
    "views_stats": {
      "stats": {
        "field": "views"
      }
    }
  }
}'
```

返回示例：

```json
{
  "aggregations": {
    "views_stats": {
      "count": 13,
      "min": 100.0,
      "max": 680.0,
      "avg": 296.5,
      "sum": 3854.5
    }
  }
}
```

### 3.5 value_count — 去重计数

```bash
# 统计有多少位不同的作者
curl -X GET "http://localhost:9200/posts/_search?pretty" -H 'Content-Type: application/json' -d '{
  "size": 0,
  "aggs": {
    "author_count": {
      "value_count": {
        "field": "author"
      }
    }
  }
}'
```

### 3.6 cardinality — 基数估算（近似去重）

```bash
# 近似计算不同标签的数量（大数据量时比 value_count 快）
curl -X GET "http://localhost:9200/posts/_search?pretty" -H 'Content-Type: application/json' -d '{
  "size": 0,
  "aggs": {
    "unique_tags": {
      "cardinality": {
        "field": "tags"
      }
    }
  }
}'
```

```
┌──────────────────────────────────────────────────────────────┐
│              value_count vs cardinality                       │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  value_count                                                  │
│  - 精确计数（每个文档都算）                                    │
│  - 不去重                                                     │
│  - 例：3 篇文章各有 2 个标签 → value_count = 6               │
│                                                              │
│  cardinality                                                  │
│  - 基数估算（HyperLogLog 算法）                               │
│  - 近似去重计数                                               │
│  - 例：3 篇文章共有 ["kafka", "spark", "kafka"]              │
│    → cardinality ≈ 2（实际去重后的不同值数）                  │
│  - 精度约 0.5%，大数据量时性能远好于精确去重                  │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 四、桶聚合（Bucket Aggregation）

桶聚合将文档按条件分成多个"桶"。

### 4.1 terms — 按词项分组

```bash
# 按分类统计文章数量
curl -X GET "http://localhost:9200/posts/_search?pretty" -H 'Content-Type: application/json' -d '{
  "size": 0,
  "aggs": {
    "by_category": {
      "terms": {
        "field": "category",
        "size": 10,
        "order": {
          "_count": "desc"
        }
      }
    }
  }
}'
```

返回示例：

```json
{
  "aggregations": {
    "by_category": {
      "buckets": [
        { "key": "大数据", "doc_count": 10 },
        { "key": "运维", "doc_count": 2 },
        { "key": "存储", "doc_count": 1 },
        { "key": "架构", "doc_count": 1 }
      ]
    }
  }
}
```

```
┌──────────────────────────────────────────────────────────────┐
│              terms 聚合的重要参数                              │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  field    → 按哪个字段分组（必须是 keyword 或数值类型）        │
│  size     → 返回前 N 个桶（默认 10）                          │
│  order    → 排序方式                                          │
│            _count → 按文档数排序                               │
│            _key   → 按词项字典序排序                           │
│            也可以按子聚合的值排序                              │
│  min_doc_count → 最小文档数阈值（默认 1）                     │
│                  设为 0 可以返回空桶                          │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 4.2 range — 范围分组

```bash
# 按浏览量范围分组
curl -X GET "http://localhost:9200/posts/_search?pretty" -H 'Content-Type: application/json' -d '{
  "size": 0,
  "aggs": {
    "views_range": {
      "range": {
        "field": "views",
        "ranges": [
          { "key": "低热度", "from": 0, "to": 200 },
          { "key": "中热度", "from": 200, "to": 400 },
          { "key": "高热度", "from": 400 }
        ]
      }
    }
  }
}'
```

### 4.3 date_histogram — 时间直方图

按时间间隔分组，非常适合时间序列分析：

```bash
# 按月统计文章发布数量
curl -X GET "http://localhost:9200/posts/_search?pretty" -H 'Content-Type: application/json' -d '{
  "size": 0,
  "aggs": {
    "posts_per_month": {
      "date_histogram": {
        "field": "created_at",
        "calendar_interval": "month",
        "format": "yyyy-MM"
      }
    }
  }
}'
```

```
┌──────────────────────────────────────────────────────────────┐
│              date_histogram 的时间间隔                         │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  calendar_interval（日历间隔，推荐）：                         │
│  minute、hour、day、week、month、quarter、year                │
│  自动处理闰年、月末等问题                                     │
│                                                              │
│  fixed_interval（固定间隔）：                                 │
│  1ms、1s、1m、1h、1d                                         │
│  固定时间长度，不考虑日历边界                                  │
│                                                              │
│  例：calendar_interval=month                                  │
│  2026-01-01 ~ 2026-01-31  → 1 月                             │
│  2026-02-01 ~ 2026-02-28  → 2 月                             │
│                                                              │
│  例：fixed_interval=30d                                       │
│  2026-01-01 ~ 2026-01-31  → 第 1 桶                          │
│  2026-02-01 ~ 2026-03-03  → 第 2 桶                          │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 4.4 filter — 过滤桶

创建一个只包含满足过滤条件的文档的桶：

```bash
# 统计已发布文章的浏览量
curl -X GET "http://localhost:9200/posts/_search?pretty" -H 'Content-Type: application/json' -d '{
  "size": 0,
  "aggs": {
    "published_posts": {
      "filter": {
        "term": { "is_published": true }
      },
      "aggs": {
        "avg_views": { "avg": { "field": "views" } }
      }
    }
  }
}'
```

### 4.5 filters — 多个过滤桶

```bash
# 同时创建多个过滤桶
curl -X GET "http://localhost:9200/posts/_search?pretty" -H 'Content-Type: application/json' -d '{
  "size": 0,
  "aggs": {
    "posts_by_status": {
      "filters": {
        "filters": {
          "published": { "term": { "is_published": true } },
          "top": { "term": { "is_top": true } },
          "high_views": { "range": { "views": { "gte": 300 } } }
        }
      }
    }
  }
}'
```

## 六、搜索与聚合结合

聚合可以和搜索查询同时使用，先过滤出符合条件的文档，再对过滤后的结果做聚合。

```bash
# 搜索标题包含 "大数据" 的文章，同时统计这些文章的分类分布
curl -X GET "http://localhost:9200/posts/_search?pretty" -H 'Content-Type: application/json' -d '{
  "query": {
    "match": {
      "title": "大数据"
    }
  },
  "size": 5,
  "aggs": {
    "by_category": {
      "terms": {
        "field": "category",
        "size": 10
      }
    },
    "views_stats": {
      "stats": {
        "field": "views"
      }
    }
  }
}'
```

```
┌──────────────────────────────────────────────────────────────┐
│              搜索 + 聚合的执行顺序                             │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  1. 执行 query 查询，获取匹配的文档                           │
│  2. 在匹配的文档上执行聚合分析                                │
│  3. 同时返回搜索结果和聚合结果                                │
│                                                              │
│  这样聚合的结果只反映搜索范围内的数据                         │
│  而不是整个索引的数据                                         │
│                                                              │
│  例：搜索 "大数据"，聚合按分类统计                             │
│  → 只统计标题包含 "大数据" 的文章的分类分布                   │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 八、面试题

### 面试题 1：Elasticsearch 的聚合和 SQL 的 GROUP BY 有什么区别？

**参考答案：**

相似之处：
- 都可以将数据按某个维度分组
- 都可以在分组后计算统计指标（COUNT、SUM、AVG 等）

不同之处：

1. **功能范围**：ES 聚合比 SQL GROUP BY 更强大
   - SQL 只能做分组 + 聚合函数
   - ES 支持嵌套聚合（桶里套桶）、范围分组、时间直方图、地理距离聚合等

2. **搜索结合**：ES 聚合可以和搜索查询同时使用
   - SQL 需要先用 WHERE 过滤，再 GROUP BY
   - ES 的 query 先过滤，再在过滤结果上做聚合

3. **性能**：ES 聚合可以利用分片并行计算
   - SQL 的 GROUP BY 通常在单个节点上执行
   - ES 的聚合在每个分片上并行执行，然后在协调节点合并

4. **实时性**：ES 聚合是近实时的，SQL 是强一致性的

5. **近似算法**：ES 提供了 cardinality（HyperLogLog）、percentiles（TDigest）等近似聚合，在大数据量下性能更好

---

### 面试题 2：terms 聚合返回的文档数不准确怎么办？

**参考答案：**

terms 聚合在大数据量下可能出现文档数不准确的情况，原因如下：

1. **分片问题**：terms 聚合是先在每个分片上独立计算 Top N，然后在协调节点合并。如果某个词项在各分片上都不是 Top N，但在全局是 Top N，就会被遗漏。

2. **shard_size 参数**：可以通过增大 `shard_size` 来提高准确性。
   - `shard_size` 默认等于 `size`
   - 设大一些可以让每个分片返回更多候选桶，减少遗漏概率
   - 但同时增加内存和网络开销

3. **show_term_doc_count_error**：设置 `show_term_doc_count_error: true` 可以查看每个桶的误差范围。

4. **精确方案**：如果需要精确结果，可以用以下方式：
   - 使用 `composite` 聚合（分页式聚合）
   - 使用 `terms` 聚合的 `execution_mode: map`（全量聚合，适合小数据量）

### 面试题 3：如何实现类似 SQL 的 SELECT category, COUNT(*) ... GROUP BY category HAVING COUNT(*) > 5？

**参考答案：**

ES 没有直接的 HAVING 子句，但可以通过 **bucket_selector 管道聚合** 实现：

```json
{
  "size": 0,
  "aggs": {
    "by_category": {
      "terms": { "field": "category", "size": 100 },
      "aggs": {
        "post_count": { "value_count": { "field": "_id" } },
        "having_filter": {
          "bucket_selector": {
            "buckets_path": { "count": "post_count" },
            "script": "params.count > 5"
          }
        }
      }
    }
  }
}
```

`bucket_selector` 会检查每个桶，只保留满足条件的桶（类似 HAVING）。
