## 二、创建访问日志表

### 2.1 完整建表脚本

所有建表 SQL 在 `code/init_tables.sql` 中，我们逐步讲解。

```sql
-- 创建业务数据库
CREATE DATABASE IF NOT EXISTS demo;
USE demo;

-- 访问日志明细表
CREATE TABLE IF NOT EXISTS demo.access_logs (
    log_id     UInt64,                          -- 日志唯一ID
    user_id    UInt32,                          -- 用户ID
    post_id    UInt32,                          -- 文章ID
    action     Enum8(                           -- 行为类型
        'view' = 1, 'click' = 2, 'comment' = 3,
        'like' = 4, 'share' = 5, 'search' = 6
    ),
    ip         String,                          -- IP地址
    city       LowCardinality(String),          -- 城市
    device     LowCardinality(String),          -- 设备类型
    timestamp  DateTime                         -- 访问时间
)
ENGINE = MergeTree()
PARTITION BY toYYYYMMDD(timestamp)
ORDER BY (timestamp, user_id, post_id)
SETTINGS index_granularity = 8192;
```

### 2.2 表结构设计决策

```
┌──────────────────────────────────────────────────────────────────┐
│                    表设计思路                                       │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  写入模式：                                                      │
│  - 批量写入（每 5-10 分钟一批，每批几千到几万条）                  │
│  - 只追加，不修改（日志数据不需要 UPDATE/DELETE）                  │
│  → MergeTree 引擎完美匹配                                        │
│                                                                  │
│  查询模式：                                                      │
│  - 时间范围查询：WHERE timestamp BETWEEN ... AND ...              │
│  - 维度聚合：GROUP BY city / device / action                      │
│  - 排行查询：ORDER BY views DESC LIMIT 10                        │
│  → 排序键 (timestamp, user_id, post_id) 覆盖了大部分查询           │
│                                                                  │
│  数据量预估：                                                    │
│  - 每天 ~10 万条日志                                             │
│  - 每月 ~300 万条                                                │
│  - 每年 ~3600 万条                                               │
│  → 按天分区合理，每天一个分区目录                                  │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 2.3 统计表和物化视图

```sql
-- 每日统计表（物化视图的目标表）
CREATE TABLE IF NOT EXISTS demo.daily_stats (
    date        Date,
    city        LowCardinality(String),
    device      LowCardinality(String),
    pv          UInt64,
    uv          UInt64,
    comment_cnt UInt64,
    like_cnt    UInt64,
    share_cnt   UInt64
)
ENGINE = SummingMergeTree()
PARTITION BY toYYYYMM(date)
ORDER BY (date, city, device);

-- 物化视图：自动从日志表聚合到统计表
CREATE MATERIALIZED VIEW IF NOT EXISTS demo.mv_daily_stats
TO demo.daily_stats
AS
SELECT
    toDate(timestamp)                AS date,
    city,
    device,
    count()                          AS pv,
    count(DISTINCT user_id)          AS uv,
    countIf(action = 'comment')      AS comment_cnt,
    countIf(action = 'like')         AS like_cnt,
    countIf(action = 'share')        AS share_cnt
FROM demo.access_logs
GROUP BY date, city, device;
```

## 四、实时数据报表

### 4.1 每日 PV/UV 趋势

```sql
-- 每日 PV/UV 趋势（从明细表查询）
SELECT
    toDate(timestamp)                    AS date,
    count()                              AS pv,
    count(DISTINCT user_id)              AS uv,
    countIf(action = 'comment')          AS comments,
    countIf(action = 'like')             AS likes,
    countIf(action = 'share')            AS shares
FROM demo.access_logs
GROUP BY date
ORDER BY date;

-- 从物化视图查询（更快）
SELECT
    date,
    sum(pv)                              AS pv,
    sum(uv)                              AS uv,
    sum(comment_cnt)                     AS comments,
    sum(like_cnt)                        AS likes,
    sum(share_cnt)                       AS shares
FROM demo.daily_stats
GROUP BY date
ORDER BY date;
```

### 4.2 文章热度排行

```sql
-- 文章浏览量 TOP 10
SELECT
    post_id,
    countIf(action = 'view')             AS views,
    countIf(action = 'comment')          AS comments,
    countIf(action = 'like')             AS likes,
    countIf(action = 'share')            AS shares
FROM demo.access_logs
WHERE post_id > 0
GROUP BY post_id
ORDER BY views DESC
LIMIT 10;

-- 文章综合热度（加权计算）
SELECT
    post_id,
    countIf(action = 'view')             AS views,
    countIf(action = 'comment') * 5      AS comment_score,  -- 评论权重 5x
    countIf(action = 'like')    * 3      AS like_score,     -- 点赞权重 3x
    countIf(action = 'share')   * 10     AS share_score,    -- 分享权重 10x
    countIf(action = 'view')
        + countIf(action = 'comment') * 5
        + countIf(action = 'like') * 3
        + countIf(action = 'share') * 10 AS total_score     -- 综合得分
FROM demo.access_logs
WHERE post_id > 0
GROUP BY post_id
ORDER BY total_score DESC
LIMIT 10;
```

### 4.3 用户地域分布

```sql
-- 各城市 PV/UV 统计
SELECT
    city,
    count()                              AS pv,
    count(DISTINCT user_id)              AS uv,
    round(count() * 100.0 / sum(count()) OVER (), 2) AS pv_percent
FROM demo.access_logs
GROUP BY city
ORDER BY pv DESC;

-- Top 5 城市详细数据
SELECT
    city,
    count()                              AS pv,
    count(DISTINCT user_id)              AS uv,
    countIf(action = 'comment')          AS comments,
    countIf(action = 'like')             AS likes
FROM demo.access_logs
GROUP BY city
ORDER BY pv DESC
LIMIT 5;
```

### 4.4 设备类型分布

```sql
-- 设备分布统计
SELECT
    device,
    count()                              AS pv,
    count(DISTINCT user_id)              AS uv,
    round(count() * 100.0 / sum(count()) OVER (), 2) AS pv_percent
FROM demo.access_logs
GROUP BY device
ORDER BY pv DESC;

-- 各设备的行为类型分布
SELECT
    device,
    countIf(action = 'view')             AS views,
    countIf(action = 'comment')          AS comments,
    countIf(action = 'like')             AS likes,
    countIf(action = 'share')            AS shares,
    round(countIf(action = 'comment') * 100.0 / count(), 2) AS comment_rate
FROM demo.access_logs
GROUP BY device
ORDER BY views DESC;
```

### 4.5 搜索热词统计

```sql
-- 假设搜索关键词存在 action='search' 的 post_id 字段中
-- 这里用 user_id 模拟搜索关键词的场景
SELECT
    user_id AS search_keyword,  -- 实际场景中替换为搜索关键词字段
    count() AS search_count
FROM demo.access_logs
WHERE action = 'search'
GROUP BY search_keyword
ORDER BY search_count DESC
LIMIT 10;

-- 搜索热词趋势（每天的热词变化）
SELECT
    toDate(timestamp) AS date,
    count()           AS search_count
FROM demo.access_logs
WHERE action = 'search'
GROUP BY date
ORDER BY date;
```

### 4.6 时间趋势深度分析

```sql
-- 按小时统计（发现用户活跃时间段）
SELECT
    toHour(timestamp)                    AS hour,
    count()                              AS pv,
    count(DISTINCT user_id)              AS uv
FROM demo.access_logs
GROUP BY hour
ORDER BY hour;

-- 按星期统计（发现一周中的高峰日）
SELECT
    toDayOfWeek(timestamp)               AS weekday,
    count()                              AS pv,
    count(DISTINCT user_id)              AS uv
FROM demo.access_logs
GROUP BY weekday
ORDER BY weekday;

-- 每日 PV 环比增长
SELECT
    date,
    pv,
    lagInFrame(pv) OVER (ORDER BY date) AS prev_pv,
    if(prev_pv > 0,
       round((pv - prev_pv) * 100.0 / prev_pv, 2),
       0
    ) AS growth_rate
FROM (
    SELECT toDate(timestamp) AS date, count() AS pv
    FROM demo.access_logs
    GROUP BY date
)
ORDER BY date;
```

## 五、物化视图实现实时聚合

### 5.1 创建多个物化视图

```sql
-- 物化视图 1：每日 + 城市 + 设备 维度聚合
-- （前面已创建，这里回顾）
CREATE MATERIALIZED VIEW demo.mv_daily_stats
TO demo.daily_stats
AS SELECT
    toDate(timestamp) AS date, city, device,
    count() AS pv, count(DISTINCT user_id) AS uv,
    countIf(action = 'comment') AS comment_cnt,
    countIf(action = 'like') AS like_cnt,
    countIf(action = 'share') AS share_cnt
FROM demo.access_logs
GROUP BY date, city, device;

-- 物化视图 2：文章热度聚合
CREATE TABLE IF NOT EXISTS demo.post_stats (
    post_id     UInt32,
    date        Date,
    view_cnt    UInt64,
    comment_cnt UInt64,
    like_cnt    UInt64,
    share_cnt   UInt64
)
ENGINE = SummingMergeTree()
PARTITION BY toYYYYMM(date)
ORDER BY (post_id, date);

CREATE MATERIALIZED VIEW demo.mv_post_stats
TO demo.post_stats
AS SELECT
    post_id,
    toDate(timestamp) AS date,
    countIf(action = 'view')    AS view_cnt,
    countIf(action = 'comment') AS comment_cnt,
    countIf(action = 'like')    AS like_cnt,
    countIf(action = 'share')   AS share_cnt
FROM demo.access_logs
WHERE post_id > 0
GROUP BY post_id, date;

-- 物化视图 3：每小时实时统计（更细粒度）
CREATE TABLE IF NOT EXISTS demo.hourly_stats (
    hour        DateTime,
    pv          UInt64,
    uv          UInt64
)
ENGINE = SummingMergeTree()
PARTITION BY toYYYYMM(hour)
ORDER BY hour;

CREATE MATERIALIZED VIEW demo.mv_hourly_stats
TO demo.hourly_stats
AS SELECT
    toStartOfHour(timestamp) AS hour,
    count() AS pv,
    count(DISTINCT user_id) AS uv
FROM demo.access_logs
GROUP BY hour;
```

### 5.2 查询物化视图

```sql
-- 从 daily_stats 查询每日趋势（极快）
SELECT
    date,
    sum(pv)  AS pv,
    sum(uv)  AS uv,
    sum(comment_cnt) AS comments,
    sum(like_cnt)    AS likes
FROM demo.daily_stats
GROUP BY date
ORDER BY date;

-- 从 post_stats 查询文章累计热度
SELECT
    post_id,
    sum(view_cnt)    AS total_views,
    sum(comment_cnt) AS total_comments,
    sum(like_cnt)    AS total_likes
FROM demo.post_stats
GROUP BY post_id
ORDER BY total_views DESC
LIMIT 10;

-- 从 hourly_stats 查询实时流量
SELECT
    hour,
    sum(pv) AS pv,
    sum(uv) AS uv
FROM demo.hourly_stats
WHERE hour >= now() - INTERVAL 24 HOUR
GROUP BY hour
ORDER BY hour;
```

## 六、Kafka 引擎直接消费

ClickHouse 可以直接作为 Kafka 消费者，无需 Flink/Spark 中间层：

### 6.1 Kafka 表引擎

```sql
-- 创建 Kafka 引擎表（作为数据通道）
CREATE TABLE IF NOT EXISTS demo.access_logs_kafka (
    log_id     UInt64,
    user_id    UInt32,
    post_id    UInt32,
    action     String,
    ip         String,
    city       String,
    device     String,
    timestamp  String
)
ENGINE = Kafka()
SETTINGS
    kafka_broker_list = 'localhost:9092',
    kafka_topic_list = 'demo-access-logs',
    kafka_group_name = 'clickhouse-consumer',
    kafka_format = 'JSONEachRow',       -- 消息格式
    kafka_num_consumers = 1,             -- 消费者线程数
    kafka_max_block_size = 1048576;      -- 每次批量拉取的最大字节数

-- 创建物化视图，从 Kafka 表自动消费并写入目标表
CREATE MATERIALIZED VIEW demo.mv_kafka_to_logs
TO demo.access_logs
AS SELECT
    log_id,
    user_id,
    post_id,
    action,
    ip,
    city,
    device,
    toDateTime(timestamp) AS timestamp
FROM demo.access_logs_kafka;
```

### 7.2 完整示例

完整的 Python 脚本在 `code/data_analytics.py`，核心代码如下：

```python
from clickhouse_driver import Client
from datetime import datetime

# 创建连接
client = Client(
    host='localhost',
    port=9100,          # 注意：用你配置的端口
    database='demo',
)

# 测试连接
result = client.execute('SELECT version()')
print(f'ClickHouse 版本: {result[0][0]}')

# 批量插入数据
data = [
    (2001, 1001, 1, 'view',  '192.168.1.1', '北京', 'desktop',
     datetime(2026, 4, 15, 9, 0, 0)),
    (2002, 1002, 2, 'view',  '10.0.0.1',    '上海', 'mobile',
     datetime(2026, 4, 15, 9, 1, 0)),
]
client.execute(
    'INSERT INTO demo.access_logs '
    '(log_id, user_id, post_id, action, ip, city, device, timestamp) '
    'VALUES',
    data
)

# 查询数据
result = client.execute('''
    SELECT city, count() AS pv, count(DISTINCT user_id) AS uv
    FROM demo.access_logs
    GROUP BY city
    ORDER BY pv DESC
''')

for row in result:
    print(f'城市: {row[0]}, PV: {row[1]}, UV: {row[2]}')
```

```python
# 查询并导出为 CSV
result = client.execute('''
    SELECT
        toDate(timestamp) AS date,
        count() AS pv,
        count(DISTINCT user_id) AS uv
    FROM demo.access_logs
    GROUP BY date
    ORDER BY date
''')

with open('daily_report.csv', 'w', newline='') as f:
    writer = csv.writer(f)
    writer.writerow(['date', 'pv', 'uv'])  # 表头
    writer.writerows(result)                # 数据

print('导出完成：daily_report.csv')
```

## 八、面试题

### 面试题 1：ClickHouse 导入数据有哪些方式？

**参考答案：**

ClickHouse 支持多种数据导入方式：

**1. INSERT VALUES**：适合少量数据，直接在 SQL 中写入。

**2. INSERT FROM FILE**：通过 clickhouse-client 导入文件，支持 CSV、TSV、JSONEachRow、Parquet 等格式。

**3. HTTP 接口**：通过 curl 或 HTTP 客户端发送 POST 请求导入数据。

**4. Kafka 引擎**：创建 Kafka 表引擎 + 物化视图，ClickHouse 自动消费 Kafka 消息。

**5. JDBC/ODBC**：通过 clickhouse-jdbc-bridge 连接外部数据库。

**6. HDFS/S3 引擎**：直接读写 HDFS 或 S3 上的文件。

**7. Python/Java 客户端**：通过客户端库批量插入。

**性能建议**：
- 批量插入比单条插入快 100-1000 倍
- 建议每批 1万-10万条
- 避免频繁的小批量插入，会产生大量小 Part 影响性能

### 面试题 2：物化视图的数据来源问题怎么解决？

**参考答案：**

ClickHouse 的物化视图有一个重要限制：它只捕获创建**之后**写入的数据，源表中的历史数据不会自动触发物化视图。

解决方案：

**方案一：手动填充历史数据**

```sql
-- 手动将历史数据聚合后插入目标表
INSERT INTO demo.daily_stats
SELECT
    toDate(timestamp) AS date, city, device,
    count() AS pv, count(DISTINCT user_id) AS uv,
    countIf(action = 'comment') AS comment_cnt,
    countIf(action = 'like') AS like_cnt,
    countIf(action = 'share') AS share_cnt
FROM demo.access_logs
GROUP BY date, city, device;
```

**方案二：先导入数据，再创建物化视图**

```sql
-- 先导入所有历史数据到源表
INSERT INTO access_logs FORMAT CSVWithNames < data.csv;

-- 再创建物化视图，之后的新数据自动聚合
CREATE MATERIALIZED VIEW mv_daily_stats TO daily_stats AS ...;
```

**方案三：使用 POPULATE 关键字（不推荐，已被废弃）**

早期版本支持 `CREATE MATERIALIZED VIEW ... POPULATE`，但这个操作在创建时会扫描全表，可能阻塞写入，且在分布式环境下有数据不一致的风险。
