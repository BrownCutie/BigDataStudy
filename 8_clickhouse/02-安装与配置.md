在 macOS（Apple Silicon）上安装 ClickHouse 有几种方式：

```

│  ┌──────────────────────────────────────────────────────┐       │

│  │ 缺点：版本可能不是最新                                 │       │
│  └──────────────────────────────────────────────────────┘       │
```

## 二、使用 Homebrew 安装

### 2.1 安装

```bash
# 安装 ClickHouse
brew install clickhouse
```

安装完成后，Homebrew 会自动下载并安装以下组件：

```
┌──────────────────────────────────────────────────────────────────┐
│                  ClickHouse 核心组件                               │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  clickhouse-server    服务端程序（接收查询、管理数据）              │
│  clickhouse-client    命令行客户端（连接服务端执行 SQL）            │
│  clickhouse-local     本地模式（不需要服务端，直接处理文件）        │
│  clickhouse-benchmark 性能测试工具                                 │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 2.2 重要提示：端口冲突

ClickHouse 默认使用以下端口：

| 端口 | 协议 | 用途 |
|------|------|------|
| 8123 | HTTP | REST API / HTTP 接口 |
| 9000 | Native TCP | clickhouse-client 连接 |
| 9009 | Native TCP | 集群间通信 |
| 8443 | HTTPS | 加密 HTTP 接口 |

**注意：9000 端口很可能和 Hadoop NameNode 冲突！**

如果你之前安装了 Hadoop（HDFS），NameNode 默认监听 9000 端口。两者不能同时运行。

### 2.3 处理端口冲突

有两种方案：

**方案一：修改 ClickHouse 端口（推荐）**

编辑 ClickHouse 配置文件，把 9000 改成其他端口：

```bash
# 找到配置文件位置
# Homebrew 安装的配置文件通常在：
ls /opt/homebrew/etc/clickhouse-server/
# 或者
ls /usr/local/etc/clickhouse-server/
```

```xml
<!-- config.xml 中找到 <tcp_port> 配置项，修改为 9100 -->
<tcp_port>9100</tcp_port>

<!-- 如果 9009 也冲突，也改掉 -->
<interserver_http_port>9109</interserver_http_port>
```

**方案二：停止 Hadoop**

```bash
# 如果不需要 Hadoop，先停掉它
stop-dfs.sh
```

## 三、核心配置文件说明

ClickHouse 的配置文件通常是 `config.xml`，路径取决于安装方式：

```
┌──────────────────────────────────────────────────────────────────┐
│                    配置文件结构                                    │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  /opt/homebrew/etc/clickhouse-server/                            │
│  ├── config.xml              主配置文件                           │
│  ├── users.xml               用户和权限配置                       │
│  └── config.d/               额外配置（覆盖 config.xml）          │
│      └── custom.xml          自定义配置                           │
│                                                                  │
│  数据目录（默认）：                                               │
│  /opt/homebrew/var/lib/clickhouse/                               │
│  ├── data/                   数据文件                             │
│  ├── metadata/               表元数据（.sql 文件）                │
│  └── tmp/                    临时文件                             │
│                                                                  │
│  日志目录（默认）：                                               │
│  /opt/homebrew/var/log/clickhouse-server/                        │
│  ├── clickhouse-server.err   错误日志                             │
│  └── clickhouse-server.log   运行日志                             │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 3.1 config.xml 核心配置

```xml
<?xml version="1.0"?>
<clickhouse>
    <!-- 日志级别：trace, debug, information, warning, error -->
    <logger>
        <level>information</level>
        <log>/opt/homebrew/var/log/clickhouse-server/clickhouse-server.log</log>
        <errorlog>/opt/homebrew/var/log/clickhouse-server/clickhouse-server.err</errorlog>
        <size>100M</size>  <!-- 单个日志文件最大大小 -->
        <count>3</count>    <!-- 保留的日志文件数量 -->
    </logger>

    <!-- HTTP 端口（8123） -->
    <http_port>8123</http_port>

    <!-- Native TCP 端口（默认 9000，建议改为 9100 避免冲突） -->
    <tcp_port>9100</tcp_port>

    <!-- 集群间通信端口 -->
    <interserver_http_port>9109</interserver_http_port>

    <!-- 数据存储目录 -->
    <path>/opt/homebrew/var/lib/clickhouse/</path>

    <!--临时文件目录 -->
    <tmp_path>/opt/homebrew/var/lib/clickhouse/tmp/</tmp_path>

    <!-- 用户配置文件（独立文件，方便管理） -->
    <users_config>users.xml</users_config>

    <!-- 最大连接数 -->
    <max_connections>4096</max_connections>

    <!-- 最大并发查询数 -->
    <max_concurrent_queries>100</max_concurrent_queries>

    <!-- 内存限制（默认使用系统可用内存） -->
    <max_server_memory_usage>0</max_server_memory_usage>
</clickhouse>
```

### 3.2 users.xml 用户配置

```xml
<?xml version="1.0"?>
<clickhouse>
    <users>
        <!-- 默认用户（仅限本机访问） -->
        <default>
            <password></password>  <!-- 空密码 -->
            <networks>
                <ip>::1</ip>        <!-- IPv6 localhost -->
                <ip>127.0.0.1</ip>  <!-- IPv4 localhost -->
            </networks>
            <profile>default</profile>
            <quota>default</quota>
        </default>

        <!-- 自定义用户（带密码） -->
        <demo_admin>
            <password_sha256_hex>加密后的密码</password_sha256_hex>
            <networks>
                <ip>::/0</ip>       <!-- 允许所有 IP -->
            </networks>
            <profile>default</profile>
            <quota>default</quota>
            <access_management>1</access_management>  <!-- 管理权限 -->
        </demo_admin>
    </users>

    <!-- 配置文件 -->
    <profiles>
        <default>
            <max_memory_usage>10000000000</max_memory_usage>  <!-- 10GB -->
        </default>
    </profiles>

    <!-- 配额 -->
    <quotas>
        <default>
            <interval>
                <duration>3600</duration>
                <queries>0</queries>       <!-- 0 表示不限制 -->
                <errors>0</errors>
            </interval>
        </default>
    </quotas>
</clickhouse>
```

## 四、启动与停止

### 4.1 使用 Homebrew 服务管理

```bash
# 启动 ClickHouse 服务
brew services start clickhouse

# 停止 ClickHouse 服务
brew services stop clickhouse

# 重启 ClickHouse 服务
brew services restart clickhouse

# 查看服务状态
brew services info clickhouse
```

### 4.2 手动启动（调试用）

```bash
# 前台启动（能看到日志输出，适合调试）
clickhouse-server --config-file=/opt/homebrew/etc/clickhouse-server/config.xml

# 后台启动
clickhouse-server --daemon
```

### 4.3 验证启动

```bash
# 检查进程是否在运行
ps aux | grep clickhouse

# 检查端口是否在监听
# HTTP 端口
curl http://localhost:8123/ping
# 输出：Ok.（表示服务正常）

# TCP 端口（注意用你配置的端口）
clickhouse-client --port 9100 --query "SELECT 'Hello ClickHouse!'"
```

## 五、clickhouse-client 基本使用

### 5.1 连接服务端

```bash
# 使用默认配置连接（本机、默认用户）
clickhouse-client --port 9100

# 指定用户和密码连接
clickhouse-client --port 9100 --user demo_admin --password your_password

# 连接后进入交互式终端
ClickHouse client version 24.3.x.x
Connecting to localhost:9100 as user default.
Connected to ClickHouse server version 24.3.x.

clickhouse-client :) SELECT 1
SELECT 1

┌─1─┐
│ 1 │
└───┘
1 rows in set. Elapsed: 0.001 sec.
```

### 5.2 常用命令

```bash
# 执行单条 SQL（非交互模式）
clickhouse-client --port 9100 --query "SELECT version()"

# 执行 SQL 文件
clickhouse-client --port 9100 --multiquery < init_tables.sql

# 指定数据库
clickhouse-client --port 9100 --database demo

# 格式化输出
clickhouse-client --port 9100 --query "SELECT 1" --format Pretty
clickhouse-client --port 9100 --query "SELECT 1" --format JSONEachRow

# 导出查询结果到文件
clickhouse-client --port 9100 --query "SELECT * FROM demo.access_logs FORMAT CSV" > result.csv
```

### 5.3 HTTP 接口使用

ClickHouse 的 8123 端口提供 HTTP 接口，可以直接通过 REST 方式查询：

```bash
# 简单查询
curl "http://localhost:8123/?query=SELECT%201"

# 带参数查询
curl "http://localhost:8123/?query=SELECT%20%27Hello%27"

# 指定数据库
curl "http://localhost:8123/?database=demo&query=SELECT%20count()%20FROM%20access_logs"

# POST 请求（适合复杂查询）
curl "http://localhost:8123/" -d "SELECT count() FROM demo.access_logs"
```

### 7.2 创建一张简单的测试表

```sql
-- 创建一张简单的访问日志测试表
CREATE TABLE IF NOT EXISTS demo.test_logs
(
    id         UInt64,
    user_id    UInt32,
    action     String,
    timestamp  DateTime
)
ENGINE = MergeTree()
ORDER BY (timestamp, id);

-- 插入一些测试数据
INSERT INTO demo.test_logs VALUES
    (1, 1001, 'view',  '2026-04-01 08:30:00'),
    (2, 1002, 'view',  '2026-04-01 08:31:00'),
    (3, 1001, 'like',  '2026-04-01 08:32:00'),
    (4, 1003, 'view',  '2026-04-01 08:33:00'),
    (5, 1002, 'comment', '2026-04-01 08:34:00');

-- 查询
SELECT * FROM demo.test_logs ORDER BY timestamp;

-- 聚合查询
SELECT
    action,
    count()   AS cnt,
    count(DISTINCT user_id) AS unique_users
FROM demo.test_logs
GROUP BY action
ORDER BY cnt DESC;
```

### 7.3 完整的建表脚本

详细的建表脚本见 `code/init_tables.sql`，包含：

- `access_logs`：访问日志明细表（MergeTree）
- `daily_stats`：每日统计表（SummingMergeTree）
- `post_stats`：文章热度表（SummingMergeTree）
- `demo_daily_summary`：数据平台汇总表（ReplacingMergeTree）
- `mv_daily_stats`：每日统计物化视图
- `mv_post_stats`：文章热度物化视图

执行建表脚本：

## 八、常见问题

### Q1：启动报错 "Address already in use"

```
┌──────────────────────────────────────────────────────────────────┐
│  错误：Listening for TCP connections on 0.0.0.0:9000 failed:     │
│        Address already in use                                    │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  原因：9000 端口被其他程序占用（通常是 Hadoop NameNode）           │
│                                                                  │
│  解决：                                                          │
│  1. 查看谁占用了 9000 端口：                                      │
│     lsof -i :9000                                               │
│                                                                  │
│  2. 修改 ClickHouse 的 tcp_port 为 9100（见 3.1 节）             │
│     或停止 Hadoop：stop-dfs.sh                                  │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### Q2：Homebrew 安装后找不到 clickhouse-client

```bash
# 确保 Homebrew 的 bin 目录在 PATH 中
echo $PATH

# 如果没有，添加到 ~/.zshrc
export PATH="/opt/homebrew/bin:$PATH"

# 重新加载
source ~/.zshrc
```

### Q3：权限不足

```bash
# 确保数据目录和日志目录有正确的权限
sudo chown -R $(whoami) /opt/homebrew/var/lib/clickhouse/
sudo chown -R $(whoami) /opt/homebrew/var/log/clickhouse-server/
```

### Q4：Apple Silicon 兼容性

ClickHouse 从 21.3 版本开始原生支持 ARM64（Apple Silicon），无需 Rosetta 转译。如果你通过 Homebrew 安装，它会自动下载 ARM64 版本。

## 九、面试题

### 面试题 1：ClickHouse 有哪些端口？分别用于什么？

**参考答案：**

ClickHouse 默认使用以下端口：

- **8123**（HTTP）：REST API 接口，可以通过 curl 或任何 HTTP 客户端发送查询。适合程序集成和语言无关的访问。
- **9000**（Native TCP）：ClickHouse 自定义二进制协议，clickhouse-client 使用此端口连接。性能比 HTTP 更高，适合高频查询。
- **9009**（Interserver HTTP）：集群节点间通信使用，用于数据副本同步和分布式查询。
- **8443**（HTTPS）：加密的 HTTP 接口，用于安全场景。
- **9090**（Prometheus）：可选的 Prometheus 指标导出端口。

注意：9000 端口和 Hadoop NameNode 默认端口冲突，在生产环境中建议修改 ClickHouse 的 TCP 端口。

---

### 面试题 2：ClickHouse 的配置文件有哪些？如何管理？

**参考答案：**

ClickHouse 主要有两个配置文件：

**config.xml**（主配置）：负责服务端全局配置，包括端口、日志路径、数据目录、内存限制等。路径通常在 `/etc/clickhouse-server/config.xml`。

**users.xml**（用户配置）：负责用户账号、密码、权限、配额等。独立于主配置文件，方便权限管理。

**配置覆盖机制**：ClickHouse 支持 `config.d/` 目录下的额外配置文件，这些文件中的配置项会覆盖 `config.xml` 中的同名配置。这样不需要修改主配置文件就能调整参数，方便升级。

**推荐实践**：自定义配置放在 `config.d/` 和 `users.d/` 目录下，不要直接修改 `config.xml` 和 `users.xml`。
