# 02 - 环境搭建详解：从零开始搭建 Hadoop 3.4.2

## 一、搭建前的准备工作

在动手之前，我们先了解整体流程：

```
┌────────────────────────────────────────────────────────────────┐
│                    Hadoop 环境搭建流程                          │
│                                                                │
│  ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   │
│  │ 安装 JDK │ → │ 配置 SSH │ → │ 安装     │ → │ 配置     │   │
│  │          │   │ 免密登录  │   │ Hadoop   │   │ Hadoop   │   │
│  └──────────┘   └──────────┘   └──────────┘   └──────────┘   │
│                                                     │          │
│                  ┌──────────┐   ┌──────────┐        │          │
│                  │ 格式化   │ → │ 启动     │ ←──────┘          │
│                  │ NameNode │   │ HDFS     │                    │
│                  └──────────┘   └──────────┘                    │
│                                                       │          │
│                  ┌──────────┐   ┌──────────┐        │          │
│                  │ 验证     │ ← │ 访问     │ ←──────┘          │
│                  │ 安装成功  │   │ Web UI   │                    │
│                  └──────────┘   └──────────┘                    │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

### 环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | macOS / CentOS / Ubuntu |
| JDK | 17 或 21（Hadoop 3.4.2 要求 JDK 8+，推荐 JDK 21） |
| SSH | OpenSSH（Hadoop 进程间通信需要） |
| 内存 | 至少 4GB（推荐 8GB+） |
| 磁盘 | 至少 20GB 可用空间 |

本文档以 **macOS 环境** 为例，Linux 系统的操作基本类似，会特别说明差异。

## 二、安装 JDK 21

Hadoop 是用 Java 编写的，所以必须先安装 JDK。

### 2.1 使用 Homebrew 安装（macOS）

```bash
# 安装 JDK 21
brew install openjdk@21

# 配置软链接
sudo ln -sfn /opt/homebrew/opt/openjdk@21/libexec/openjdk.jdk /Library/Java/JavaVirtualMachines/openjdk-21.jdk
```

### 2.2 手动安装（Linux）

```bash
# 下载 JDK 21（以 CentOS 为例）
wget https://download.oracle.com/java/21/latest/jdk-21_linux-x64_bin.tar.gz

# 解压
tar -zxvf jdk-21_linux-x64_bin.tar.gz -C /usr/local/

# 重命名
mv /usr/local/jdk-21* /usr/local/java
```

### 2.3 配置环境变量

编辑 `~/.zshrc`（macOS）或 `~/.bashrc`（Linux）：

```bash
# >>> Java 环境变量 >>>
export JAVA_HOME=/Library/Java/JavaVirtualMachines/openjdk-21.jdk/Contents/Home
# Linux 用户使用: export JAVA_HOME=/usr/local/java

export PATH=$JAVA_HOME/bin:$PATH
# <<< Java 环境变量 <<<
```

使配置生效：

```bash
source ~/.zshrc    # macOS
# source ~/.bashrc  # Linux
```

验证安装：

```bash
java -version
# 预期输出：
# openjdk version "21.0.x" 2024-xx-xx
# OpenJDK Runtime Environment (build 21.0.x+x)
# OpenJDK 64-Bit Server VM (build 21.0.x+x, mixed mode, sharing)
```

## 三、配置 SSH 免密登录

### 3.1 为什么 Hadoop 需要 SSH？

Hadoop 中的 NameNode 需要通过 SSH 协议来**远程启动和管理 DataNode 进程**。即使是单机伪分布式模式，Hadoop 也需要通过 SSH 连接到 localhost 来启动各个服务。

```
┌──────────────────────────────────────────────────────────┐
│              Hadoop 使用 SSH 的原理                        │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  NameNode 启动时：                                        │
│                                                          │
│  NameNode                                                │
│  ┌──────────┐     SSH      ┌──────────────┐             │
│  │          │ ──────────→ │ DataNode     │             │
│  │ hadoop-  │             │ (远程机器上)   │             │
│  │ daemons  │             │              │             │
│  │ start-   │             │ ssh: 启动     │             │
│  │ dfs.sh   │             │ DataNode     │             │
│  │          │             │ 进程         │             │
│  └──────────┘             └──────────────┘             │
│                                                          │
│  如果没有配置免密登录，每次启动都会要求输入密码，            │
│  在集群环境下这是不可接受的。                               │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 3.2 配置步骤

```bash
# 步骤 1：检查是否已安装 SSH
which ssh
which ssh-keygen

# 步骤 2：如果未安装（macOS 默认已安装）
# macOS: 系统设置 → 通用 → 共享 → 远程登录（开启）
# Ubuntu: sudo apt install openssh-server openssh-client
# CentOS: sudo yum install openssh-server openssh-client

# 步骤 3：生成 SSH 密钥对
ssh-keygen -t rsa -b 4096 -C "hadoop@localhost"
# 一路回车，使用默认路径和空密码

# 步骤 4：将公钥追加到授权文件
cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys

# 步骤 5：设置正确的权限（非常重要！）
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys

# 步骤 6：验证免密登录
ssh localhost
# 第一次连接会提示确认指纹，输入 yes
# 之后应该能直接登录，不需要输入密码
# 输入 exit 退出
```

### 3.3 如果 SSH 连接失败

```bash
# 检查 SSH 服务是否运行
# macOS:
sudo systemsetup -getremotelogin
# 应显示: Remote Login: On

# Linux:
sudo systemctl status sshd

# 如果还是不行，检查权限
ls -la ~/.ssh/
# 应该看到:
# drwx------  .ssh/
# -rw-------  authorized_keys
# -rw-------  id_rsa
# -rw-r--r--  id_rsa.pub
```

## 五、核心配置文件详解

配置文件位于 `$HADOOP_HOME/etc/hadoop/` 目录下。

### 5.1 hadoop-env.sh — Java 路径配置

这是最基础的配置，告诉 Hadoop 去哪里找 Java。

```bash
# 编辑 hadoop-env.sh
vim $HADOOP_HOME/etc/hadoop/hadoop-env.sh
```

找到 `JAVA_HOME` 那一行，修改为：

```bash
# 明确指定 JAVA_HOME（推荐，避免使用自动检测）
export JAVA_HOME=/Library/Java/JavaVirtualMachines/openjdk-21.jdk/Contents/Home
# Linux: export JAVA_HOME=/usr/local/java

# 可选：配置 Hadoop 进程使用的用户
# export HADOOP_OS_TYPE=${HADOOP_OS_TYPE:-$(uname -s)}

# 可选：配置 Hadoop 守护进程的堆内存大小
# export HDFS_NAMENODE_OPTS="-Xms1024m -Xmx1024m"
# export HDFS_DATANODE_OPTS="-Xms512m -Xmx512m"
```

### 5.2 core-site.xml — Hadoop 核心配置

这个文件是 Hadoop 的核心配置，定义了 HDFS 的默认名称和基础 I/O 设置。

```xml
<?xml version="1.0" encoding="UTF-8"?>
<?xml-stylesheet type="text/xsl" href="configuration.xsl"?>
<configuration>

    <!--
        fs.defaultFS
        ─────────────
        定义 HDFS 的默认访问地址。
        格式: hdfs://<主机名>:<端口>
        端口 8020 是 Hadoop 3.x 的默认 NameNode RPC 端口。

        当你执行 hdfs dfs -ls / 时，如果不指定完整路径，
        就会使用这个默认地址。
    -->
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://localhost:8020</value>
    </property>

    <!--
        hadoop.tmp.dir
        ─────────────
        Hadoop 的基础临时目录。
        NameNode 和 DataNode 的数据都会存储在这个目录下。

        默认值是 /tmp/hadoop-${user.name}
        但 /tmp 在系统重启后会被清空！所以一定要改。

        建议修改为持久化目录。
    -->
    <property>
        <name>hadoop.tmp.dir</name>
        <value>/Users/你的用户名/Programme/hadoop/tmp</value>
    </property>

    <!--
        fs.trash.interval
        ───────────────
        回收站保留时间（分钟）。
        默认 0，表示不启用回收站（删除即永久删除）。

        设为 1440 表示保留 1 天（24 小时）。
        开启后，hdfs dfs -rm 不会立即删除文件，
        而是移到回收站，超过时间后自动清理。
    -->
    <property>
        <name>fs.trash.interval</name>
        <value>1440</value>
    </property>

    <!--
        io.file.buffer.size
        ───────────────────
        I/O 缓冲区大小，默认 4096（4KB）。
        建议改为 131072（128KB），提高大文件读写性能。
    -->
    <property>
        <name>io.file.buffer.size</name>
        <value>131072</value>
    </property>

</configuration>
```

### 5.3 hdfs-site.xml — HDFS 核心配置

```xml
<?xml version="1.0" encoding="UTF-8"?>
<?xml-stylesheet type="text/xsl" href="configuration.xsl"?>
<configuration>

    <!--
        dfs.replication
        ───────────────
        HDFS 数据块的副本数量。
        默认值: 3

        伪分布式模式设为 1 即可（只有一台机器）。
        生产环境建议设为 3，保证数据可靠性。
    -->
    <property>
        <name>dfs.replication</name>
        <value>1</value>
    </property>

    <!--
        dfs.namenode.name.dir
        ─────────────────────
        NameNode 存储 fsimage 和 edits 的目录。
        可以配置多个目录，用逗号分隔（实现元数据冗余）。

        重要：这是 NameNode 的核心数据目录，
        格式化 HDFS 后数据就存在这里。
    -->
    <property>
        <name>dfs.namenode.name.dir</name>
        <value>file://${hadoop.tmp.dir}/dfs/name</value>
    </property>

    <!--
        dfs.datanode.data.dir
        ─────────────────────
        DataNode 存储数据块的目录。
        可以配置多个目录，HDFS 会在每个目录下轮流存储数据块。

        生产环境中，建议每个磁盘配置一个目录，
        充分利用多磁盘的 I/O 带宽。
    -->
    <property>
        <name>dfs.datanode.data.dir</name>
        <value>file://${hadoop.tmp.dir}/dfs/data</value>
    </property>

    <!--
        dfs.namenode.http-address
        ───────────────────────
        NameNode Web UI 的地址和端口。
        浏览器访问这个地址可以查看 HDFS 的状态。

        注意：Hadoop 3.x 默认端口是 9870（2.x 是 50070）。
    -->
    <property>
        <name>dfs.namenode.http-address</name>
        <value>0.0.0.0:9870</value>
    </property>

    <!--
        dfs.namenode.secondary.http-address
        ───────────────────────────────────
        SecondaryNameNode Web UI 的地址和端口。
    -->
    <property>
        <name>dfs.namenode.secondary.http-address</name>
        <value>0.0.0.0:9868</value>
    </property>

    <!--
        dfs.blocksize
        ──────────────
        HDFS 数据块大小。
        默认值: 128MB (134217728)

        如果你的文件主要是超大文件（TB 级），
        可以增大到 256MB (268435456)。

        格式: 直接写数字，单位是字节。
    -->
    <property>
        <name>dfs.blocksize</name>
        <value>134217728</value>
    </property>

    <!--
        dfs.permissions.enabled
        ───────────────────────
        是否启用 HDFS 权限检查。
        学习阶段可以设为 false，避免权限问题。
        生产环境务必设为 true。
    -->
    <property>
        <name>dfs.permissions.enabled</name>
        <value>false</value>
    </property>

    <!--
        dfs.namenode.handler.count
        ─────────────────────────
        NameNode 处理 RPC 请求的线程数。
        默认值: 10

        集群规模较大时，可以适当增加。
        一般设为 集群 DataNode 数量的 20% 左右。
    -->
    <property>
        <name>dfs.namenode.handler.count</name>
        <value>10</value>
    </property>

</configuration>
```

## 六、格式化 NameNode

### 6.1 什么是格式化？

格式化 NameNode 是**初始化 HDFS 文件系统**的过程。它会：

```
┌──────────────────────────────────────────────────────────────┐
│                NameNode 格式化过程                            │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  执行 hdfs namenode -format 时：                             │
│                                                              │
│  1. 创建 dfs/name/ 目录                                     │
│                                                              │
│  2. 初始化 fsimage 文件（空的文件系统镜像）                   │
│     包含: 根目录 "/" 的元信息                                │
│                                                              │
│  3. 创建 edits_0000000000000000001 文件（空的编辑日志）      │
│                                                              │
│  4. 生成 clusterID（集群唯一标识）                           │
│     格式: CID-xxxx-xxxx-xxxx-xxxx                            │
│                                                              │
│  5. 生成 dataTransferVersion、storageType 等元信息           │
│                                                              │
│  ┌────────────────────────────────────────┐                 │
│  │  格式化后的目录结构:                    │                 │
│  │                                        │                 │
│  │  dfs/name/current/                     │                 │
│  │  ├── VERSION                           │                 │
│  │  │   └── clusterID=CID-xxx-xxx        │                 │
│  │  ├── fsimage_0000000000000000000       │                 │
│  │  │   └── (空的文件系统镜像)            │                 │
│  │  ├── fsimage_0000000000000000000.md5   │                 │
│  │  │   └── (校验和)                      │                 │
│  │  └── seen_txid                         │                 │
│  │      └── (事务ID: 0)                   │                 │
│  │                                        │                 │
│  └────────────────────────────────────────┘                 │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 6.2 格式化命令

```bash
# 格式化 NameNode
hdfs namenode -format

# 预期输出（关键部分）:
# ...
# Storage directory /Users/xxx/Programme/hadoop/tmp/dfs/name has been successfully formatted.
# ...
```

### 6.3 注意事项

```
┌──────────────────────────────────────────────────────────┐
│              ★ 格式化 NameNode 的警告 ★                   │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  ⚠ 格式化会清除 HDFS 中的所有数据！                       │
│                                                          │
│  ⚠ 格式化只能执行一次！重复格式化会导致:                   │
│                                                          │
│    1. 新的 clusterID 与 DataNode 的 clusterID 不一致     │
│    2. DataNode 启动后无法连接 NameNode                   │
│    3. 数据块报告失败，DataNode 会被踢出集群               │
│                                                          │
│  如果需要重新格式化，必须:                                 │
│    1. 停止 HDFS: stop-dfs.sh                              │
│    2. 删除 NameNode 数据目录:                              │
│       rm -rf ${hadoop.tmp.dir}/dfs/name                  │
│    3. 删除 DataNode 数据目录:                              │
│       rm -rf ${hadoop.tmp.dir}/dfs/data                  │
│    4. 然后重新执行 hdfs namenode -format                  │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

## 七、启动和停止 HDFS

### 7.1 启动 HDFS

```bash
# 启动 HDFS（会启动 NameNode、DataNode、SecondaryNameNode）
start-dfs.sh

# 预期输出:
# Starting namenodes on [localhost]
# Starting datanodes
# Starting secondary namenodes [你的主机名]
```

### 7.2 启动过程详解

```
┌────────────────────────────────────────────────────────────────┐
│                start-dfs.sh 启动流程                           │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  执行 start-dfs.sh 后，按顺序执行以下操作:                     │
│                                                                │
│  1. 读取 workers 文件，获取 DataNode 列表                     │
│                                                                │
│  2. 通过 SSH 连接 localhost，执行:                             │
│     hadoop-daemon.sh start namenode                           │
│     → 启动 NameNode 进程                                      │
│                                                                │
│  3. 通过 SSH 连接每个 worker，执行:                            │
│     hadoop-daemon.sh start datanode                           │
│     → 启动 DataNode 进程                                      │
│                                                                │
│  4. 通过 SSH 连接 localhost，执行:                             │
│     hadoop-daemon.sh start secondarynamenode                  │
│     → 启动 SecondaryNameNode 进程                             │
│                                                                │
│  每个节点上运行的 Java 进程:                                   │
│  ┌────────────────────────────────────────────────────────┐   │
│  │  localhost                                            │   │
│  │  ├── NameNode        (端口: 8020 RPC, 9870 Web)       │   │
│  │  ├── DataNode        (端口: 9866 数据传输, 9864 Web)  │   │
│  │  └── SecondaryNameNode (端口: 9868 Web)               │   │
```

### 7.3 验证进程

```bash
# 方法 1：使用 jps 命令查看 Java 进程
jps
# 预期输出:
# 12345 NameNode
# 12456 DataNode
# 12567 SecondaryNameNode
# 12678 Jps

# 如果看到这三个进程，说明启动成功！
```

### 7.4 停止 HDFS

```bash
# 停止 HDFS
stop-dfs.sh

# 预期输出:
# Stopping namenodes on [localhost]
# Stopping datanodes
# Stopping secondary namenodes [你的主机名]

# 验证进程已停止
jps
# 应该只剩下 Jps 一个进程
```
