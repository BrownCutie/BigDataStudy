# 07 - YARN 与 MapReduce 配置详解

## 一、前言

在前面的文档中，我们已经配置了基本的 `core-site.xml`、`hdfs-site.xml` 和 `yarn-site.xml`。本篇文档将深入讲解 `mapred-site.xml` 的每个配置项，以及 YARN 相关的进阶配置，帮助你理解如何合理分配集群资源。

## 二、mapred-site.xml 配置详解

`mapred-site.xml` 是 MapReduce 框架的专用配置文件。在 Hadoop 3.x 中，MapReduce 必须运行在 YARN 上，所以这个文件必须正确配置。

```xml
<?xml version="1.0" encoding="UTF-8"?>
<?xml-stylesheet type="text/xsl" href="configuration.xsl"?>
<configuration>

    <!--
        mapreduce.framework.name
        ─────────────────────────
        指定 MapReduce 运行在哪个框架上。

        可选值:
          local      - 本地模式（单机调试，不走 YARN）
          classic    - 经典模式（Hadoop 1.x 的 JobTracker/TaskTracker，已废弃）
          yarn       - YARN 模式（Hadoop 2.x/3.x 推荐使用）

        这个配置决定了 MapReduce 作业的运行方式:
          local → 在本地 JVM 中运行，适合开发调试
          yarn  → 提交到 YARN 集群运行，适合生产环境

        默认值: local
        推荐值: yarn（分布式环境）
    -->
    <property>
        <name>mapreduce.framework.name</name>
        <value>yarn</value>
    </property>

    <!--
        mapreduce.application.classpath
        ───────────────────────────────
        MapReduce 应用运行时的 classpath。

        当你提交 MapReduce 作业到 YARN 时，NodeManager 需要知道
        Hadoop 的 jar 包在哪里。如果不配置这个，运行时可能会报
        ClassNotFoundException。

        可以通过 hadoop classpath 命令获取正确的值。
    -->
    <property>
        <name>mapreduce.application.classpath</name>
        <value>
            $HADOOP_MAPRED_HOME/share/hadoop/mapreduce/*,
            $HADOOP_MAPRED_HOME/share/hadoop/mapreduce/lib/*,
            $HADOOP_COMMON_HOME/share/hadoop/common/*,
            $HADOOP_COMMON_HOME/share/hadoop/common/lib/*,
            $HADOOP_HDFS_HOME/share/hadoop/hdfs/*,
            $HADOOP_HDFS_HOME/share/hadoop/hdfs/lib/*,
            $HADOOP_YARN_HOME/share/hadoop/yarn/*,
            $HADOOP_YARN_HOME/share/hadoop/yarn/lib/*
        </value>
    </property>

    <!--
        mapreduce.map.memory.mb
        ──────────────────────────
        每个 Map Task 的内存上限（MB）。

        这个值决定了 Map Task 运行的 Container 分配多少内存。
        YARN 的 NM 会为这个 Container 分配相应大小的内存。

        默认值: 1024 (1GB)

        建议值:
          - 小数据量: 1024
          - 中等数据量: 2048-4096
          - 大数据量: 4096-8192

        注意: 这个值不能超过 yarn.scheduler.maximum-allocation-mb
    -->
    <property>
        <name>mapreduce.map.memory.mb</name>
        <value>1024</value>
    </property>

    <!--
        mapreduce.reduce.memory.mb
        ───────────────────────────
        每个 Reduce Task 的内存上限（MB）。

        Reduce 通常比 Map 需要更多内存，因为 Reduce 需要:
        1. 保存 Shuffle 过来的大量数据
        2. 在内存中排序和合并数据

        默认值: 1024 (1GB)
        推荐值: 通常是 Map 内存的 2 倍
    -->
    <property>
        <name>mapreduce.reduce.memory.mb</name>
        <value>2048</value>
    </property>

    <!--
        mapreduce.map.java.opts
        ──────────────────────────
        Map Task 的 JVM 堆内存参数。

        这个值应该小于 mapreduce.map.memory.mb，
        留一些内存给 JVM 自身的非堆内存使用。

        推荐公式: mapreduce.map.memory.mb * 0.8

        示例: 如果 map.memory.mb = 1024，则 java.opts = -Xmx820m
    -->
    <property>
        <name>mapreduce.map.java.opts</name>
        <value>-Xmx820m</value>
    </property>

    <!--
        mapreduce.reduce.java.opts
        ───────────────────────────
        Reduce Task 的 JVM 堆内存参数。

        同样应该小于 mapreduce.reduce.memory.mb。

        推荐公式: mapreduce.reduce.memory.mb * 0.8

        示例: 如果 reduce.memory.mb = 2048，则 java.opts = -Xmx1638m
    -->
    <property>
        <name>mapreduce.reduce.java.opts</name>
        <value>-Xmx1638m</value>
    </property>

    <!--
        mapreduce.map.cpu.vcores
        ──────────────────────────
        每个 Map Task 请求的 CPU 虚拟核心数。

        默认值: 1
    -->
    <property>
        <name>mapreduce.map.cpu.vcores</name>
        <value>1</value>
    </property>

    <!--
        mapreduce.reduce.cpu.vcores
        ───────────────────────────
        每个 Reduce Task 请求的 CPU 虚拟核心数。

        默认值: 1
    -->
    <property>
        <name>mapreduce.reduce.cpu.vcores</name>
        <value>1</value>
    </property>

    <!--
        mapreduce.task.io.sort.mb
        ──────────────────────────
        Map 端 Shuffle 排序时的内存缓冲区大小（MB）。

        Map 输出的数据会先写入这个内存缓冲区，
        缓冲区满了后 Spill 到磁盘。缓冲区越大，Spill 次数越少，性能越好。

        默认值: 100

        注意: 这个值不能超过 mapreduce.map.java.opts 中 -Xmx 的值
    -->
    <property>
        <name>mapreduce.task.io.sort.mb</name>
        <value>100</value>
    </property>

    <!--
        mapreduce.task.io.sort.factor
        ─────────────────────────────
        合并 Spill 文件时，一次合并的文件数量。

        值越大，合并次数越少，但消耗更多内存。

        默认值: 10
    -->
    <property>
        <name>mapreduce.task.io.sort.factor</name>
        <value>10</value>
    </property>

    <!--
        mapreduce.reduce.shuffle.parallelcopies
        ────────────────────────────────────────
        Reduce 端并行从 Map Task 拉取数据的线程数。

        增大这个值可以加快 Shuffle 速度，但会增加网络压力。

        默认值: 5
        推荐值: 5-10（根据集群规模和网络带宽调整）
    -->
    <property>
        <name>mapreduce.reduce.shuffle.parallelcopies</name>
        <value>5</value>
    </property>

    <!--
        mapreduce.reduce.shuffle.input.buffer.percent
        ───────────────────────────────────────────
        Reduce 端用于存放 Shuffle 数据的堆内存比例。

        值范围: 0.0 - 1.0
        默认值: 0.7（即 Reduce JVM 堆内存的 70% 用于 Shuffle 缓冲）

        如果 Shuffle 数据量很大，可以适当增大。
    -->
    <property>
        <name>mapreduce.reduce.shuffle.input.buffer.percent</name>
        <value>0.7</value>
    </property>

    <!--
        mapreduce.job.reduce.slowstart.completedmaps
        ─────────────────────────────────────────────
        Reduce Task 开始运行的时机。

        当已完成的 Map Task 数量占 Map Task 总数的比例达到这个值时，
        开始调度 Reduce Task。

        默认值: 0.05（即 5% 的 Map 完成后就开始 Reduce）

        设为 1.0 表示等所有 Map 完成后才开始 Reduce（适合小作业）
        设为 0.0 表示第一个 Map 完成后就开始 Reduce

        推荐值:
          - 小作业: 0.8-1.0（减少资源占用）
          - 大作业: 0.05-0.2（Map 和 Reduce 重叠执行，节省时间）
    -->
    <property>
        <name>mapreduce.job.reduce.slowstart.completedmaps</name>
        <value>0.8</value>
    </property>

    <!--
        mapreduce.map.maxattempts
        ──────────────────────────
        Map Task 最大重试次数。

        当 Map Task 失败后，YARN 会在其他节点上重新执行。
        达到最大重试次数后，整个作业失败。

        默认值: 4
    -->
    <property>
        <name>mapreduce.map.maxattempts</name>
        <value>4</value>
    </property>

    <!--
        mapreduce.reduce.maxattempts
        ───────────────────────────
        Reduce Task 最大重试次数。

        默认值: 4
    -->
    <property>
        <name>mapreduce.reduce.maxattempts</name>
        <value>4</value>
    </property>

    <!--
        mapreduce.map.speculative
        ──────────────────────────
        是否启用 Map Task 的推测执行。

        推测执行: 当某个 Map Task 明显比其他 Task 慢时，
        YARN 会在另一个节点上启动一个相同的 Task（备份 Task），
        哪个先完成就用哪个的结果。

        好处: 避免"慢节点"拖慢整个作业
        坏处: 浪费集群资源

        默认值: true
        建议: 负载高的集群关闭，资源充足的集群可以开启
    -->
    <property>
        <name>mapreduce.map.speculative</name>
        <value>false</value>
    </property>

    <!--
        mapreduce.reduce.speculative
        ───────────────────────────
        是否启用 Reduce Task 的推测执行。

        默认值: true
    -->
    <property>
        <name>mapreduce.reduce.speculative</name>
        <value>false</value>
    </property>

    <!--
        mapreduce.jobhistory.address
        ────────────────────────────
        MapReduce JobHistory Server 的 RPC 地址。

        JobHistory Server 记录已完成的作业信息，
        可以通过 Web UI 查看历史作业的详情。
    -->
    <property>
        <name>mapreduce.jobhistory.address</name>
        <value>localhost:10020</value>
    </property>

    <!--
        mapreduce.jobhistory.webapp.address
        ───────────────────────────────────
        JobHistory Server 的 Web UI 地址。
    -->
    <property>
        <name>mapreduce.jobhistory.webapp.address</name>
        <value>localhost:19888</value>
    </property>

    <!--
        mapreduce.jobhistory.intermediate-done-dir
        ─────────────────────────────────────────
        作业运行过程中的日志存放目录（在 HDFS 上）。

        作业运行时产生的中间日志会写入这个目录。
    -->
    <property>
        <name>mapreduce.jobhistory.intermediate-done-dir</name>
        <value>/mr-history/tmp</value>
    </property>

    <!--
        mapreduce.jobhistory.done-dir
        ────────────────────────────
        作业完成后的日志存放目录（在 HDFS 上）。

        作业完成后，中间日志会被移动到这个目录。
        JobHistory Server 从这个目录读取历史作业信息。
    -->
    <property>
        <name>mapreduce.jobhistory.done-dir</name>
        <value>/mr-history/done</value>
    </property>

</configuration>
```

## 三、yarn-site.xml 补充配置

在上一篇文档中，我们配置了基本的 `yarn-site.xml`。这里补充一些重要的进阶配置。

```xml
<!-- 以下配置追加到 yarn-site.xml 中 -->

    <!--
        yarn.log-aggregation.retain-seconds
        ─────────────────────────────────────
        聚合日志在 HDFS 上的保留时间（秒）。

        默认值: -1（永久保留）
        推荐值: 604800（7 天）或 2592000（30 天）

        过期后，YARN 会自动删除聚合日志文件。
    -->
    <property>
        <name>yarn.log-aggregation.retain-seconds</name>
        <value>604800</value>
    </property>

    <!--
        yarn.log-aggregation.retain-check-interval-seconds
        ──────────────────────────────────────────────────
        日志清理线程的检查间隔（秒）。

        默认值: -1（使用 retain-seconds 的 1/10）
    -->
    <property>
        <name>yarn.log-aggregation.retain-check-interval-seconds</name>
        <value>86400</value>
    </property>

    <!--
        yarn.resourcemanager.am.max-attempts
        ──────────────────────────────────────
        ApplicationMaster 最大重启次数。

        当 AM 异常退出后，RM 会尝试重新启动 AM。
        达到最大次数后，作业标记为失败。

        默认值: 2
    -->
    <property>
        <name>yarn.resourcemanager.am.max-attempts</name>
        <value>2</value>
    </property>

    <!--
        yarn.nodemanager.local-dirs
        ───────────────────────────
        NodeManager 的本地工作目录（用于存放 Container 的临时文件）。

        可以配置多个目录，用逗号分隔。
        建议每个磁盘配置一个目录。
    -->
    <property>
        <name>yarn.nodemanager.local-dirs</name>
        <value>${hadoop.tmp.dir}/nm-local-dir</value>
    </property>

    <!--
        yarn.nodemanager.log-dirs
        ─────────────────────────
        NodeManager 存放 Container 日志的目录。

        可以配置多个目录。
    -->
    <property>
        <name>yarn.nodemanager.log-dirs</name>
        <value>${hadoop.tmp.dir}/nm-logs</value>
    </property>

    <!--
        yarn.scheduler.capacity.root.default.maximum-am-resource-percent
        ────────────────────────────────────────────────────────────────
        集群中用于运行 ApplicationMaster 的最大资源比例。

        默认值: 0.1（10%）

        如果同时运行很多小作业，AM 会占用较多资源。
        如果 AM 资源不足，作业会一直处于 ACCEPTED 状态。
    -->
    <property>
        <name>yarn.scheduler.capacity.root.default.maximum-am-resource-percent</name>
        <value>0.1</value>
    </property>
```

## 四、内存配置计算方法

### 4.1 资源分配原则

```
┌────────────────────────────────────────────────────────────────┐
│              YARN 内存配置计算方法                                │
│                                                                │
│  原则 1: Container 总内存 <= NM 总内存                         │
│  原则 2: Map 内存 < Reduce 内存（通常 Reduce = 2 * Map）       │
│  原则 3: JVM 堆内存 < Container 内存（留 20% 给非堆）         │
│  原则 4: 排序缓冲区 < JVM 堆内存                              │
│                                                                │
│  假设: 一台 8GB 内存的机器                                      │
│  ─────────────────────────────────────────────────             │
│                                                                │
│  Step 1: 确定 NM 总内存                                        │
│    操作系统 + 其他服务约占 2GB                                  │
│    yarn.nodemanager.resource.memory-mb = 6144 (6GB)            │
│                                                                │
│  Step 2: 确定 Map/Reduce 内存                                  │
│    mapreduce.map.memory.mb = 1024 (1GB)                        │
│    mapreduce.reduce.memory.mb = 2048 (2GB)                     │
│                                                                │
│  Step 3: 确定 JVM 堆内存                                       │
│    mapreduce.map.java.opts = -Xmx820m (1024 * 0.8)            │
│    mapreduce.reduce.java.opts = -Xmx1638m (2048 * 0.8)        │
│                                                                │
│  Step 4: 确定排序缓冲区                                        │
│    mapreduce.task.io.sort.mb = 100 (< 820m)                   │
│                                                                │
│  Step 5: 计算并发 Task 数                                      │
│    最大 Map Task = 6144 / 1024 = 6 个                         │
│    最大 Reduce Task = 6144 / 2048 = 3 个                      │
│    （Map 和 Reduce 共享 NM 内存，实际并发数取决于同时运行的量） │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

### 4.2 不同机器配置的推荐值

```
┌────────────────────────────────────────────────────────────────┐
│              不同机器配置的推荐值                                 │
├──────────┬──────────┬──────────┬──────────┬─────────────────────┤
│ 机器内存  │ NM 内存   │ Map 内存  │ Reduce 内存│ JVM 堆(Reduce)   │
├──────────┼──────────┼──────────┼──────────┼─────────────────────┤
│ 4GB      │ 2048     │ 512      │ 1024     │ -Xmx820m           │
│ 8GB      │ 6144     │ 1024     │ 2048     │ -Xmx1638m          │
│ 16GB     │ 12288    │ 2048     │ 4096     │ -Xmx3276m          │
│ 32GB     │ 24576    │ 4096     │ 8192     │ -Xmx6554m          │
│ 64GB     │ 49152    │ 4096     │ 8192     │ -Xmx6554m          │
└──────────┴──────────┴──────────┴──────────┴─────────────────────┘
```

### 4.3 配置关系图

```
┌──────────────────────────────────────────────────────────────────┐
│              配置项之间的关系                                      │
│                                                                  │
│  物理内存 (Physical RAM)                                         │
│  └── NM 总内存 (yarn.nodemanager.resource.memory-mb)             │
│      │                                                           │

│      │   └── JVM 堆 (mapreduce.map.java.opts = -Xmx...)         │

│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

## 五、JobHistory 历史服务器配置和使用

### 5.1 为什么需要 JobHistory？

```
┌──────────────────────────────────────────────────────────────┐
│              为什么需要 JobHistory？                            │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  YARN ResourceManager 只显示正在运行和最近完成的作业。        │
│  如果你想查看几天前某个 MapReduce 作业的详细信息:            │
│                                                              │
│  - 每个 Task 的运行时间和进度                                 │
│  - 每个 Task 的日志                                           │
│  - 作业的 Counter（计数器）                                   │
│  - 作业的配置参数                                             │
│  - 作业失败的原因                                             │
│                                                              │
│  这些信息在 RM 中已经被清理了。                               │
│  JobHistory Server 就是用来保存和查看这些历史信息的。          │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 5.2 配置 JobHistory

`mapred-site.xml` 中已包含 JobHistory 的配置（见上面的配置说明）。你还需要确保以下目录在 HDFS 上存在：

```bash
# 创建 JobHistory 所需的 HDFS 目录
hdfs dfs -mkdir -p /mr-history/tmp
hdfs dfs -mkdir -p /mr-history/done

# 设置权限（确保所有用户都能读写）
hdfs dfs -chmod -R 1777 /mr-history/tmp
hdfs dfs -chmod -R 1777 /mr-history/done
```

### 5.3 启动 JobHistory Server

```bash
# 启动 JobHistory Server
mapred --daemon start historyserver

# 停止 JobHistory Server
mapred --daemon stop historyserver

# 验证启动
jps | grep JobHistoryServer
# 12345 JobHistoryServer
```

打开浏览器访问 http://localhost:19888

```bash
# 查看所有已完成的作业

# 查看特定作业的详情
mapred job -history all /mr-history/done/user/xxx/job_xxx
```

## 六、日志聚合配置

### 6.1 什么是日志聚合？

在默认情况下，MapReduce 任务的日志散落在各个 NodeManager 的本地磁盘上。如果集群有很多节点，查看日志就变得非常困难。

**日志聚合**功能会将所有 Container 的日志收集并存储到 HDFS 上的统一目录中，方便集中查看。

```
┌──────────────────────────────────────────────────────────────────┐
│              日志聚合原理                                          │
│                                                                  │
```

### 6.2 日志聚合配置

在 `yarn-site.xml` 中配置（已在前面列出）：

```xml
<!-- 启用日志聚合 -->
<property>
    <name>yarn.log-aggregation-enable</name>
    <value>true</value>
</property>

<!-- 日志保留时间: 7 天 -->
<property>
    <name>yarn.log-aggregation.retain-seconds</name>
    <value>604800</value>
</property>
```

### 6.3 查看聚合日志

```bash
# 方法 1: 通过 yarn logs 命令
yarn logs -applicationId application_1713138000000_0001

# 方法 2: 只查看特定 Container 的日志
yarn logs -applicationId application_1713138000000_0001 \
    -containerId container_1713138000000_0001_01_000001

# 方法 3: 通过 RM Web UI (http://localhost:8088)
# 点击某个 Application → 点击 Logs 链接
```

### 6.4 日志聚合的目录结构

```
/tmp/logs/${user}/logs/${application_id}/
├── ${container_id}/
│   ├── stdout       ← System.out 输出
│   ├── stderr       ← System.err 输出
│   └── syslog       ← 日志框架（log4j）输出
├── ${container_id}/
│   ├── stdout
│   ├── stderr
│   └── syslog
└── ...
```
