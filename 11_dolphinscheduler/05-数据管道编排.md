### 2.2 节点详细配置

**节点 1：日志文件检查（Shell）**

```bash
#!/bin/bash

# 设置日期变量
DT=${system.biz.date}
LOG_DIR=/var/log/nginx
LOG_FILE=${LOG_DIR}/demo_access_${DT}.log.gz

# 检查日志文件是否存在
if [ -f "${LOG_FILE}" ]; then
    FILE_SIZE=$(du -h "${LOG_FILE}" | awk '{print $1}')
    echo "日志文件存在: ${LOG_FILE}, 大小: ${FILE_SIZE}"
    echo '${setValue(log_exists=true)}'
    echo '${setValue(log_file='${LOG_FILE}')}'
    echo '${setValue(log_size='${FILE_SIZE}')}'
else
    echo "警告: 日志文件不存在: ${LOG_FILE}"
    echo '${setValue(log_exists=false)}'
fi

# 同时检查搜索日志
SEARCH_LOG=${LOG_DIR}/demo_search_${DT}.log.gz
if [ -f "${SEARCH_LOG}" ]; then
    echo "搜索日志存在: ${SEARCH_LOG}"
    echo '${setValue(search_log_exists=true)}'
else
    echo '${setValue(search_log_exists=false)}'
fi
```

**节点 2：日志上传 HDFS（Shell）**

```bash
#!/bin/bash

# 上传访问日志到 HDFS
HDFS_PATH=/user/hive/warehouse/demo_access/dt=${system.biz.date}

# 创建 HDFS 目录（如果不存在）
hadoop fs -mkdir -p ${HDFS_PATH}

# 上传日志文件
if [ "${log_exists}" = "true" ]; then
    hadoop fs -put -f ${log_file} ${HDFS_PATH}/access.log.gz
    echo "访问日志上传成功: ${log_file} → ${HDFS_PATH}/access.log.gz"
    echo '${setValue(access_log_path='${HDFS_PATH}/access.log.gz')}'
else
    echo "访问日志不存在，跳过上传"
    echo '${setValue(access_log_path=)}'
fi

# 上传搜索日志
SEARCH_HDFS_PATH=/user/hive/warehouse/demo_search_log/dt=${system.biz.date}
hadoop fs -mkdir -p ${SEARCH_HDFS_PATH}

if [ "${search_log_exists}" = "true" ]; then
    SEARCH_LOG_FILE=/var/log/nginx/demo_search_${system.biz.date}.log.gz
    hadoop fs -put -f ${SEARCH_LOG_FILE} ${SEARCH_HDFS_PATH}/search.log.gz
    echo "搜索日志上传成功"
fi

# 验证上传
echo "HDFS 中的文件列表:"
hadoop fs -ls ${HDFS_PATH}
```

**节点 2：行数校验（SQL）**

```sql
-- 检查当天的数据量是否正常
-- 与前一天的行数做对比，偏差超过 50% 则标记异常

WITH today_stats AS (
    SELECT
        COUNT(*) AS today_rows,
        COUNT(DISTINCT user_id) AS today_users
    FROM demo_access
    WHERE dt = '${system.biz.date}'
),
yesterday_stats AS (
    SELECT
        COUNT(*) AS yesterday_rows,
        COUNT(DISTINCT user_id) AS yesterday_users
    FROM demo_access
    WHERE dt = DATE_SUB('${system.biz.date}', 1)
)
SELECT
    t.today_rows,
    COALESCE(y.yesterday_rows, 0) AS yesterday_rows,
    t.today_users,
    COALESCE(y.yesterday_users, 0) AS yesterday_users,
    CASE
        WHEN y.yesterday_rows = 0 THEN 'NO_BASELINE'
        WHEN ABS(t.today_rows - y.yesterday_rows) * 1.0 / y.yesterday_rows > 0.5
        THEN 'ANOMALY'
        ELSE 'NORMAL'
    END AS quality_status
FROM today_stats t
LEFT JOIN yesterday_stats y ON 1=1;
```

**节点 3：Switch 条件分支**

```

分支：
- true → [空值检查]
- false（默认）→ [发送告警]
```

**节点 4：空值检查（SQL）**

```sql
-- 检查关键字段的空值率

SELECT
    COUNT(*) AS total_rows,
    SUM(CASE WHEN user_id IS NULL OR user_id = '' THEN 1 ELSE 0 END) AS user_id_null,
    SUM(CASE WHEN post_id IS NULL OR post_id = '' THEN 1 ELSE 0 END) AS post_id_null,
    SUM(CASE WHEN ip IS NULL OR ip = '' THEN 1 ELSE 0 END) AS ip_null,
    ROUND(
        SUM(CASE WHEN user_id IS NULL OR user_id = '' THEN 1 ELSE 0 END) * 100.0
        / NULLIF(COUNT(*), 0), 2
    ) AS user_id_null_rate
FROM demo_access
WHERE dt = '${system.biz.date}';
```

对应的核心逻辑（如果使用 spark-submit 的方式）：

```python
# 也可以用 Python 任务 + PySpark 实现
from pyspark.sql import SparkSession
from pyspark.sql import functions as F

spark = SparkSession.builder \
    .appName("DemoDAU") \
    .enableHiveSupport() \
    .getOrCreate()

dt = "${system.biz.date}"

# 计算 DAU
dau = spark.sql(f"""
    SELECT
        '{dt}' AS stat_date,
        COUNT(DISTINCT user_id) AS dau,
        COUNT(DISTINCT CASE WHEN user_id != 'anonymous' THEN user_id END) AS real_dau,
        COUNT(DISTINCT ip) AS unique_ip
    FROM demo_access
    WHERE dt = '{dt}'
""")

# 写入 Iceberg
dau.writeTo("iceberg.demo_daily_stats").using("iceberg").append()
```

**节点 2c：热门文章（Spark）**

```python
spark = SparkSession.builder \

top_posts = spark.sql(f"""

        '{dt}' AS stat_date,
```

## 八、补数（Backfill）操作

### 8.1 什么是补数？

当某天的数据流水线因为故障没有执行，或者需要重新计算历史数据时，就需要"补数"。

```
┌──────────────────────────────────────────────────────────────┐
│                    补数场景                                    │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  场景 1：4/10 - 4/12 三天的定时任务因为 HDFS 维护没有执行    │
│  → 需要回填这三天的数据                                      │
│                                                              │
│  场景 2：发现 4/8 的指标计算 SQL 有 bug，结果不准确          │
│  → 需要修复 SQL 后重新计算 4/8 的数据                       │
│                                                              │
│  场景 3：新加了一个指标，需要回填最近 30 天的历史数据        │
│  → 需要批量执行过去 30 天的工作流                            │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 8.2 在 Web UI 中执行补数

```
操作步骤：

1. 进入工作流定义页面
2. 找到需要补数的工作流（如 demo_data_collect）
3. 点击「运行」按钮（不要直接点，选择下拉菜单中的「补数」）
4. 在补数配置页面：
   - 补数方式：日期补数
   - 开始日期：2026-04-10
   - 结束日期：2026-04-12
   - 失败策略：结束（遇到失败停止）/ 继续（跳过失败继续）
   - 调度日期：选择需要补数的日期范围
5. 点击「执行」

DolphinScheduler 会按照日期顺序，依次生成并执行多个工作流实例：
- demo_data_collect (2026-04-10)
- demo_data_collect (2026-04-11)
- demo_data_collect (2026-04-12)

注意：
- 补数期间，${system.biz.date} 会自动替换为对应的补数日期
- 如果工作流之间有 Dependent 依赖，下游工作流会自动被触发
- 补数可能产生大量任务实例，注意 Worker 资源
```
