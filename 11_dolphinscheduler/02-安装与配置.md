## 一、安装前准备

### 1.2 检查前置条件

```bash
# 检查 JDK 版本（需要 JDK 8+，我们用 JDK 21）
java -version
# 输出：openjdk version "21.0.x"

# 检查 MySQL 是否运行
mysql -u root -p -e "SELECT VERSION();"
# 输出：8.0.x

# 确认安装目录存在
ls /opt/
```

## 二、下载与安装

### 2.1 下载安装包

```bash
# 创建安装目录
mkdir -p /opt/dolphinscheduler

# 从阿里云镜像下载（国内速度快）
cd ~/Programme/claude-study
curl -O https://mirrors.aliyun.com/apache/dolphinscheduler/3.4.1/apache-dolphinscheduler-3.4.1-bin.tar.gz
```

### 2.2 解压安装

```bash
# 解压到安装目录
tar -xzf apache-dolphinscheduler-3.4.1-bin.tar.gz -C /opt/dolphinscheduler/

# 重命名为版本号目录（方便管理多版本）
mv /opt/dolphinscheduler/apache-dolphinscheduler-3.4.1-bin /opt/dolphinscheduler/3.4.1

# 确认安装
ls /opt/dolphinscheduler/3.4.1/
```

### 2.3 配置环境变量

```bash
# 编辑 ~/.zshrc，添加以下内容
export DOLPHINSCHEDULER_HOME=/opt/dolphinscheduler/3.4.1
export PATH=$PATH:$DOLPHINSCHEDULER_HOME/bin

# 使环境变量生效
source ~/.zshrc
```

## 三、数据库初始化（MySQL）

默认情况下，Standalone 模式使用内嵌的 H2 数据库。但在生产环境（以及我们数据分析项目）中，需要使用外部 MySQL 数据库来持久化元数据。

### 3.1 创建 DolphinScheduler 数据库

```sql
-- 登录 MySQL
mysql -u root -p

-- 创建数据库
CREATE DATABASE dolphinscheduler DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建专用用户（可选，也可以用 root）
CREATE USER 'dolphin'@'%' IDENTIFIED BY 'Dolphin_123';
GRANT ALL PRIVILEGES ON dolphinscheduler.* TO 'dolphin'@'%';
FLUSH PRIVILEGES;
```

### 3.2 下载 MySQL 驱动

DolphinScheduler 默认不带 MySQL 驱动，需要手动下载：

```bash
# 下载 MySQL Connector/J 8.x（与 MySQL 8.0 匹配）
# 注意：根据 DolphinScheduler 3.4.1 要求，使用 8.0.33 版本
curl -O https://repo1.maven.org/maven2/com/mysql/mysql-connector-j/8.0.33/mysql-connector-j-8.0.33.jar

# 放到 DolphinScheduler 的 lib 目录
mv mysql-connector-j-8.0.33.jar /opt/dolphinscheduler/3.4.1/libs/
```

### 3.3 修改数据库配置

编辑配置文件：

```bash
vim /opt/dolphinscheduler/3.4.1/standalone-server/conf/application.yaml
```

找到 `spring.datasource` 配置段，修改为：

```yaml
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://localhost:3306/dolphinscheduler?useUnicode=true&characterEncoding=UTF-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: dolphin
    password: Dolphin_123
```


## 四、核心配置文件详解

DolphinScheduler 的配置文件位于 `standalone-server/conf/` 目录下：

```
/opt/dolphinscheduler/3.4.1/standalone-server/conf/
├── application.yaml        # 主配置文件（数据库、端口等）
├── application-api.yaml    # API Server 配置
├── logback-spring.xml      # 日志配置
└── dolphinscheduler-env.sh  # 环境变量（JDK 路径等）
```

### 4.1 application.yaml 核心配置

```yaml
# ==================== 服务器配置 ====================
server:
  port: 12345                    # Web UI 端口

# ==================== 数据源配置 ====================
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://localhost:3306/dolphinscheduler?...
    username: dolphin
    password: Dolphin_123

# ==================== 注册中心配置 ====================
registry:
  type: jdbc                     # Standalone 模式用 jdbc
  # 集群模式用 zookeeper：
  # type: zookeeper
  # zookeeper:
  #   quorum: localhost:2181

# ==================== Master 配置 ====================
master:
  listen-port: 25666             # Master 监听端口
  worker-group: default          # 默认 Worker 分组

# ==================== Worker 配置 ====================
worker:
  listen-port: 1234              # Worker 监听端口
  groups:
    - default                    # Worker 所属分组
  # 执行任务的并行度
  exec-threads: 20
  # 任务执行完成后日志保留天数
  tenant-auto-create: true       # 自动创建 Linux 租户

# ==================== 告警配置 ====================
alert:
  port: 50052                    # Alert Server 端口

# ==================== 日志配置 ====================
# 任务执行日志存放在以下目录：
# /opt/dolphinscheduler/3.4.1/standalone-server/logs/
```

### 4.2 dolphinscheduler-env.sh 环境变量

```bash
# 必须配置！指定 JDK 路径
export JAVA_HOME=$(/usr/libexec/java_home -v 21)

# DolphinScheduler 安装目录
export DOLPHINSCHEDULER_HOME=/opt/dolphinscheduler/3.4.1

# DolphinScheduler 数据目录（存放日志、资源等）
export DOLPHINSCHEDULER_DATA=$DOLPHINSCHEDULER_HOME/standalone-server/data

# 日志目录
export DOLPHINSCHEDULER_LOG=$DOLPHINSCHEDULER_HOME/standalone-server/logs

# 任务执行时的环境变量（会传递给每个任务）
export PATH=$JAVA_HOME/bin:$PATH

# 如果需要使用 Hadoop/Hive/Spark，需要配置对应的环境变量
export HADOOP_HOME=/opt/hadoop/3.4.2
export HADOOP_CONF_DIR=$HADOOP_HOME/etc/hadoop
export HIVE_HOME=/opt/hive/4.0.1
export SPARK_HOME=/opt/spark/3.5.8
export PATH=$HADOOP_HOME/bin:$HIVE_HOME/bin:$SPARK_HOME/bin:$PATH
```

```
┌──────────────────────────────────────────────────────────┐
│            dolphinscheduler-env.sh 配置要点                │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  JAVA_HOME 必须正确设置                                   │
│  ├── Standalone 模式下所有组件都在同一 JVM 中              │
│  └── Worker 执行任务时也会用到这个 JAVA_HOME               │
│                                                          │
│  如果 Worker 需要提交 Spark/Flink 任务：                  │
│  ├── 配置 SPARK_HOME，Worker 才能找到 spark-submit        │
│  ├── 配置 HADOOP_HOME，任务才能访问 HDFS                  │
│  └── 配置 HIVE_HOME，SQL 任务才能连接 Hive                │
│                                                          │
│  重要：修改此文件后需要重启 DolphinScheduler               │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

## 五、启动与停止

### 5.1 启动 Standalone 服务

```bash
# 启动（包含 Master + Worker + API + Alert + Logger）
dolphinscheduler-daemon.sh start standalone-server
```

启动过程中，控制台会输出启动日志：

```
┌──────────────────────────────────────────────────────────┐
│                  启动过程解析                               │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  1. 加载 application.yaml 配置                            │
│  2. 连接 MySQL，初始化数据库表（首次启动）                  │
│  3. 启动 Alert Server    → 告警服务就绪                   │
│  4. 启动 Logger Server   → 日志服务就绪                   │
│  5. 启动 Master Server   → 调度服务就绪                   │
│  6. 启动 Worker Server   → 执行服务就绪                   │
│  7. 启动 API Server      → Web UI 就绪                   │
│  8. 输出：Standalone server started                      │
│                                                          │
│  全部组件打包在一个 JVM 中运行（Standalone 模式）          │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 5.2 验证启动状态

```bash
# 方式 1：检查 Java 进程
jps | grep StandaloneServer
# 输出：12345 StandaloneServer

# 方式 2：检查端口
lsof -i :12345
# 输出：java  12345  ...  TCP *:12345 (LISTEN)

# 方式 3：查看日志
tail -f /opt/dolphinscheduler/3.4.1/standalone-server/logs/dolphinscheduler-standalone.log
```

### 5.3 停止服务

```bash
# 停止
dolphinscheduler-daemon.sh stop standalone-server

# 确认已停止
jps | grep StandaloneServer
# 无输出说明已停止
```

## 八、常见问题排查

### 8.1 启动失败：端口被占用

```bash
# 查看端口占用
lsof -i :12345

# 杀掉占用端口的进程
kill -9 <PID>

# 重新启动
dolphinscheduler-daemon.sh start standalone-server
```

### 8.2 启动失败：数据库连接失败

```
现象：日志中出现 "Communications link failure" 或 "Access denied"

排查步骤：
1. 确认 MySQL 服务是否运行
   mysql -u root -p -e "SELECT 1"

2. 确认数据库和用户是否创建
   mysql -u root -p -e "SHOW DATABASES"
   mysql -u dolphin -p'Dolphin_123' -e "USE dolphinscheduler"

3. 确认 MySQL 驱动是否放到了 libs/ 目录
   ls /opt/dolphinscheduler/3.4.1/libs/mysql-connector-*

4. 确认 application.yaml 中的数据库配置正确
   cat /opt/dolphinscheduler/3.4.1/standalone-server/conf/application.yaml
```

### 8.3 Web UI 无法访问

```
现象：浏览器打开 http://localhost:12345 无响应

排查步骤：
1. 确认服务是否启动
   jps | grep StandaloneServer

2. 确认端口是否监听
   lsof -i :12345

3. 查看启动日志是否有报错
   tail -100 /opt/dolphinscheduler/3.4.1/standalone-server/logs/dolphinscheduler-standalone.log

4. 确认防火墙没有阻止端口访问
   curl http://localhost:12345/dolphinscheduler/ui
```

### 8.4 Shell 任务执行失败

```
现象：Shell 任务状态为红色（失败），日志显示 "command not found"

原因：dolphinscheduler-env.sh 中的 PATH 没有包含所需命令的路径

解决方案：
1. 编辑 dolphinscheduler-env.sh
2. 确保 PATH 包含所有需要的路径：
   export PATH=$JAVA_HOME/bin:/usr/local/bin:/usr/bin:/bin:$PATH
3. 重启 DolphinScheduler
```

### 8.5 常用排查命令速查

```bash
# 查看 Standalone 进程状态
jps -l | grep Standalone

# 查看启动日志（最后 100 行）
```
