## 二、Iceberg 集成方式

Iceberg 本身不需要单独安装，它是一个**表格式规范 + 运行时库**。我们只需要把 Iceberg 的 JAR 包加载到 Spark 中即可。

### 方式一：--packages 参数（推荐学习使用）

每次启动 Spark 时自动从 Maven 中央仓库下载：

```bash
spark-sql --packages org.apache.iceberg:iceberg-spark-runtime-3.5_2.12:1.10.1 \
  --conf spark.sql.extensions=org.apache.iceberg.spark.extensions.IcebergSparkSessionExtensions
```

**优点：** 简单，不需要手动下载 JAR 包
**缺点：** 每次启动需要联网，首次下载较慢（约 200MB）

### 方式二：JAR 包手动下载（离线使用）

```bash
# 1. 下载 Iceberg Spark Runtime JAR
curl -O https://repo1.maven.org/maven2/org/apache/iceberg/iceberg-spark-runtime-3.5_2.12/1.10.1/iceberg-spark-runtime-3.5_2.12-1.10.1.jar

# 2. 复制到 Spark 的 jars 目录
mv iceberg-spark-runtime-3.5_2.12-1.10.1.jar /opt/spark/3.5.8/jars/

# 3. 还需要 MySQL JDBC 驱动（JdbcCatalog 需要）
curl -O https://repo1.maven.org/maven2/com/mysql/mysql-connector-j/8.0.33/mysql-connector-j-8.0.33.jar
mv mysql-connector-j-8.0.33.jar /opt/spark/3.5.8/jars/
```

### 方式三：spark-defaults.conf 持久化配置（推荐生产使用）

将配置写入 Spark 的默认配置文件，这样每次启动都不需要重复指定参数。

编辑 `/opt/spark/3.5.8/conf/spark-defaults.conf`：

```properties
# ===== Iceberg 基础配置 =====

# Iceberg SQL 扩展（必须，支持 MERGE INTO、时间旅行等语法）
spark.sql.extensions=org.apache.iceberg.spark.extensions.IcebergSparkSessionExtensions

# ===== Iceberg Catalog 配置（JdbcCatalog 方式）=====

# 注册名为 iceberg 的 Catalog
spark.sql.catalog.iceberg=org.apache.iceberg.spark.SparkCatalog

# 使用 JdbcCatalog（直连 MySQL，不依赖 Hive Metastore）
spark.sql.catalog.iceberg.catalog-impl=org.apache.iceberg.jdbc.JdbcCatalog

# MySQL 连接地址（iceberg 为数据库名）
spark.sql.catalog.iceberg.uri=jdbc:mysql://localhost:3306/iceberg

# MySQL 用户名和密码
spark.sql.catalog.iceberg.jdbc.user=root
spark.sql.catalog.iceberg.jdbc.password=你的MySQL密码

# 数据仓库路径（HDFS 上的存储位置）
spark.sql.catalog.iceberg.warehouse=hdfs://localhost:9000/iceberg-warehouse

# ===== 可选优化配置 =====

# 并发写入优化
spark.sql.catalog.iceberg.lock-impl=org.apache.iceberg.jdbc.JdbcLock
```

## 三、MySQL 数据库准备

JdbcCatalog 需要一个 MySQL 数据库来存储 Iceberg 的元数据。

```sql
-- 登录 MySQL
mysql -u root -p

-- 创建 Iceberg 元数据库
CREATE DATABASE IF NOT EXISTS iceberg CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;

-- 验证
USE iceberg;
SHOW TABLES;
-- 刚创建时应该是空的，Iceberg 会自动创建需要的表
```

## 五、spark-sql 验证 Iceberg 功能

### 5.1 启动 spark-sql

```bash
# 使用 --packages 方式启动
spark-sql \
  --packages org.apache.iceberg:iceberg-spark-runtime-3.5_2.12:1.10.1 \
  --conf spark.sql.extensions=org.apache.iceberg.spark.extensions.IcebergSparkSessionExtensions \
  --conf spark.sql.catalog.iceberg=org.apache.iceberg.spark.SparkCatalog \
  --conf spark.sql.catalog.iceberg.catalog-impl=org.apache.iceberg.jdbc.JdbcCatalog \
  --conf spark.sql.catalog.iceberg.uri=jdbc:mysql://localhost:3306/iceberg \
  --conf spark.sql.catalog.iceberg.jdbc.user=root \
  --conf spark.sql.catalog.iceberg.jdbc.password=你的MySQL密码 \
  --conf spark.sql.catalog.iceberg.warehouse=hdfs://localhost:9000/iceberg-warehouse
```

> 如果已经在 `spark-defaults.conf` 中配置好了，直接运行 `spark-sql` 即可。

### 5.2 基本功能验证

```sql
-- 1. 查看 Catalog 列表
SHOW NAMESPACES IN iceberg;

-- 2. 创建业务数据库
CREATE NAMESPACE IF NOT EXISTS iceberg.demo;

-- 3. 验证数据库已创建
SHOW NAMESPACES IN iceberg;
-- 输出应该包含 demo

-- 4. 创建第一张 Iceberg 表
CREATE TABLE IF NOT EXISTS iceberg.demo.test_table (
    id INT,
    name STRING,
    created_at TIMESTAMP
) USING iceberg
TBLPROPERTIES ('format-version'='2');

-- 5. 写入测试数据
INSERT INTO iceberg.demo.test_table VALUES
    (1, 'hello_iceberg', TIMESTAMP '2026-04-15 10:00:00'),
    (2, 'first_article', TIMESTAMP '2026-04-15 11:00:00');

-- 6. 查询数据
SELECT * FROM iceberg.demo.test_table;

-- 7. 查看快照历史
SELECT snapshot_id, committed_at, operation, summary
FROM iceberg.demo.test_table.snapshots;

-- 8. 查看文件信息
SELECT file_path, file_format, record_count, file_size_in_bytes
FROM iceberg.demo.test_table.files;

-- 9. 清理测试表
DROP TABLE iceberg.demo.test_table;
```

### 5.3 验证结果检查

如果以上操作全部成功，说明 Iceberg 已经正确集成。可以在 MySQL 中确认元数据：

```sql
-- 在 MySQL 中查看 Iceberg 自动创建的表
USE iceberg;
SHOW TABLES;

-- 查看 Iceberg 表记录
SELECT * FROM iceberg_tables;
```

## 六、PySpark 使用 Iceberg

### 6.1 基础配置

创建一个 PySpark 脚本 `iceberg_demo.py`：

```python
from pyspark.sql import SparkSession

# 创建 SparkSession，配置 Iceberg
spark = SparkSession.builder \
    .appName("IcebergDemo") \
    .config("spark.jars.packages", "org.apache.iceberg:iceberg-spark-runtime-3.5_2.12:1.10.1") \
    .config("spark.sql.extensions", "org.apache.iceberg.spark.extensions.IcebergSparkSessionExtensions") \
    .config("spark.sql.catalog.iceberg", "org.apache.iceberg.spark.SparkCatalog") \
    .config("spark.sql.catalog.iceberg.catalog-impl", "org.apache.iceberg.jdbc.JdbcCatalog") \
    .config("spark.sql.catalog.iceberg.uri", "jdbc:mysql://localhost:3306/iceberg") \
    .config("spark.sql.catalog.iceberg.jdbc.user", "root") \
    .config("spark.sql.catalog.iceberg.jdbc.password", "你的MySQL密码") \
    .config("spark.sql.catalog.iceberg.warehouse", "hdfs://localhost:9000/iceberg-warehouse") \
    .getOrCreate()

# 设置日志级别，减少干扰输出
spark.sparkContext.setLogLevel("WARN")

print("SparkSession 创建成功，Iceberg 已集成！")
print(f"Iceberg Catalog URI: jdbc:mysql://localhost:3306/iceberg")
```

### 6.2 运行方式

```bash
# 方式一：通过 spark-submit 提交
spark-submit \
  --packages org.apache.iceberg:iceberg-spark-runtime-3.5_2.12:1.10.1 \
  iceberg_demo.py

# 方式二：直接用 python 运行（需要 pyspark）
python iceberg_demo.py
```

## 七、数据分析项目：创建第一个 Iceberg 表

现在让我们为数据分析项目创建一个完整的 Iceberg 表结构：

```sql
-- 进入 spark-sql（假设已配置好 spark-defaults.conf）

-- 1. 创建业务数据库
CREATE NAMESPACE IF NOT EXISTS iceberg.demo;
DESCRIBE NAMESPACE iceberg.demo;

-- 2. 创建博客文章表（核心表）
CREATE TABLE IF NOT EXISTS iceberg.demo.posts (
    id          INT COMMENT '文章ID',
    title       STRING COMMENT '文章标题',
    content     STRING COMMENT '文章内容（摘要）',
    author      STRING COMMENT '作者',
    tags        ARRAY<STRING> COMMENT '标签列表',
    status      STRING COMMENT '发布状态：draft/published/archived',
    created_at  TIMESTAMP COMMENT '创建时间',
    updated_at  TIMESTAMP COMMENT '更新时间',
    views       INT COMMENT '阅读量',
    word_count  INT COMMENT '字数'
) USING iceberg
PARTITIONED BY (days(created_at))
TBLPROPERTIES (
    'format-version' = '2',
    'write.format.default' = 'parquet',
    'write.target-file-size-bytes' = '134217728'  -- 128MB
);

-- 3. 创建评论表
CREATE TABLE IF NOT EXISTS iceberg.demo.comments (
    id          INT COMMENT '评论ID',
    post_id     INT COMMENT '关联文章ID',
    user_name   STRING COMMENT '评论者',
    content     STRING COMMENT '评论内容',
    created_at  TIMESTAMP COMMENT '评论时间',
    likes       INT COMMENT '点赞数'
) USING iceberg
PARTITIONED BY (days(created_at))
TBLPROPERTIES ('format-version' = '2');

-- 4. 创建页面浏览日志表
CREATE TABLE IF NOT EXISTS iceberg.demo.page_views (
    event_id    STRING COMMENT '事件ID',
    post_id     INT COMMENT '文章ID',
    user_ip     STRING COMMENT '用户IP',
    user_agent  STRING COMMENT '浏览器UA',
    referrer    STRING COMMENT '来源页面',
    event_time  TIMESTAMP COMMENT '事件时间',
    duration    INT COMMENT '停留时长（秒）'
) USING iceberg
PARTITIONED BY (days(event_time))
TBLPROPERTIES (
    'format-version' = '2',
    'write.format.default' = 'parquet'
);

-- 5. 写入示例数据
INSERT INTO iceberg.demo.posts VALUES
    (1, 'Kafka 入门指南', 'Apache Kafka 是一个分布式流处理平台...',
     'browncutie', ARRAY('kafka', '大数据', '消息队列'), 'published',
     TIMESTAMP '2026-04-15 10:00:00', TIMESTAMP '2026-04-15 10:00:00', 150, 3500),
    (2, 'Spark 实战教程', 'Spark 是一个快速的大数据处理引擎...',
     'browncutie', ARRAY('spark', '大数据', '分布式计算'), 'published',
     TIMESTAMP '2026-04-15 14:00:00', TIMESTAMP '2026-04-16 08:00:00', 230, 5200),
    (3, 'HDFS 原理详解', 'HDFS 是 Hadoop 的分布式文件系统...',
     'browncutie', ARRAY('hdfs', 'hadoop', '分布式存储'), 'published',
     TIMESTAMP '2026-04-16 09:00:00', TIMESTAMP '2026-04-16 09:00:00', 89, 4800);

INSERT INTO iceberg.demo.comments VALUES
    (1, 1, '读者A', '写得很好，学到了！', TIMESTAMP '2026-04-15 12:00:00', 5),
    (2, 1, '读者B', '期待更多系列文章', TIMESTAMP '2026-04-15 14:30:00', 3),
    (3, 2, '读者C', '代码示例很实用', TIMESTAMP '2026-04-16 10:00:00', 8);

-- 6. 验证数据
SELECT id, title, author, views FROM iceberg.demo.posts;

SELECT post_id, user_name, content FROM iceberg.demo.comments ORDER BY created_at;

-- 7. 查看表结构
DESCRIBE EXTENDED iceberg.demo.posts;

-- 8. 查看分区信息
SELECT * FROM iceberg.demo.posts.partitions;

-- 9. 查看快照信息
SELECT snapshot_id, committed_at, operation, summary['added-data-files'] AS added_files
```

## 八、常见问题排查

### 问题 1：连接 MySQL 失败

```
错误信息：Access denied for user 'root'@'localhost'
```

**解决：** 检查 MySQL 用户名和密码是否正确。

```bash
# 测试 MySQL 连接
mysql -u root -p -h localhost -e "SHOW DATABASES;"
```

### 问题 2：HDFS 目录权限问题

```
错误信息：Permission denied: user=browncutie, access=WRITE
```

**解决：** 修改 HDFS 目录权限。

```bash
hdfs dfs -chmod 777 /iceberg-warehouse
# 或者更精确：
hdfs dfs -chown -R browncutie:supergroup /iceberg-warehouse
```

### 问题 3：JAR 包冲突

```
错误信息：ClassNotFoundException 或 NoSuchMethodError
```

**解决：** 检查 Spark jars 目录中是否有冲突的 JAR 包。Iceberg Runtime JAR 是一个"uber jar"，已经包含了所有需要的依赖，但可能与 Spark 自带的某些 JAR 冲突。

```bash
# 检查是否有重复的 JAR 包
ls /opt/spark/3.5.8/jars/ | grep -E "iceberg|avro|guava"

# 如果有冲突，可以移除旧版本
```

### 问题 4：Catalog 类型错误

```
错误信息：Metastore Thrift API version mismatch
```

**解决：** 这就是 Hive 4.0.1 Metastore 的兼容性问题。确保使用的是 JdbcCatalog 而不是 HiveCatalog。

检查配置：
```properties
# 正确配置
spark.sql.catalog.iceberg.catalog-impl=org.apache.iceberg.jdbc.JdbcCatalog
```
